> Часть 3: Общие структуры в функциональном дизайне / Глава 10: Моноиды / 10.9 Ответы на упражнения

- Когда оба входа — `None`, вариантов ноль: пиздуем чистый `None` (вернуть `Some` значило бы из жопы вытащить значение `A`, а мы не боги).
- Когда один вход `None`, а другой `Some`, вроде как выбор: можно кинуть либо `None`, либо `Some`. Но если выберем `None`, закон идентичности наебнётся в хлам — гляньте, `combine(empty, Some(x))` выдаст `None`, а закон орёт: "Должен быть `Some(x)`!". Короче, иллюзий не строим — обязаны вернуть `Some`.
- Когда оба входа `Some`, снова выбор: любое из значений прокатит. Но слить их в новое — хуй там, инфы о `A` ноль, а без неё комбайн не запустишь. Оба варианта законы моноида проходят на ура.

Вот такая реализация и выкатывается:

```scala
def firstOptionMonoid[A]: Monoid[Option[A]] = new:
  def combine(x: Option[A], y: Option[A]) = x orElse y
  val empty = None
```

Можем слепить другой моноид, который сначала `y` кидает, а потом `x`:

```scala
def lastOptionMonoid[A]: Monoid[Option[A]] = new:
  def combine(x: Option[A], y: Option[A]) = y orElse x
  val empty = None
```

Обобщим эту хуйню с перестановкой `x` и `y` — накатаем комбинатор, который из любого моноида dual выдаст:

```scala
def dual[A](m: Monoid[A]): Monoid[A] = new:
  def combine(x: A, y: A) = m.combine(y, x)
  val empty = m.empty
```

С `dual` можно `lastOptionMonoid` переписать через `firstOptionMonoid`, как по маслу:

```scala
def lastOptionMonoid[A]: Monoid[Option[A]] = dual(firstOptionMonoid)
```

Прикиньте, что dual даёт для моноидов, что мы раньше ковыряли. Dual от сложения интов — тот же самый интовый плюс, потому что порядок суммирования — похуй, результат вечный. Операцию зовём *коммутативной* (commutative), если порядок не имеет значения (типа `combine(x, y) == combine(y, x)`). Умножение интов и булевые — тоже коммутативные. А конкат строк и листов — нет, и dual у них отдельные сущности. Возвращаясь к `Monoid[Option[A]]`, вместо того чтоб constraints (ограничения) загнали нас в те моноиды, что мы слепили, а если constraints перелопатить? Законный моноид нужен, ассоциативность с identity (единицей) не трогаем, но можно больше инфы затребовать — конкретно, операцию `combine` накатать
