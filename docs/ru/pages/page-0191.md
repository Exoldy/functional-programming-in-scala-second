> Часть 2: Функциональный дизайн и библиотеки комбинаторов / Глава 7: Чисто функциональный параллелизм / 7.3 Алгебра API / 7.3.3 Нарушение закона: хитрый баг

На деле есть довольно подлая херня, которая вылезет в большинстве имплементаций `fork`. Когда юзаешь `ExecutorService` на пуле тредов с фиксированным размером (смотри `Executors.newFixedThreadPool`), словить дедлок — как нечего делать.<sup>14</sup> Допустим, у нас `ExecutorService` на пуле, где макс тредов — 1. Попробуй запустить этот пример на нашей текущей имплементации:

```scala
val a = lazyUnit(42 + 1)
val es = Executors.newFixedThreadPool(1)
println(Par.equal(es)(a, fork(a)))
```

Большинство имплементаций `fork` загонят этот код в дедлок. Видишь, в чём подвох? Давай ещё раз глянем на нашу реализацию `fork`:

![Страница 0191, изображение 1](./assets/page-0191/image-01.png)

> Ждёт результат одной вызываемой задачи (`Callable`) внутри другой вызываемой задачи (`Callable`)

```scala
def fork[A](a: => Par[A]): Par[A] =
es => es.submit(new Callable[A] { def call = a(es).get })
```

Заметь: сначала сабмитим `Callable`, а внутри этого `Callable` сабмитим ещё один `Callable` в `ExecutorService` и блочимся на его результате (вспомни, `a(es)` сабмитит `Callable` в `ExecutorService` и ловит `Future`). Пиздец, если пул размера `1`. Внешний `Callable` сабмитится и подхватывается единственным тредом; внутри него, до финиша, сабмитим и блочимся на втором `Callable`, но свободных тредов нет, чтоб запустить этот `Callable`. Они друг друга ждут, как два алкаша в очереди за пивом без налички — классический круглый танец, и код в дедлоке.

![Страница 0191, изображение 2](./assets/page-0191/image-02.png)

#### УПРАЖНЕНИЕ 7.9

*Жёсткое*: Покажи, что любой пул тредов фиксированного размера можно загнать в дедлок с этой имплементацией `fork`.

Когда натыкаешься на такие контрпримеры — два варианта: фиксить имплементацию, чтоб закон не хромал, или уточнить закон, явно прописав условия (типа, "требую пулы, которые растут как на дрожжах, без лимита"). Даже это полезная херня — заставляет вытащить на свет инварианты и ассумпшены, которые раньше прятались в тени. Можем ли пофиксить `fork` для фиксированных пулов? Глянем на альтернативную имплементацию:

```scala
def fork[A](fa: => Par[A]): Par[A] =
es => fa(es)
```
