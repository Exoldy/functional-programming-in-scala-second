> Часть 1: Введение в функциональное программирование / Глава 5: Строгость и ленивость / 5.1 Строгие и нестрогие функции

В Scala по умолчанию функции строгие (и почти все, что мы ковыряли до этого, были именно такими, чтоб не расслаблялись). Возьмём для примера вот эту хрень:

```scala
def square(x: Double): Double = x * x
```

Когда ты фигачишь `square(41.0 + 1.0)`, `square` жрёт уже готовенькие `42.0`, потому что строгая сука — требует вычисления аргументов заранее, как строгий препод на экзамене. А если `square(sys.error("failure"))` — привет, исключение (exception) в ебало, ещё до того, как тело функции моргнёт, потому что `sys.error("failure")` рванёт на входе. Мы все это проходили в проде, когда мина под аргументом бабахнула на код-ревью, и вся команда материлась хором.

Хотя синтаксис для нестрогости в Scala мы ещё не ковыряли толком (ну, кроме тизера с `getOrElse` и `orElse` в прошлой главе), концепт вам знаком, как родная жопа. Взять те же короткозамыкающие булевы `&&` и `||`, что везде торчат, включая Scala, — чистая нестрогая кайф.

Вы привыкли видеть в `&&` и `||` встроенный синтаксис языка, типа "это не функция, это магия", но на деле это функции, которые могут сказать аргументам: "сидите, суки, не вычисляйтесь, пока не попрошу". `&&` жрёт два `Boolean` (булевых значения), но второй трогает, только если первый `true`:

```scala
scala> false && { println("!!"); true } // does not print anything
res0: Boolean = false
```

А `||` вторую ногу шевелит, только если первая `false`:

```scala
scala> true || { println("!!"); false } // doesn't print anything either
res1: Boolean = true
```

Ещё классика нестрогости — `if` в Scala, эта ветвящаяся хуйня:

```scala
val result = if input.isEmpty then sys.error("empty input") else input
```

Хоть `if` и встроенная языковая конструкция, представьте её функцией на три параметра: условие `Boolean`, выражение типа `A` для `true`-ветки и такое же `A` для `false`. Эта `if`-функция нестрогая по определению — не жрёт все аргументы сразу, как обжора.

Точнее скажем: строгая по условию (всегда жрёт его, чтоб решить, куда сворачивать, как GPS на развилке), а по веткам `true`/`false` — чистая лень, вычисляет только одну, в зависимости от условия. В Scala нестрогие функции лепим, принимая аргументы невычисленными — типа "приходи, но не вычисляйся пока". Покажем явно (explicitly), чтоб мозг прояснить, что под капотом, а потом закинем красивый Scala-синтаксис. Вот наша нестрогая `if`:

```scala
def if2[A](cond: Boolean, onTrue: () => A, onFalse: () => A): A =
if cond then onTrue() else onFalse()
```
