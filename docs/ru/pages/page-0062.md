## 33 2.6 Заключение

![Страница 0062, изображение 1](./assets/page-0062/image-01.png)

#### РЕШЕНИЕ 2.3

```scala
def curry[A, B, C](f: (A, B) => C): A => (B => C) =
a => b => f(a, b)
```

Решение юзает тот же фокус, что мы засунули в определение `partial1`, но теперь дважды — один раз, чтоб выудить значение типа `A`, второй — для `B`. А потом пихаем эти два параметра в функцию, и вуаля, как по маслу.

![Страница 0062, изображение 2](./assets/page-0062/image-02.png)

#### РЕШЕНИЕ 2.4

```scala
def uncurry[A, B, C](f: A => B => C): (A, B) => C =
(a, b) => f(a)(b)
```

Опять тот же трюк из `partial1` и `curry`, но теперь возвращаем функцию, которая жрёт два параметра: значение типа `A` и ещё одно типа `B`. Внутри этой функции лепим `a` к оригинальной `f`, и получаем новую функцию типа `B => C`. Потом прилепляем `b` к этой херне, и вот тебе финальное значение типа `C`. Классика curry (куринг), пацаны, без неё никуда.

![Страница 0062, изображение 3](./assets/page-0062/image-03.png)

#### РЕШЕНИЕ 2.5

```scala
def compose[A, B, C](f: B => C, g: A => B): A => C =
a => f(g(a))
```

Возвращаем анонимную функцию от `A` до `C`. Внутри неё у нас параметр `a` плюс внешние значения `f: B => C` и `g: A => B`. Единственный способ сдвинуться с мёртвой точки — прилепить `a` к функции `g`, и вуаля, значение типа `B`. С этим `B` только одно можно сделать — скормить ему `f`, и тип ровно тот, что нам и нужен на выходе. Как в пазле, где последняя деталька идеально ложится, после того как ты уже охренел от всей этой хуйни.
