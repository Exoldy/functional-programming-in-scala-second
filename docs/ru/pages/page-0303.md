#### ОТВЕТ 10.5

Сворачиваем, стартуя с пустышки моноида, и склеиваем накопленную хрень с тем, что текущий элемент через переданную функцию выдаст — классика, как конвейер на заводе, где первый блин не комом.

```scala
def foldMap[A, B](as: List[A], m: Monoid[B])(f: A => B): B =
as.foldLeft(m.empty)((b, a) => m.combine(b, f(a)))
```

#### ОТВЕТ 10.6

Давайте глянем на сигнатуру этого `foldRight`:

```scala
def foldRight[A, B](as: List[A])(acc: B)(f: (A, B) => B): B = ???
```

Ключевой инсайт — это как фокус с кроликом из шляпы: каррируем `f`, превращаем из `(A, B) => B` в `A => (B => B)`. Потом `foldMap` по списку с этой закаррированной хуйнёй и нашим старым добрым `endoMonoid`. Выплюнет одну функцию `B => B`, которую хлопнем на начальную `acc` — и вуаля.

```scala
def foldRight[A, B](as: List[A])(acc: B)(f: (A, B) => B): B =
foldMap(as, endoMonoid)(f.curried)(acc)
```

Компилятся, сука, но результаты — полная жопа. Взгляните на примерок:

```scala
scala> Monoid.foldRight(List("a", "b", "c"))("")(_ + _)
val res0: String = cba
```

Проблема в том, что композиция функций идёт в обратку, как пайпы в bash'е, если их вверх ногами склеить. Фиксим флипом параметров `endoMonoid` через его dual — переворачиваем логику, и порядок встает на место, как в нормальном код-ревью после пинка.

```scala
def foldRight[A, B](as: List[A])(acc: B)(f: (A, B) => B): B =
foldMap(as, dual(endoMonoid))(f.curried)(acc)
```
