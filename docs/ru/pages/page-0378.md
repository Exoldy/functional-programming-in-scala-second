#### ОТВЕТ 12.15

Самый простой подход — нарыть контрпример, тип данных, у которого есть инстанс `Foldable`, но хуй там с `Functor`. Взгляните на этот красавец:

```scala
case class Iteration[A](a: A, f: A => A, n: Int)
```

У этого типа вполне законный инстанс `Foldable`, без подвохов:

```scala
given iterationFoldable: Foldable[Iteration] with
extension [A](i: Iteration[A])
override def foldMap[B](g: A => B)(using m: Monoid[B]): B =
def iterate(n: Int, b: B, c: A): B =
if n <= 0 then b else iterate(n - 1, g(c), i.f(i.a))
iterate(i.n, m.empty, i.a)
```

А когда пытаемся забабахать `Functor[Iteration]` — бам, типы нас посылают нахуй:

```scala
given iterationFunctor: Functor[Iteration] with
extension [A](i: Iteration[A])
def map[B](g: A => B): Iteration[B] =
Iteration(g(i.a), b => ???, i.n)
```

Типы намекают: тебе надо слепить функцию `B` `=>` `B`, а у тебя в арсенале только `i.f:` `A` `=>` `A` да `g:` `A` `=>` `B`. Ни хуя их не склеишь под `B` `=>` `B` — как ни ебись. Думайте так: `Foldable` позволяет пробежаться по каждому элементу, как по списку гостей на код-ревью, `Functor` даёт поменять элементы, не сломав общую хуйню структуры. А `Foldable`? Не даёт тебе даже пальцем тронуть новые значения этого foldable-типа — сиди и грусти.

#### ОТВЕТ 12.16

Один из способов думать про `mapAccum` — каждый элемент в `F[A]` заменяется на трансформированный кусок из `B` (получается `F[B]`), но в отличие от `map`, тут вычисления висят на накопленном состоянии, как рюкзак туриста в походе по FP-джунглям. С этой моделью `reverse` сводится к замене элемента на его собеса из реверса на той же позиции. Бабахиваем это так: сливаем входной `F[A]` в список, реверсим его к хуям, и запускаем `mapAccum`-проход. На каждом элементе сбрасываем входной `A`, суём голову оставшегося реверснутого списка вместо него, и возвращаем хвост оставшихся
