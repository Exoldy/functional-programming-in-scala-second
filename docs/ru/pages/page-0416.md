## 387 13.8 Почему тип IO не справляется со стриминговым I/O

С `Files` в арсенале мы могли бы набросать желаемую программу вот так, как будто это фриланс-задачка на час:

```scala
def convert[F[_]](using f: Files[F]): F[Unit] =
  for
    lines <- f.readLines("fahrenheit.txt")
    cs = lines.map(s => fahrenheitToCelsius(s.toDouble).toString)
    _ <- f.writeLines("celsius.txt", cs)
  yield ()
```

Работает, конечно, но жрёт весь `fahrenheit.txt` в память целиком — а если файл терабайтный, то привет, OutOfMemoryError (ошибка нехватки памяти), и вся твоя JVM в ахуе. Нам бы хуярить с постоянным куском памяти, как нормальный стриминг: читать по строчке или буферу строк из `fahrenheit.txt`, конвертить в Цельсии, срать в `celsius.txt` и крутить по кругу. Для такой оптимизации можно выкатить низкоуровневый файловый API (application programming interface) с доступом к хэндлам I/O — типа, отдаём ресурсы в аренду, чтоб не плодить новые IO на каждом шагу:

```scala
trait Files[F[_]]:
  def openRead(file: String): F[HandleR]
  def openWrite(file: String): F[HandleW]
  def readLine(h: HandleR): F[Option[String]]
  def writeLine(h: HandleW, line: String): F[Unit]

trait HandleR
trait HandleW
```

Единственная засада — теперь приходится лепить монолитный цикл, как в старые добрые времена imperative-кода, когда рекурсия была роскошью для элиты:

```scala
def loop[F[_]](
  hr: HandleR, hw: HandleW
)(using f: Files[F]): F[Unit] =
  f.readLine(hr).flatMap:
    case None => IO.unit(())
    case Some(s) =>
      f.writeLine(hw, fahrenheitToCelsius(s.toDouble))
        .flatMap(_ => loop(f, c))
```

> Вместо for-comprehension (for-выражений) юзаем явный flatMap, чтоб монадическая рекурсия села строго в хвостовую позицию — иначе стек переполнится, и привет, StackOverflowException (ошибка переполнения стека) на миллиарде строк.

```scala
def convertFiles[F[_]](using f: Files[F]): F[Unit] =
  for
    hr <- f.openRead("fahrenheit.txt")
    hw <- f.openWrite("celsius.txt")
    _ <- loop(hr, hw)
  yield ()
```

В принципе, ничего криминального в таком монолите нет, но компоновать его — как пытаться слепить Lego из куска бетона. Допустим, потом решишь добавить скользящее среднее по пяти температурам — ковыряться в `loop` придётся с матами и кофе литрами. А теперь сравни с кодом на `List`: там просто определяешь функцию `movingAvg` и лепишь её до или после конвертации в Цельсии — композабельно, как в FP-мечтах, без единого пота.
