## 65 3.6 Ответы на упражнения

![Страница 0094, изображение 1](./assets/page-0094/image-01.png)

#### ОТВЕТ 3.26

```scala
enum Tree[+A]:
def depth: Int = this match
case Leaf(_) => 0
case Branch(l, r) => 1 + (l.depth.max(r.depth))
```

Эта хрень жрёт любое дерево на завтрак, так что лепим её методом прямо на тип `Tree`. 
Берём ту же структурную рекурсию, что и в `maximum` — проверенная временем схема, как старый добрый шаблон из FP-богачей. 
Разница чисто в том, что справа от `=>` подкидываем свои значения под каждый матч, и вуаля, дерево перелопачивается без единого мутабельного вздоха.

![Страница 0094, изображение 2](./assets/page-0094/image-02.png)

#### ОТВЕТ 3.27

```scala
enum Tree[+A]:
def map[B](f: A => B): Tree[B] = this match
case Leaf(a) => Leaf(f(a))
case Branch(l, r) => Branch(l.map(f), r.map(f))
```

Реализация — один в один первая версия `map` на `List`: структурная рекурсия, матчим по веткам, и на каждом шаге лепим свежий экземпляр того же конструктора данных. 
В `List` мы конвертили `Nil` в `Nil`, а `Cons` в `Cons`. 
А тут в `Tree` каждый `Leaf` превращаем в `Leaf`, каждый `Branch` — в `Branch`, и рекурсивно прогоняем `map` по веткам. 
Как map, только для FP-деревьев, без лишнего говна в памяти.

![Страница 0094, изображение 3](./assets/page-0094/image-03.png)

#### ОТВЕТ 3.28

Сначала зафигачим `fold`:

```scala
enum Tree[+A]:
def fold[B](f: A => B, g: (B,B) => B): B = this match
case Leaf(a) => f(a)
case Branch(l, r) => g(l.fold(f, g), r.fold(f, g))
```

Матчим по структуре дерева, как по анатомии ебаного FP-монстра. 
Наткнулись на листок — трансформируем его значение через подсунутую `f: A => B`, чисто и без побочек. 
Упёрлись в ветку — сворачиваем каждое поддерево в fold'e, а потом склеиваем результаты через `g: (B, B) => B`. 
Универсальный хак, пацаны, теперь все предыдущие функции лепятся из этого одним махом, без дублирования кода — чистый DRY в духе Scala-продакшена.

```scala
enum Tree[+A]:
def size: Int =
fold(a => 1, 1 + _ + _)
```
