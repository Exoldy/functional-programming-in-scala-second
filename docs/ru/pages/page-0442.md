## 413 15.1 Проблемы с императивным I/O: пример

```scala
lines
.filter(!_.trim.isEmpty)
.take(40000)
.map(_.head)
.indexOfSlice("abracadabra".toList)
```

Мы хотим лепить что-то вроде этого предыдущего примера при чтении из реального файла. Проблема в том, что у нас нет `LazyList[String]` под рукой; у нас файл, из которого можно сосать данные. Можем наебать систему, написав функцию `lines`, которая возвращает `IO[LazyList[String]]`:

```scala
def lines(filename: String): IO[LazyList[String]] = IO:
val src = io.Source.fromFile(filename)
src.getLines().to(LazyList).append {
src.close
LazyList.empty
}
```

Это называется *ленивый I/O* (lazy I/O). Хитрим мы тут по-чёрному, потому что `LazyList[String]` внутри монады `IO` — это не чистое значение, а такой себе волк в овечьей шкуре. Как только элементы стрима форсишь — бац, побочки вылазят, файл читается, а закроется он только если весь стрим до жопы проглотишь. Ленивый I/O заманчиво выглядит — композицию частично возвращает, как старого друга с пивом, — но на деле это мина замедленного действия по нескольким фронтам:

- *Не безопасно для ресурсов, блядь.* Ресурс (файл тут) освободится только при полном проходе стрима до конца. А мы ж любители early exit (early exit): `exists` свалит при первом совпадении, не дойдя до финиша, и каждый такой раз — привет, утечка ресурсов, file handle'ы (file handles) висят как мертвецы в продакшене.

- *Ничто не мешает прогнать тот же `LazyList` заново после закрытия файла.* Кончится это либо memory bomb'ой (memory bomb; если мемоизирует — все элементы в куче осядут, как хлам в гараже), либо пиздецом с чтением из закрытого хэндла (file handle; если не мемоизирует — второе чтение рвёт всё нахуй).

- *Форсинг элементов стрима — это побочка I/O (I/O side effect), так что два треда на одном `LazyList`, и привет, рандомный пиздец.* Непредсказуемость на уровне "угадай, что вылетит".

- *В реальной жизни ты не всегда в курсе, что там с `LazyList[String]` творится.* Передашь его в чужую функцию — она запихнёт в структуру данных на месяцы, прежде чем глянуть. Правильное использование теперь требует телепатии: нельзя просто манипулировать как чистым значением, надо знать его грязное происхождение. А это композицию ебёт в рот — по идее, тип должен хватать, и всё.
