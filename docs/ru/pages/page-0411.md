> Часть 4: Эффекты и I/O / Глава 13: Внешние эффекты и I/O / 13.5 Неблокирующий и асинхронный I/O / 13.5.1 Композиция свободных алгебр

Короче, перелопачиваем дефы под variance, чтоб компилятор не срал кирпичами на код-ревью. Когда параметр типа ковариантным делаем, каждое контравариантное использование (компилятор тебе чётко ткнёт) заменяем на свежий тип-параметр — супертип нашего ковариантного. Взять `flatMap`: Scala орёт, что `F` в контравариантной позиции торчит, ну и ввели новый тип-пар `F2[x]` `>:` `F[x]`, заменив контравариантный `F` на `F2`. С этой новой обёрткой `Free` `union` становится проще, как после рефакторинга в Cats Effect — без лишней хуйни:

```scala
enum Free[+F[_], A]:
...
def union[G[_]]: Free[[x] =>> F[x] | G[x], A] = this
```

Можно даже `union` обобщить до новой операции, где явно тип для `F` пихаем, не повторяя всю сигнатуру выражения, как в каком-то verbose Java:

```scala
enum Free[+F[_], A]:
...
def covary[F2[x] >: F[x]]: Free[F2, A] = this
```

Операция `covary` новой магией не блещет — чистый сахарок для синтаксиса, чтоб не тошнило от boilerplate'а. Для любого значения `fa:` `Free[F,` `A]` и типа `F2[x]` `>:` `F[x]` `fa.covary[F2]` всегда можно расписать как `fa:` `Free[F2,` `A]`. Оба этих трюка — `covary` и `union` — больше намекают на подводные камни inference'а в Scala, чем тащат свежий функционал. Ладно, возвращаемся к `cat` — с новой дефой `Free` композим free-программы в алгебрах `Files` и `Console` с минимумом копипасты, как в нормальном FP-продакшене:

```scala
def cat(file: String): Free[[x] =>> Files[x] | Console[x], Unit] =
Files.readLines(file).flatMap: lines =>
Console.printLn(lines.mkString("\n"))
```

Тип для дефа `cat` приходится явно впихивать — Scala union сам не выведет, ленивый сука. Но это не подвох, а святое дело: явные типы возврата в методах — как ремни безопасности в тачке, все нормальные так ездят.
