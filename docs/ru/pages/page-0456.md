## 427 15.3 Расширяемые пулы и стримы

А вот операция ``uncons`` – это уже поинтереснее, пацаны, как турбонаддув на старой тачке. Вспомните предыдущее определение, чистенькое такое:

```scala
def uncons: Pull[Nothing, Either[R, (O, Pull[O, R])]] =
Pull.done >> Result(step)
```

Подгоним это под эффектный вариант пула – замутим узел ``Eval``, который завернёт вызов ``step``, чтоб не ебаться с голым железом:

```scala
def uncons[F2[x] >: F[x], O2 >: O, R2 >: R](
using Monad[F2]
): Pull[F2, Nothing, Either[R2, (O2, Pull[F2, O2, R2])]] =
Eval(step)
```

Но бля, раз мы тыкаем в ``step``, то сразу монадный constraint на эффектный тип навязывается, как штраф за парковку в центре. И привет – теперь ``uncons`` на чистом, неэффектном пуле не прокатит, том самом с ``F` `=` `Nothing``, потому что инстанса ``Monad[Nothing]`` нету, сука. То же дерьмо и с любой операцией, что юзает ``uncons``, типа ``take`` – все потащат за собой ``Monad[F]``, как цепной пёс. Фиксим по-умному: отложим вызов ``step`` до момента, когда пул реально начнёт крутиться. Замутим свежий конструктор данных ``Uncons``, чтоб ссылку на исходный пул припрятать. А потом апгрейдим ``step``, чтоб этот гад хэндлил:

```scala
case Uncons(source: Pull[F, O, R])
extends Pull[F, Nothing, Either[R, (O, Pull[F, O, R])]]
def step[F2[x] >: F[x], O2 >: O, R2 >: R](
using F: Monad[F2]
): F2[Either[R2, (O2, Pull[F2, O2, R2])]] =
this match
case Result(r) => F.unit(Left(r))
case Output(o) => F.unit(Right(o, Pull.done))
case Eval(action) => action.map(Left(_))
case Uncons(source) =>
source.step.map(s => Left(s.asInstanceOf[R2]))
case FlatMap(source, f) =>
source match
case FlatMap(s2, g) =>
s2.flatMap(x => g(x).flatMap(y => f(y))).step
case other => other.step.flatMap:
case Left(r) => f(r).step
case Right((hd, tl)) => F.unit(Right((hd, tl.flatMap(f))))
```

Определение ``Uncons`` – чистый кайф, оно растягивает всю цепочку ``Pull[F,` `Nothing,` `Either[R,` `(O,` `Pull[F,` `O,` `R])]]`` и держит под капотом базовый ``Pull[F,` `O,` `R]``. Тип орёт: unconsишь ``Pull[F,` `O,` `R]`` – и вуаля, пул, который может пошарить в эффектах ``F``, элементов не выплюнет, а в финале либо финалка ``R`` от исходника, либо выходное значение ``O`` плюс остаток ``Pull[F,` `O,` `R]``. Как ленивый мем с котом – "позже разберёмся", но работает как часы в проде.
