# Страница 0461

[<- Страница 0460](./page-0460) | [Указатель страниц](./) | [Страница 0462 ->](./page-0462)

> Часть 4: Эффекты и I/O / Глава 15: Обработка потоков и инкрементальный ввод-вывод / 15.3 Расширяемые pull'ы и стримы / 15.3.2 Обработка ошибок

```scala
val a: Task[Int] = Task("asdf".toInt)
val b: Task[Int] = a.map(_ + 1)
val c: Task[Int] = b.handleErrorWith(_ => Task(0))
```

``handleErrorWith`` — это как тёмный близнец ``flatMap``: даёт секвенсинг, но не по каналу результата, а по ошибкам, чтоб те не вылезли в самый неподходящий момент. А если мы их скрестим — ``Task`` с ``Stream``? Можем юзать ``eval`` ровно так же, как с ``IO``, без лишнего геморроя:

```scala
val s: Stream[Task, Int] = Stream.eval(Task("asdf".toInt))
```

Когда этот стрим конвертим в таски через любой элиминатор (типа ``toList`` или ``run``), вылезет фейл, который как матрёшка обернёт ``NumberFormatException``. Полезно хэндлить такие приколы прямо в API ``Stream`` и ``Pull``, чтоб не бегать потом с дебаггером по логам в три ночи. Так что лепим к каждому ``handleErrorWith``:

```scala
enum Pull[+F[_], +O, +R]:
case Handle(
source: Pull[F, O, R],
handler: Throwable => Pull[F, O, R]
) extends Pull[F, O, R]
def handleErrorWith[F2[x] >: F[x], O2 >: O, R2 >: R](
handler: Throwable => Pull[F2, O2, R2]): Pull[F2, O2, R2] =
Pull.Handle(this, handler)
extension [F[_], O](self: Stream[F, O])
def handleErrorWith(handler: Throwable => Stream[F, O]): Stream[F, O] =
Pull.Handle(self, handler)
```

Добавили свежий конструктор в ``Pull`` — ловит оригинальный pull и хэндлер ошибок, чтоб не потерялись в потоке. Теперь апгрейдим ``step``, чтоб он интерпретировал этот ``Handle``-конструктор. При степе натыкаемся на ``Handle`` — степпим внутренний pull и ловим все ошибки, что вылезут, как в продакшене на Black Friday. Но вот подвох: ``step`` дефайнится для любого эффекта ``F[_]`` с монадой, а монада — она про композицию, а не про "ой, бля, ошибка". Чтоб не мучаться, вводим новый type class — добавляет к ``Monad`` суперсилу хэндлить ошибки:

```scala
trait MonadThrow[F[_]] extends Monad[F]:
extension [A](fa: F[A])
def attempt: F[Try[A]]
def handleErrorWith(h: Throwable => F[A]): F[A] =
attempt.flatMap:
case Failure(t) => h(t)
case Success(a) => unit(a)
def raiseError[A](t: Throwable): F[A]
```

``MonadThrow`` расширяет ``Monad`` и даёт рейзить ошибки, а потом их хэндлить — как нормальный взрослый FP-код, а не детский сад с unchecked exceptions.
