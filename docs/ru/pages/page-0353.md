# Страница 0353

> Часть 3: Типовые структуры в функциональном дизайне / Глава 12: Аппликативные и траверсибельные функторы / 12.4 Плюшки аппликативных функторов / 12.4.1 Не все аппликативы — монады, и это нормально

```scala
validBirthdate(birthdate),
validPhone(phone)
)(WebForm(_, _, _))
```

Если хоть одна функция (или все сразу) выдаст `Invalid`, то вся наша `validateWebForm`-метода соберёт эти фейлы в один жирный комок. Заметьте, мы юзаем `List[String]` как тип ошибки — это позволяет возвращать пачку ошибок разом (нужен инстанс `Monoid[List[A]]`, который валяется в гитхаб-репозитории). 

Минус такой схемы: приходится ковыряться в кейсе с пустым `Invalid(Nil)`. Что за пиздец — валидация накрылась, а ошибок ноль? Давай запилим защиту от этой хуйни, сменим тип для накопления ошибок. Вместо `List[String]` сделаем список, который по определению не пустой:

```scala
case class NonEmptyList[+A](head: A, tail: List[A]):
def toList: List[A] = head :: tail
object NonEmptyList:
def apply[A](head: A, tail: A*): NonEmptyList[A] =
NonEmptyList(head, tail.toList)
```

Проблема решена? Хуй там, пацаны. Вспомните: аппликатив для `Validated` требовал монаойд для типа ошибки. Давайте попробуем слепить инстанс `Monoid[NonEmptyList[A]]`:

```scala
given nelMonoid: Monoid[NonEmptyList[A]] with
def combine(x: NonEmptyList[A], y: NonEmptyList[A]) =
NonEmptyList(x.head, x.tail ++ y.toList)
val empty: NonEmptyList[A] = ???
```

`combine` — это раз плюнуть, но с `empty` встряли в полный тупик. Пустой непустой список не слепить, а одиночный — тоже хуйня, потому что значение типа `A` из жопы не вытащишь (да и если б синглтон запилили, инстанс всё равно обосрался бы на законах идентичности монаойда). 

А теперь гляньте поближе на определение `Applicative[Validated[E, _]]` — наша имплементация жрала только `combine`, а до `empty` не добралась. Вытащим из `Monoid` трейт чисто с `combine`. Назовём его `Semigroup`<sup>4</sup>, но можно и как угодно — хоть `SemigroupMotherfucker`:

```scala
trait Semigroup[A]:
def combine(a1: A, a2: A): A
trait Monoid[A] extends Semigroup[A]:
def empty: A
```

<sup>4</sup> Имя `Semigroup` из абстрактной алгебры, где копошатся с *группами* — это монаоиды с инверсом для каждого элемента, типа "разъебал — собери обратно". Семигруппы, как ясно из названия, полегче: комбайн ассоциативный и на множестве закрытый, без лишнего дроча. Теория групп — это зоопарк алгебраических уродцев, но в FP-тусовке семигруппы и монаоиды — короли бала, остальные в клетках.
