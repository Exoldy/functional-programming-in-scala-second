## 35 3.1 Определение функциональных структур данных

Эта глава — сплошной полигон для упражнений, особенно по написанию и обобщению чистых функций. Некоторые из них вломят по полной, как баг в продакшене перед дедлайном; ответы дам, но сначала сам ковыряйся в каждом, чтоб техники въехали в башку намертво. Практика — мать FP, без неё никуда. Ещё загляни на наш GitHub (https://github.com/fpinscala/fpinscala) — там среда для сборки упражнений и ответов, чтоб не ебаться с настройкой.

### 3.1 Определение функциональных структур данных

Функциональная структура данных — это херня, которую трогают только чистыми функциями, без мутаций и побочек. Помнишь, чистая функция не меняет данные на месте и не гадит сайд-эффектами? Вот поэтому функциональные структуры по определению неизменяемые, как закон сохранения энергии. Пустой список (пишется `List()` или `Nil` в Скале) вечен и неизменяем, как целые `3` или `4`. И вычисление `3` `+` `4` рождает новый номер `7`, не трогая ни `3`, ни `4`. Точно так же конкатенация двух списков (синтаксис `a` `++` `b` для `a` и `b`) выдаёт новый список, а старые оставляет нетронутыми. А это ж значит тонну лишнего копирования данных? Ха, неожиданно, но нет, и мы разберём, почему так. Но сперва разберём самую ходовую функциональную хрень — однонаправленный связный список. Определение здесь в духе (хоть и проще) `List` из стандартной либы Скалы. В этой главе своём `List` напишем и будем юзать, а дальше перейдём на встроенный `List`. Следующий листинг кидает кучу новой синтаксической хуйни и концептов — разберём по полочкам.

**Листинг 3.1** Однонаправленные связные списки

```scala
package fpinscala.datastructures
```

> Конструктор данных List для пустого списка

> Ещё один конструктор для непустых списков. Заметил, tail — это тоже List[A], может быть Nil или другой Cons.

```scala
enum List[+A]:
  case Nil
  case Cons(head: A, tail: List[A])
```

> Это компаньон-объект List. Там функции для создания и работы со списками.

```scala
object List:
  def apply[A](as: A*): List[A] =
    if as.isEmpty then Nil
    else Cons(as.head, apply(as.tail*))
```

> Это вариадический синтаксис функции, чтоб лепить List[Int] как List(1, 2, 3).

> List — тип данных, параметризованный типом A.

Сначала глянем на определение типа данных — начинается с ключевого слова `enum`. `enum` — это тип данных из кучи *конструкторов данных*, каждый через `case`. Скальские enum похожи на Java/C, но не ограничены голыми значениями (типа `Red`, `Green`, `Blue`), а могут жрать любые аргументы.[^1]

[^1]: Enum ввели в Scala 3. В Scala 2 юзали `sealed trait` (запечатанный трейт) с пачкой `case class` и `case object`, где каждый кейс — подтип трейта. Подробности в секции про enum vs sealed traits (enum против sealed traits) в конце главы.
