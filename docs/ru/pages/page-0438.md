Портировать оригинальное определение `qs` — проще простого, логика не меняется ни на йоту. 
Единственный хак — возвращай тот самый `ST[S, Unit]`, который мы слепили, вместо классического `Unit`. 
И вуаля, без лишнего говна.

```scala
def qs[S](a: STArray[S,Int], l: Int, r: Int): ST[S, Unit] =
if l < r then for
pi <- partition(a, l, r, l + (r - l) / 2)
_ <- qs(a, l, pi - 1)
_ <- qs(a, pi + 1, r)
yield () else ST[S, Unit](())
```

#### ОТВЕТ 14.3

Давай замутим класс-обёртку вокруг `HashMap[K, B]`, точь-в-точь как с `STArray`: 
в компаньоне методы-конструкторы, чтоб задать начальный `HashMap`. 
А потом навешаем на класс методы под каждую операцию — чтение или мутация мапы, — 
и завернём делегированный вызов в `ST`. 
Теперь эта хрень под присмотром, как пёс на цепи, без побегов в мутабельный ад.

```scala
final class STMap[S, K, V] private (private val table: HashMap[K, V]):
def size: ST[S, Int] = ST(table.size)
def apply(k: K): ST[S, V] = ST(table(k))
def get(k: K): ST[S, Option[V]] = ST(table.get(k))
def +=(kv: (K, V)): ST[S, Unit] = ST(table += kv)
def -=(k: K): ST[S, Unit] = ST(table -= k)
object STMap:
def empty[S, K, V]: ST[S, STMap[S, K, V]] =
ST(new STMap(HashMap.empty))
def fromMap[S, K, V](m: Map[K, V]): ST[S, STMap[S, K, V]] =
ST(new STMap((HashMap.newBuilder ++= m).result()))
```
