#### ОТВЕТ 3.5

```scala
def dropWhile[A](as: List[A], f: A => Boolean): List[A] =
  as match
    case Cons(hd, tl) if f(hd) => dropWhile(tl, f)
    case _ => as
```

Мы тут паттерн-матчим на списке, но кейсы подкрутили под задачу — не как обычно. Первый ловит `Cons`, где голова проходит предикат, как строгий вышибала у входа: pattern guard (guard паттерна) в деле, это if-ушка на паттерне — `if <condition>`. Биндим `hd` и `tl` на голову и хвост этого `Cons` и вешаем guard `if f(hd)`. Матчит только если `f(hd)` истинно, тогда рекурсивно `dropWhile` на `tl` — и вперёд. Остальные случаи — `Cons` с фейловой головой или чистый `Nil` — летят в wildcard и возвращают исходный список как есть. Кстати, guards тут чисто для красоты, можно обойтись без них и замэтчить чисто по структуре списка:

```scala
def dropWhile[A](as: List[A], f: A => Boolean): List[A] =
  as match
    case Cons(hd, tl) =>
      if f(hd) then dropWhile(tl, f)
      else as
    case Nil => as
```

Оба подхода заебись для такой мелкой хуйни. Знай оба приёма, как старый волк, и выбирай по ситуации тот, что проще читать — без лишнего геморроя на код-ревью.

#### ОТВЕТ 3.6

```scala
def init[A](as: List[A]): List[A] = as match
  case Nil => sys.error("init of empty list")
  case Cons(_, Nil) => Nil
  case Cons(hd, tl) => Cons(hd, init(tl))
```

Паттерн-матчим на входном списке, как в мясорубку. `Nil` — сразу exception в ебало, как мы в `tail` делали, чтоб не трындеть зря. `Cons(_, Nil)` — список из одного элемента, сливаем его нахуй и `Nil` назад. Иначе — `Cons(hd, tl)`, где `tl` точно непустой (иначе бы уже зацепило выше, Scala не дура). Значит, берём `init` от хвоста и cons'им `hd` спереди на результат.
