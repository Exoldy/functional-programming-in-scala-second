> Глава 4: Обработка ошибок без исключений /  
> 4.4 Тип данных Either /  
> 4.4.1 Накопление ошибок

С этой новой функцией мы можем слепить альтернативу `Person.make`, которая вернёт обе ошибки разом, без этой хуйни с выбором одной:

```scala
object Person:
  def makeBoth(name: String, age: Int): Either[List[String], Person] =
    map2Both(Name(name), Age(age), Person(_, _))
```

Когда оба входа полное говно, мы получим обе ошибки в списке, аккуратно упакованном в `Left`:

```scala
scala> val p = Person.makeBoth("", -1)
val p: Either[List[String], Person] =
Left(List(Name is empty., Age is out of range.))
```

К сожалению, `map2Both` — штука с ограниченной полезностью, как тот ножичек из армейского НЗ, который и хлеб режет, и консерву вскрывает, но на большее не тянет. Взгляните, что творится, когда пытаемся слить результаты двух вызовов `Person.makeBoth`:

```scala
scala> val p1 = Person.makeBoth("Curry", 34)
val p1: Either[List[String], Person] = Right(Person(Name(Curry),Age(34)))
scala> val p2 = Person.makeBoth("Howard", 44)
val p2: Either[List[String], Person] = Right(Person(Name(Howard),Age(44)))
scala> val pair = map2Both(p1, p2, (_, _))
val pair: Either[List[List[String]], (Person, Person)] =
Right((Person(Name(Curry),Age(34)),Person(Name(Howard),Age(44))))
```

Компилится на ура, но присмотритесь к inferred-типу (выводимому типу) `pair` — левая сторона `Either` теперь с вложенными списками! Каждый следующий `map2Both` наращивает ещё один слой `List` к типу ошибки, как матрёшка из ошибок, блядь, бесконечная рекурсия в аду. Чиним это лёгким тюнингом `map2Both`. Заставим входные значения уже тащить `List[E]` слева. Назовём эту мутировавшую версию `map2All`:

```scala
def map2All[E, A, B, C](
  a: Either[List[E], A],
  b: Either[List[E], B],
  f: (A, B) => C
): Either[List[E], C] =
  (a, b) match
    case (Right(aa), Right(bb)) => Right(f(aa, bb))
    case (Left(es), Right(_)) => Left(es)
    case (Right(_), Left(es)) => Left(es)
    case (Left(es1), Left(es2)) => Left(es1 ++ es2)
```

Кроме подкрутки типов параметров `a` и `b`, отличия только в ветках с `Left`-значениями. Если одинокий `Left` прилетел, отдаём его как есть (не оборачивая в singleton-list (одиночный список), как в старом `map2Both`). Если оба входа — `Left`, конкатим их и оборачиваем в `Left`. С `map2All` держим тип в узде, пока склеиваем сколько угодно `Either`-ов — чистый кайф, без этой вложенной хуйни.
