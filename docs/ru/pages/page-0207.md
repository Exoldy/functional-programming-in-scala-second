> Часть 2: Функциональный дизайн и библиотеки комбинаторов / Глава 7: Чисто функциональный параллелизм / 7.6 Ответы на упражнения

Чтобы набить `flatMap` на `join`, проще пареной репы: проапай `f` по исходному `pa` — бац, и у тебя `Par[Par[B]]`. Дальше просто джойн её, и привет:

```scala
extension [A](pa: Par[A]) def flatMap[B](f: A => Par[B]): Par[B] =
join(pa.map(f))
```

А вот `join` через `flatMap` — это уже поебаться посерьёзнее, подвох на подвохе, как в продакшене с таймингаутами. `flatMap` позволяет перекрутить `Par[X]` в `Par[Y]`, если подкинешь функцию `X` `=>` `Par[Y]`. Фишка в том, чтоб взять `X` `=` `Par[A]` и `Y` `=` `A` — чисто как "hold my beer", мемная классика. Короче, нужна функция-идентитета `Par[A]` `=>` `Par[A]`, чтоб ничего не сломать:

```scala
def join[A](ppa: Par[Par[A]]): Par[A] =
ppa.flatMap(identity)
```
