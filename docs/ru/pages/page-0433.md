### 14.2.5 Чисто функциональная quicksort на месте (in-place quicksort)

Компоненты для `quicksort` теперь пишутся в `ST` на одном дыхании, без этой бесконечной возни с побочками. 
Взять хотя бы `swap` — функцию, которая свапает два элемента в массиве: хуйня вопрос, метод на `STArray`, и поехали:

```scala
def swap(i: Int, j: Int): ST[S, Unit] =
  for
    x <- read(i)
    y <- read(j)
    _ <- write(i, y)
    _ <- write(j, x)
  yield ()
```

#### УПРАЖНЕНИЕ 14.2

Напиши чисто функциональные версии `partition` и `qs` — чтоб всё было по-честному, без подглядываний в мутабельный ад:

```scala
def partition[S](a: STArray[S, Int], l: Int, r: Int, pivot: Int): ST[S, Int]
def qs[S](a: STArray[S, Int], l: Int, r: Int): ST[S, Unit]
```

С этими примитивами `quicksort` собирается из них в монаде `ST`, как пазл из Lego — клик-клик, и готово:

```scala
def quicksort(xs: List[Int]): List[Int] =
  if xs.isEmpty then xs else ST.run([s] => () =>
    for
      arr <- STArray.fromList[s, Int](xs)
      size <- arr.size
      _ <- qs(arr, 0, size - 1)
      sorted <- arr.freeze
    yield sorted
  )
```

Видишь, монада `ST` даёт нам писать чистые функции, которые тем не менее ебут данные в мутацию, но только под присмотром. 
Типовая система Скалы — как строгий сержант на плацу: не даст слить всё в одну кучу и накосячить по-глупому. 
Прошёл через это в проде — знаю, где собака зарыта.
