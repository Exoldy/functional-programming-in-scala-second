## 401 14.2 Тип данных для принуждения области видимости побочных эффектов

```scala
x
<- r1.read
y
<- r2.read
_
<- r1.write(y + 1)
_
<- r2.write(x + 1)
a
<- r1.read
b
<- r2.read
yield (a, b)
```

Теперь мы готовы завести эту хуйню ``run``, которая вызовет полиморфную функцию, сунув ей произвольный тип для ``S``. Поскольку экшн полиморфный по ``S``, он гарантированно нихуя не юзает значение, которое ему впихнули — как кот в коробке из мема, который даже не чешется. Короче, пиздец безопасно впарить ``()`` типа ``Unit``! Определим эту ``run`` в компаньоне ``ST``, где все знают, что ``ST[S,`` ``A]`` — это тот же самый ``S`` ``=>`` ``(A,`` ``S)``, только в другой обёртке, как шаманский ритуал для продакшена.

```scala
object ST:
def run[A](st: [s] => () => ST[s, A]): A =
val su: ST[Unit, A] = st[Unit]()
su(())(0)
```

Теперь можем запустить нашу тривиальную программку ``p`` из ранее — и она отработает как часы:

```scala
scala> val p = [s] => () =>
|
for
|
r1 <- STRef[s, Int](1)
|
r2 <- STRef[s, Int](2)
|
x
<- r1.read
|
y
<- r2.read
|
_
<- r1.write(y + 1)
|
_
<- r2.write(x + 1)
|
a
<- r1.read
|
b
<- r2.read
|
yield (a, b)
val p: [s] => () => ST[s, (Int, Int)] = <function0>
scala> val r = ST.run(p)
val r: (Int, Int) = (3,2)
```

Выражение ``run(p)`` внутри юзает мутабельщину, но снаружи — чист как слеза младенца, никаких побочек. Для любого другого кода это просто пара интов, как из коробки, всегда одна и та же, и нихуя больше не делает. Но это не главное, пацаны. Самое пиздатое — мы не можем запустить программу, которая пытается выкинуть мутабельную ссылку наружу. Невозможно слепить полиморфную функцию, которая возвращает голый ``STRef``, — компилятор её порвёт в клочья, как я рвал багрепорты в 2010-м, когда FP только осваивал.

```scala
scala> val p: [s] => () => ST[s, STRef[Nothing, Int]] =
|
[s] => () => STRef(1)
-- [E007] Type Mismatch Error: ------------------------------------------
2 |
[s] => () => STRef(1)
|
^
|
Found:
[s] => () =>
```
