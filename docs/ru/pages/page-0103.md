## СЦЕНАРИИ ИСПОЛЬЗОВАНИЯ БАЗОВЫХ ФУНКЦИЙ OPTION

Хотя можно явно паттерн-матчить (pattern matching) на `Option`, на деле мы почти всегда берём эти высшие функции (higher-order functions) из арсенала. Тут дам пару ориентиров, когда какую херачить. Свободно жонглировать ими — вопрос практики, а цель — чтобы базово въехали. 

В следующий раз, когда полезешь писать функциональный код с `Option`, попробуй сам угадать паттерны, которые эти функции запаковали, до того как нырнёшь в паттерн-матчинг руками. 

Начнём с `map`. Функция `map` как раз для того, чтоб трансформировать содержимое внутри `Option`, если оно defined (определено). Думай о ней как о продолжении вычислений в режиме «всё ок, ошибки нет» — это ещё и хитрый способ откинуть обработку фейла на будущий код, типа «потом разберёмся»:

```scala
case class Employee(
  name: String,
  department: String,
  manager: Option[Employee]
)
def lookupByName(name: String): Option[Employee] = ...
val joeDepartment: Option[String] =
  lookupByName("Joe").map(_.department)
```

Здесь `lookupByName("Joe")` выдаёт `Option[Employee]`, которую мы через `map` лепим в `Option[String]`, выдирая отдел. Заметь, не надо вручную ковыряться в результате `lookupByName("Joe")` на `isDefined` или херню какую; просто прёшь дальше вычисления, будто в аргументе `map` всё чисто. Если `lookupByName("Joe")` отвалится на `None`, цепочка прервётся нахуй, и `map` вообще не тронет функцию `_.department`. Рисунок 4.2 показывает ещё цепочки в деле. 

`flatMap` похож, только функция-трансформер сама может обосраться.

#### УПРАЖНЕНИЕ 4.2

Забабахай функцию `variance` через `flatMap`. Если среднее последовательности — `m`, то дисперсия (variance) — это среднее от `math.pow(x - m, 2)` для каждого элемента `x` в последовательности. Глянь определение дисперсии (variance) на Wolfram MathWorld (https://mng.bz/7ZgQ):

```scala
def mean(xs: Seq[Double]): Option[Double] =
  if xs.isEmpty then None
  else Some(xs.sum / xs.length)

def variance(xs: Seq[Double]): Option[Double]
```
