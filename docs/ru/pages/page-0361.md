### 12.7.1 От моноидов к аппликативным функторам

Мы только что просекли, что `traverse` — это как танк по сравнению с `map`, мощнее и универсальнее. А теперь держитесь за стул: `traverse` ещё и `foldMap` выражает на раз-два, а заодно и `foldLeft` с `foldRight` в кармане! Гляньте ещё разок на сигнатуру этого зверя:

```scala
extension [A](fa: F[A])
def traverse[G[_]: Applicative, B](f: A => G[B]): G[F[B]]
```

Представьте, что наш `G` — это конструктор типа `ConstInt`, который любой тип в `Int` лепит, `A` нахуй сливает и выдаёт чистый `Int`, как будто и не было его:

```scala
type ConstInt[A] = Int
```

Тогда в сигнатуре `traverse`, если подставить `G = ConstInt`, выходит такая хрень:

```scala
extension [A](fa: F[A])
def traverse(f: A => Int): Int
```

Это ж вылитый `foldMap` из `Foldable`, бля! Точно: если `F` — типа `List`, то чтобы это дело реализовать, нужно склеивать `Int`-ы, которые `f` выдаёт для каждого элемента, плюс стартовая хуйня для пустого списка. Короче, нужен только `Monoid[Int]`, а он всегда под боком, как кофе в офисе. Кстати, с таким константным функтором, как у нас, любой `Monoid` можно в `Applicative` перековать — магия чистой воды.

Листинг 12.8. `Monoid` в `Applicative` — алхимия на марше

```scala
type Const[A, B] = A
```

> Это `ConstInt`, обобщённый на любой `A`, не только `Int`. Сигнатура выглядит не как `apply` из `Applicative`, но из-за `Const[M, _]` нормальная `def apply[A, B](fab: F[A => B])(fa: F[A]): F[B]` упрощается до этой хуйни.

![Страница 0361, изображение 2](./assets/page-0361/image-02.png)

```scala
given monoidApplicative[M](
using m: Monoid[M]
): Applicative[Const[M, _]] with
def unit[A](a: => A): M = m.empty
override def apply[A, B](m1: M)(m2: M): M =
m.combine(m1, m2)
```

Короче, `Traverse` теперь может `Foldable` расширять, и дефолтный `foldMap` лепим через `traverse` — чистый кайф, без лишней жопы:

> Трейт может тащить несколько супер-трейтов, запятыми разделяя.

```scala
trait Traverse[F[_]] extends Functor[F], Foldable[F]:
...
extension [A](fa: F[A])
override def foldMap[B: Monoid](f: A => B): B =
```

> Scala тут не догоняет частично применённый `Const`-псевдоним, так что аннотацию впариваем вручную, классика жанра.

```scala
fa.traverse[Const[B, _], Nothing](f)
```

Теперь `Traverse` тянет и `Foldable`, и `Functor` — полный фарш! Но вот засада: сам `Foldable` `Functor` расширять не может. Хотя для большинства фолдабельных структур, типа `List`, `map` через fold слепить можно, но в общем случае — хуй там, не выйдет, как ни ебись.
