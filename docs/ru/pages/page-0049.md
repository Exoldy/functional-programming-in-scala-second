> Часть 1: Введение в функциональное программирование / Глава 2: Первые шаги с FP в Scala / 2.2 Объекты и пространства имён

Это охуенно удобно, когда юзаешь Scala для скриптинга — типа, как нож в руках у хирурга. Интерпретатор шарпит по файлу MyProgram.scala, выискивает любой объект с методом `@main` нужной сигнатуры и бац — вызывает его. Альтернатива — интерактивный режим интерпретатора, REPL (Read-Eval-Print Loop), запускается просто `scala` без аргументов. Держи REPL-окно всегда наготове, пацаны, — будешь тыкать эксперименты на лету, пока код в Scala ковыряешь. Мы загрузим свой исходник в REPL и потестируем (твой консольный вывод может чуть отличаться, не парься):

![Страница 0049, изображение 1](./assets/page-0049/image-01.png)

> `:load` — команда REPL, чтоб интерпретировать Scala-исходник. (К сожалению, не прокатит для файлов с объявлениями package (package declarations), Scala такая злюка иногда.)

```scala
> scala
Welcome to Scala.
Type in expressions for evaluation. Or try :help.
scala> :load MyProgram.scala
// defined object MyProgram
```

> Вбиваем Scala-выражения прямо на промпте — REPL ждёт, не зевай.

```scala
scala> MyProgram.abs(-42)
val res0: Int = 42
```

> REPL вычисляет твоё выражение, сплёвывает ответ, даёт ему имя `res0` (чтоб потом ссылаться, как на старого кореша), и тип показывает — тут `Int`, классика.

Можно ещё копипастить строчки кода в REPL — фишка огонь. Разберись с REPL по полной, его фишками, потому что как Scala-программист будешь жить в нём, как в матрице. Выход: `:quit` или Ctrl-D/Ctrl-C, чтоб не зависнуть вечно.

### 2.2 Объекты и пространства имён

Тут разберём дополнительные приколы синтаксиса Scala по объектам и пространствам имён (namespaces) — я сам через это говно прошёл, знаю, где подвох поджидает. В той REPL-сессии выше, чтоб достать метод `abs`, пришлось писать `MyProgram.abs` — потому что `abs` живёт внутри объекта `MyProgram`. `MyProgram` — это его *пространство имён (namespace)*, типа квартиры в большом доме, чтоб не путаться. Помимо мелких технических хуёвин, каждая ценность в Scala — это *объект*,^4^ и у объекта может быть ноль или куча *членов (members)*. Член (member) — метод с ключевым словом `def` или вложенный объект с `val` или `object`. Объекты тащат ещё другие виды членов (members), но пока забьём, не грузим мозг. Доступ к членам — классическая ООП-дот-нотация: пространство_имён (имя объекта).имя_члена (member), как `MyProgram.abs(-42)`. Чтоб юзать член `toString` на объекте `42`, пишем `42.toString`. Внутри объекта члены ссылаются друг на друга без префикса (типа, домашний уют), но если припрут — достают enclosing объект по специмени: `this`.^5^

^4^ В отличие от Java, примитивы вроде `Int` тоже объекты для нашей беседы — Scala не Java, бля, не ной.  
^5^ В книге *функция* — это общее понятие для standalone-функций вроде `sqrt` или `abs`, или членов (members) класса, включая методы. Контекст подскажет, а *метод* и *функция* юзаем interchangeably — важен не синтаксис вызова (`obj.method(12)` vs `method(obj, 12)`), а то, что это параметризованный блок кода.
