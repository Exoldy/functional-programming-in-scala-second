- Крутишь ручку на запертом автомате или суешь монету в разблокированный — полный пиздец, нихуя не происходит.
- Автомат, у которого конфеты кончились, на все твои телодвижения забивает хуем.

Метод `simulateMachine` должен прогнать автомат по списку твоих действий и выдать в конце, сколько монет и конфет осталось в нём. Например, если стартовый `Machine` с 10 монетами и пятью конфетами, а ты успешно отжал четыре конфетки, то результат должен быть `(14, 1)`:

```scala
def simulateMachine(inputs: List[Input]): State[Machine, (Int, Int)]
```

### 6.7 Заключение

В этой главе мы разобрались, как городить чисто функциональные проги с состоянием, без этой imperative хуйни, которая везде насрёт. Рандом-генерацию взяли за повод, но паттерн этот — как универсальный гаечный ключ, в любой домене вылазит. Идея проще простого: чистая функция жрёт текущее состояние на вход и возвращает новое состояние плюс свой результат. В следующий раз, когда императивный API с сайд-эффектами тебе мозги выебет, замутите чистую версию и наши функции отсюда — и работать станет как по маслу, без единого мутабельного подвоха.

### Итог

- Тип `scala.util.Random` генерит псевдорандомные числа, но делает это через сайд-эффекты (side-effects), как классический Java-императивный трэш.
- Псевдорандом можно смоделировать чистой функцией: сидище (seed) на входе → новое сидище + сгенерированное значение на выходе.
- Делать stateful API чистыми, заставляя их вычислять следующее состояние из текущего вместо реального мута, — это общий приём, не только для рандома, а для всего, где состояние висит как гиря.
- Когда функциональный подход кажется костылём или муторным дрочём, ищи общие паттерны и фактори их нахуй.
- Тип `Rand[A]` — это алиас для функции `Rng => (A, Rng)`. Есть куча функций для создания и трансформации `Rand`-значений: `unit`, `map`, `map2`, `flatMap` и `sequence`.
- Opaque-типы (opaque types) в своём скоупе ведут себя как алиасы, а снаружи — как совершенно чужие типы. Они инкапсулируют связь с репрезентацией, позволяя лепить новые типы без runtime-оверхеда.
- Extension-методы (extension methods) можно юзать, чтоб навесить методы на opaque-типы.
- Case-классы вместо opaque — ок, но с runtime-ценой за обёртку подложенного значения.
- Тип `State[S, A]` — opaque-алиас для функции `S => (A, S)`.
