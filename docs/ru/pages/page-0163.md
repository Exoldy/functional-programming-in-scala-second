Сначала юзаем `rng.nextInt`, чтоб насрать случайный инт — как кот в коробке, хуй поймёшь, что вылезет.  
Разбираем тюпл на этот сгенерированный инт и следующий `RNG` (r1).  
Потом генерим рандомный дабл (double) методом `double` из упражнения 6.2 — опять же, разбираем тюпл на дабл и свежий `RNG`.  
В финале лепим пару из инта и дабла как результат, а эту пару — с `RNG`, который прилетел от `double`.  
`doubleInt` можно было бы слепить так же на коленке, но нахуй, давайте переиспользуем `intDouble` — классика ленивого FP, чтоб не ебаться заново:

```scala
def doubleInt(rng: RNG): ((Double, Int), RNG) =
val ((i, d), r) = intDouble(rng)
((d, i), r)
```

Тут мы сразу два уровня тюплов разбираем, как матрёшку:  
внешний — это наша сгенерированная `(Int, Double)` плюс следующий `RNG`,  
а внутренний — `(Int, Double)`.  
Дальше строим вложенный тюпл, меняя инту с даблом местами — типа, чтоб симметрия не страдала, пост-ирония в чистом виде.  
Наконец, `double3` лепим через три вызова `double` подряд:

```scala
def double3(rng: RNG): ((Double, Double, Double), RNG) =
val (d1, r1) = double(rng)
val (d2, r2) = double(r1)
val (d3, r3) = double(r2)
((d1, d2, d3), r3)
```

Вызываем `double`, пихая оригинальный `rng` в первый раз, а потом `RNG` от предыдущего — цепная реакция, как домино в продакшене.  
Если б каждый раз тот же `rng` совать, словили бы тройку одинаковых даблов — армия клонов, мем из Star Wars для FP-шников.

#### УПРАЖНЕНИЕ 6.4

Лепим это через хвостово-рекурсивную вспомогательку — чтоб стек не рвало в проде:

```scala
def ints(count: Int)(rng: RNG): (List[Int], RNG)
def go(count: Int, r: RNG, xs: List[Int]): (List[Int], RNG) =
if count <= 0 then
(xs, r)
else
val (x, r2) = r.nextInt
go(count - 1, r2, x :: xs)
go(count, rng, Nil)
```

Вспомогателька `go` тащит на себе остаток счёта, следующий `RNG` для дела и аккумулятор с листом (List) интов.  
Когда зовём — если остаток <= нуля, кидаем накопленный лист интов и последний `RNG`.
