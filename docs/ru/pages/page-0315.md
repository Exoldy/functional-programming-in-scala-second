> Часть 3: Общие структуры в функциональном дизайне / Глава 11: Монды / 11.1 Функторы: Обобщение функции map / 11.1.1 Аксиомы функтора

Для `Functor` мы пропишем знакомую аксиому — ту самую, что замутили в главе 7 для нашего типа `Par`.[^5]

```scala
x.map(a => a) == x
```

Короче, маппинг по структуре `x` через identity-функцию должен сам быть identity — как сказать "не еби мозги, верни как есть". 

Эта хрень вполне естественная, и в части 2 мы видели, как она работает для `map` кучей других типов помимо `Par`. 

Эта аксиома (плюс её выводы из параметричности) ловит суть: `x.map` сохраняет структуру `x`, как строгий коп на входе в клуб — пропускает только чистых, без подстав. 

Имплементации под неё не могут вытворять дичь вроде бросания эксепшенов, выдирания первого элемента из `List`, превращения `Some` в `None` или другой хуйни. Только элементы внутри структуры трогаются `map`ом; сама форма, как скелет в гробу, остаётся нетронутой. 

Заметьте, эта аксиома держится для `List`, `Option`, `Par`, `Gen` и большинства типов с `map` — классика жанра! 

Чтобы не в теории, а на пальцах: вспомним `distribute` и `codistribute` из прошлого. Их сигнатуры вот:

```scala
extension [A, B](fab: F[(A, B)])
def distribute: (F[A], F[B])
extension [A, B](e: Either[F[A], F[B]])
def codistribute: F[Either[A, B]]
```

Мы нихуя не знаем про `F`, кроме того, что это функтор, — и аксиома гарантирует: возвращаемые значения будут той же формы, что аргументы, как зеркальное отражение в лифте. 

Вход в `distribute` — список пар? Выходная пара списков будет той же длины, элементы в том же порядке, без перестановок. 

Такой алгебраический роутинг — как cheat code в дебаге: экономит тонну тестов, не надо каждый раз проверять свойства вручную. 

Ещё одну аксиому для `Functor` замутим — чтоб композиция функций сохранялась, без подвохов:

```scala
x.map(f).map(g) == x.map(f andThen g)
```

Суть: сначала `f` по `x`, потом `g` по результату — это то же самое, что одна составная `f andThen g`. 

Как identity, держится для `List`, `Option`, `Par`, `Gen` и всей этой тусовки с `map`.[^6]

[^5]: Эта аксиома ещё из математического определения функтора растёт, как корни у старого дуба.

[^6]: Если впустить impure-функции, то Scala-шный immutable `Set` — яркий пример `map`, который эту аксиому нахуй посылает. Детали тут: https://mng.bz/yaQe.
