## 121 6.2 Чисто функциональная генерация случайных чисел

![Страница 0150, изображение 1](./assets/page-0150/image-01.png)

```scala
val nextRNG = SimpleRNG(newSeed)
val n = (newSeed >>> 16).toInt
(n, nextRNG)
```

Следующее состояние — это экземпляр RNG (Random Number Generator, генератор случайных чисел), созданный из нового сида.

>>> это правый бинарный сдвиг с заполнением нулями. Значение `n` — новое псевдослучайное целое. Возвращаемое значение — кортеж и с псевдослучайным интом, и с следующим состоянием RNG.

Вот пример, как юзать этот API (application programming interface, программный интерфейс) прямо в интерпретаторе, пацаны:

```scala
scala> val rng = SimpleRNG(42)
val rng: SimpleRNG = SimpleRNG(42)
```

Выберем произвольный сид: 42. Классика, как в "Автостопом по галактике", предсказуемо до тошноты.

```scala
scala> val (n1, rng2) = rng.nextInt
val n1: Int = 16159453
val rng2: RNG = SimpleRNG(1059025964525)
```

Синтаксис для объявки двух значений через деконструкцию пары из `rng.nextInt` — паттерн-матчинг в чистом виде, без говна.

```scala
scala> val (n2, rng3) = rng2.nextInt
val n2: Int = -1281479697
val rng3: RNG = SimpleRNG(197491923327988)
```

Можем гонять эту последовательность хоть тыщу раз — и всегда одни и те же значения, как по нотам. Когда зовём `rng.nextInt`, она всегда отдаёт `16159453` и свежий `RNG`, чей `nextInt` вечно вернёт `-1281479697`. Короче, этот `API` чистый, как слеза девственницы в FP-мире. Рисунок 6.1 всё это суммирует в картинке.
