> Часть 4: Эффекты и I/O / Глава 15: Обработка потоков и инкрементальный I/O / 15.3 Расширяемые пулы и стримы

Простое добавление конструктора ``Eval`` перевернуло всю ``step`` с ног на голову, блядь! Мы подцепили ограничение ``Monad`` для типа эффекта, и вместо обычной рекурсии теперь пилим монадическую — чтоб вычисления в цепочку выстроить, как нормальный пайплайн на проде. В итоге ``step`` стек-сейф только если у эффекта стек-сейфный ``flatMap`` (из-за вызова ``other.step.flatMap(…)``, который жрёт стек как пылесос). Сигура ``step`` раздулась, как типы в Java после добавления дженериков — три новых параметра с баундами, чтоб они были супертипами ``F``, ``O`` и ``R``. Без них Скала орёт: "эй, сука, ковариантные параметры в инвариантных позициях — нехуй тут выёбываться!" Как в прошлых главах делали, можно механически фиксить — вводим тип-параметры с супертип-баундами, но это быстро скатывается в типовой ад, где даже IntelliJ начинает молить о пощаде. Альтернатива — пихаем ``step`` как extension method, и новые параметры не нужны, чистый кайф:

```scala
def step(using F: Monad[F]): F[Either[R, (O, Pull[F, O, R])]] =
self match
case Result(r) => F.unit(Left(r))
case Output(o) => F.unit(Right(o, Pull.done))
case Eval(action) => action.map(Left(_))
case FlatMap(source, f) =>
source match
case FlatMap(s2, g) =>
s2.flatMap(x => g(x).flatMap(y => f(y))).step
case other => other.step.flatMap:
case Left(r) => f(r).step
case Right((hd, tl)) => F.unit(Right((hd, tl.flatMap(f))))
```

В этой главе юзаем методы, но если типовые баунды заебут как legacy на 10 лет, смело на extension methods — проще в разы, хоть тип-инференс и хромает чуток, как после спринта. Методы ``fold`` и ``toList`` фиксим аналогично, потому что ``fold`` рекурсит ``step``, а ``toList`` — ``fold``:

```scala
def fold[F2[x] >: F[x], R2 >: R, A](init: A)(f: (A, O) => A)(
using F: Monad[F2]
): F2[(R2, A)] =
step.flatMap:
case Left(r) => F.unit((r, init))
case Right((hd, tl)) => tl.fold(f(init, hd))(f)
def toList[F2[x] >: F[x]: Monad, O2 >: O]: F2[List[O2]] =
fold(List.newBuilder[O])((bldr, o) => bldr += o).map(_(1).result)
```

``fold`` больше не лепим аннотацию ``tailrec`` — теперь монадическая рекурсия, а не та простая, что стек жрёт как шаурма. Снова надеемся на стек-сейфность ``flatMap`` монады эффекта, чтоб не словить StackOverflowError на реальном трафике.
