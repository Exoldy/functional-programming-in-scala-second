> Глава 7: Чисто функциональный параллелизм /  
> 7.2 Выбор представления /  
> 7.2.1 Уточнение API

Представь, у нас есть `Par[List[Int]]` — это такая параллельная хрень, которая в итоге выплёвывает `List[Int]`, и нам охота превратить её в `Par[List[Int]]`, где результат уже отсортирован по красоте:

```scala
def sortPar(parList: Par[List[Int]]): Par[List[Int]]
```

Конечно, можно было бы просто `run`нуть этот `Par`, отсортировать список, который вылез, и запихнуть его обратно в `Par` через `unit` — классика, как в imperative-мире, где всё через жопу. Но нет, мы не хотим трогать `run`, это ж не по-функциональному, как будто `ExecutorService` (сервис исполнителей из Java) — твой пьяный дядька, которого лучше не будить раньше времени. Единственный комбинатор, который позволяет ковыряться в содержимом `Par`, — это `map2`. Так что кидаем `parList` на одну сторону `map2`, выуживаем список, сортируем его нахрен, а на вторую сторону пихаем фигню какую-нибудь — типа `unit(())`, чисто для вида, чтоб типы сошлись:

```scala
def sortPar(parList: Par[List[Int]]): Par[List[Int]] =
parList.map2(unit(()))((a, _) => a.sorted)
```

Теперь мы можем сказать нашему `Par[List[Int]]`: "Эй, братан, отсортируйся сам!" Но давай обобщим эту хуйню по полной. Мы можем поднять любую функцию `A => B` на уровень `Par`, чтоб она жрала `Par[A]` и пердела `Par[B]`, и `map`ать её поверх `Par` как хотим:

```scala
extension [A](pa: Par[A]) def map[B](f: A => B): Par[B] =
pa.map2(unit(()))((a, _) => f(a))
```

Например, `sortPar` теперь — просто конфетка:

```scala
def sortPar(parList: Par[List[Int]]) =
parList.map(_.sorted)
```

Коротко, ясно, как удар в челюсть. Мы просто подогнали операции под типы, чтоб всё лилось как по маслу. А если глянешь на имплементацию `map2` и `unit`, поймёшь, что этот `map` — не читерство, а чистая логика. Читингом было бы пихать `unit(())` в `map2` и игнорить его? Да хуй там! Это ж показывает, что `map2` — как швейцарский нож в FP, мощнее `map` в разы, и `map` из него вырастает естественно. Такое сплошь и рядом в дизайне либ: то, что кажется примитивом, вдруг оказывается композабельным из чего-то покруче.

Что ещё можно наколдовать на нашем API? А `map` по списку параллельно? `map2` склеивает две параллельки, а `parMap` (назовём его так) должен слить *N* штук в одну кучу. Кажется, это должно быть выразимо, как мем с "it just works":

```scala
def parMap[A, B](ps: List[A])(f: A => B): Par[List[B]]
```
