> Часть 3: Общие структуры в функциональном дизайне / Глава 10: Монойды / 10.7 Композиция монойдов / 10.7.1 Сборка более сложных монойдов

Листинг 10.1. Слияние пар ключ-значение `Map`

```scala
given mapMergeMonoid[K, V](using mv: Monoid[V]): Monoid[Map[K, V]] with
  def combine(a: Map[K, V], b: Map[K, V]) =
    (a.keySet ++ b.keySet).foldLeft(empty): (acc, k) =>
      acc.updated(
        k,
        mv.combine(
          a.getOrElse(k, mv.empty),
          b.getOrElse(k, mv.empty)
        )
      )
  val empty = Map()
```

С этим простым комбинатором мы можем слепить более хуёвые монойды на раз-два — просто ткнём компилятору в тип монойда, который нам подавай, и пусть он сам накопает подходящий инстанс из тех `given`-ов (given-инстансов), что болтаются в скоупе:

```scala
scala> val m = summon[Monoid[Map[String, Map[String, Int]]]]
m: Monoid[Map[String, Map[String, Int]]] = Monoid$mapMergeMonoid@1d1484fd
```

Метод `summon` роет всю толпу `given`-инстансов в скоупе в поисках значения нужного типа — не найдёт, компиляция наебнётся к херам. Поиск рекурсирует, если наткнётся на инстанс со своими контекстными параметрами. 

В нашем случае `summon` копает `Monoid[Map[String, Map[String, Int]]]` и нарывает наш `mapMergeMonoid` с тип-параметрами `K = String` и `V = Map[String, Int]`. Но `mapMergeMonoid` требует `Monoid[V]`, так что Scala снова лезет за `Monoid[Map[String, Int]]` и находит `mapMergeMonoid` — теперь уже с `K = String, V = Int`. Опять `mapMergeMonoid` просит `Monoid[V]`, Scala шарит за `Monoid[Int]` и вылавливает `intAddition`. 

В итоге через эту цепочку поисков Scala собрал `mapMergeMonoid(using mapMergeMonoid(using intAddition))`. Этот алгоритм поиска — не эксклюзив `summon`, он вшит в саму логику Scala по выуживанию `given`-ов. Кстати, мы могли бы сами замутить свой `summon` с таким же поведением, просто кинув контекстный параметр:

```scala
def mySummon[A](using a: A): A = a
```

Эта мощь автоматической сборки монойдов позволяет комбайнить вложенные выражения без лишнего кодинга и без явных ссылок на монойды — чистый FP-кек, как будто компилятор сам допилил твою идею:

```scala
scala> val a = Map("o1" -> Map("i1" -> 1, "i2" -> 2))
a: Map[String,Map[String,Int]] = Map(o1 -> Map(i1 -> 1, i2 -> 2))
scala> val b = Map("o1" -> Map("i2" -> 3))
b: Map[String,Map[String,Int]] = Map(o1 -> Map(i2 -> 3))
scala> val c = m.combine(a, b)
c: Map[String,Map[String,Int]] = Map(o1 -> Map(i1 -> 1, i2 -> 5))
```
