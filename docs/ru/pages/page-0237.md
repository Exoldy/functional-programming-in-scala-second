#### ОТВЕТ 8.14

Результат сортировки любого списка должен быть таким, чтоб каждый элемент был не больше следующего — как солдаты в шеренге, без бардака. Оформляем это свойством: зазипуем элементы списка с его хвостом и проверим каждую пару на порядок. Плюс убедимся, что в отсортированном списке все оригинальные элементы на месте, и хуйни лишней нет:

```scala
val sortedProp = Prop.forAll(smallInt.list): l =>
val ls = l.sorted
val ordered = l.isEmpty || ls.zip(ls.tail).forall((a, b) => a <= b)
ordered && l.forall(ls.contains) && ls.forall(l.contains)
```

#### ОТВЕТ 8.15

Расширим `Gen[A]`, чтоб он мог выдать полный исчерпывающий список домена, не забывая про рандом-сэмплы и бесконечные домены — как турбонаддув на старый движок. Один вариант: спариваем рандом-генератор с exhaustive-списком элементов:

```scala
case class Gen[+A](sample: State[RNG, A], exhaustive: LazyList[Option[A]])
```

Тут конечный exhaustive список домена моделируем как lazy list of Option: каждый value в `Some`, перечислили все — и empty list, типа "всё, братан". Для бесконечных доменов — список с одним `None`, чтоб сразу ясно: не весь домен охвачен. А `map2` для этой новой `Gen` запилим? Легко, как два пальца! Глядите:

```scala
case class Gen[+A](sample: State[RNG, A], exhaustive: LazyList[Option[A]]):
def map2[B, C](g: Gen[B])(f: (A, B) => C): Gen[C] =
Gen(sample.map2(g.sample)(f),
map2LazyList(exhaustive, g.exhaustive)(map2Option(_,_)(f)))
```

Тут `map2` на `LazyList` и `Option` склеивает exhaustive-значения.<sup>14</sup> Это ж пиздец какая мощь! Генерим декартово произведение двух-трех exhaustive-генераторов, как конвейер на стероидах. Далее фиксим `forAll` под новую `Gen`. Вспомните: `Prop` — это функция от `(MaxSize, TestCases, RNG)` к `Result`. Особо интересен `TestCases` — он может быть меньше размера домена, так что уважаем заданное число тестов, не выпендриваемся.

<sup>14</sup> Определения `map2LazyList` и `map2Option` опущены (смотрите `Exhaustive.scala` в исходниках ответов к главе).
