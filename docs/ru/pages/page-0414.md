## 385 13.7 Универсальный тип для I/O

```scala
trait Files[F[_]]:
  def readLines(file: String): F[List[String]]
  def writeLines(file: String, lines: List[String]): F[Unit]
  def cat[F[_]](file: String)(
    using c: Console[F], f: Files[F], m: Monad[F]
  ): F[Unit] =
    f.readLines(file).flatMap: lines =>
      c.printLn(lines.mkString("\n"))
```

Операция `cat` напрямую склеивает результаты кэпабилити (capabilities) `Console` и `Files` — без лишней хуйни и церемоний, потому что обе дают конструкторы в одной и той же таргет-монаде. Никаких юнионов или ручной композухи не требуется: трейты кэпабилити возвращают проги прямиком в этой монаде. Композировать такие проги — как два пальца: в разы проще, чем ебаться с фри-алгебрами, где всё как в конструкторе Lego из ада. А стек-сейфти (stack-safety)? Раньше мы впихивали `Free`, чтоб примитивы секвенсились без стек-оверфлоу (stack overflow). Теперь вся эта магия на таргет-монаде: если она даёт стек-сейф `flatMap`, то и прога будет в шоколаде. Короче, вместо `Function0` как таргета лучше взять `TailRec` или `[x] =>> Free[Par, x]` — это меняет только внешний слой проги, где выбираешь конкретную монаду! Любой сет I/O-операций, который можно запихнуть в фри-алгебру, запросто ложится в кэпабилити-трейт.<sup>20</sup> Кэпабилити проще в работе, как велосипед после тачки с ручным тормозом, но все плюсы фри-энкодинга на месте — я сам через это прошёл, когда стек рвал на проде.

### 13.7 Универсальный тип для I/O

Теперь можно сформулировать общую методологию для I/O-прог: бери любой сет опов, которые хочешь поддержать, — пиши кэпабилити-трейт, где каждый метод это отдельная операция. `Files` для файлового I/O, `Database` для базы данных, а для stdin/stdout что-то вроде `Console`. Для любого такого трейта лепишь инстансы под разные монады. Особо круто — инстанс для низкоуровневого I/O-типа без тормозов, того самого `Async`, о котором раньше болтали:

```scala
opaque type IO[A] = Free[Par, A]
```

Этот `IO` тянет и трамполин-секвенсинг (спасибо `Free`), и асинхронку (благодаря `Par`) — универсальный солдат, короче. В мэйн-проге инстанцируем все кэпабилити под этот безбашенный `IO`. Нужно только инстанс `IO` для каждой.

<sup>20</sup> Кэпабилити-трейты ещё зовут *tagless final* интерпретаторами, а фри-энкодинги — *initial encodings*. Полные детали про эту заёбную переписку — в *Typed Tagless Final Interpreters* Олега Киселёва, ждёт тебя по https://mng.bz/g4KG..
