## 139 6.8 Ответы на упражнения

С этой дефой `update`, `simulateMachine` лепится на раз: сначала каждый инпут превращаем в state-акшн (State-action), который ковыряет состояние машинки, как экскаватор в куче песка, а потом все эти акшны в цепочку секвенсим — и вуаля, симуляция готова:

```scala
State.sequence(
  inputs.map(i => State.modify[Machine](update(i, _)))
)
```

Эта хрень имеет тип `State[Machine, List[Unit]]`; `List[Unit]` там по одному элементу на каждый проглоченный инпут, так что этот результат можно смело в жопу засунуть, выдернуть текущее состояние `Machine` через `State.get` и выдать монеты с конфетами, поковырявшись в аккумуляторах `Machine`:

```scala
def simulateMachine(inputs: List[Input]): State[Machine, (Int, Int)] =
  for
    _ <- State.sequence(inputs.map(i => State.modify(update(i, _))))
    s <- State.get
  yield (s.coins, s.candies)
```

Стопэ, братаны! Мы тут мапим по листу, а потом секвенсим результат — чистый `traverse`, как в мeme про "mapM в Haskell, но без побочек". Вспомните из ранних глав: map за секвенсом — это `traverse` в чистом виде. Давайте вынесем `traverse` для `State`, чтоб не ебаться каждый раз:

```scala
def traverse[S, A, B](as: List[A])(f: A => State[S, B]): State[S, List[B]] =
  as.foldRight(unit[S, List[B]](Nil))((a, acc) => f(a).map2(acc)(_ :: _))
```

А теперь рефакторим `simulateMachine` под `traverse` — короче, чище, как после code review от сеньора:

```scala
def simulateMachine(inputs: List[Input]): State[Machine, (Int, Int)] =
  for
    _ <- State.traverse(inputs)(i =>
      State.modify(update(i, _))))
    s <- State.get
  yield (s.coins, s.candies)
```

Наконец, заметьте: каждый раз, когда звали `update`, курили его плейсхолдером для второго параметра; `update(i, _)` возвращала `Machine => Machine`, идеальную для `modify`. Давайте это явным сделаем — куринг в дефу `update` перетащим, а не на каждом колл-сайте дрочить:

```scala
def simulateMachine(inputs: List[Input]): State[Machine, (Int, Int)] =
  for
    _ <- State.traverse(inputs)(i => State.modify(update(i)))
    s <- State.get
  yield (s.coins, s.candies)
val update = (i: Input) => (s: Machine) =>
  (i, s) match
    case (_, Machine(_, 0, _)) => s
    case (Input.Coin, Machine(false, _, _)) => s
    case (Input.Turn, Machine(true, _, _)) => s
    case (Input.Coin, Machine(true, candy, coin)) =>
      Machine(false, candy, coin + 1)
    case (Input.Turn, Machine(false, candy, coin)) =>
      Machine(true, candy - 1, coin)
```
