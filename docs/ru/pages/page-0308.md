Мы тут слепили функцию `unstub`, которая возвращает `0`, если входная строка пустая как жопа после диареи, а иначе — `1`.

#### ОТВЕТ 10.12

Сначала добавим конкретику в трейт `Foldable` для всех абстрактных методов:

```scala
trait Foldable[F[_]]:
  import Monoid.{endoMonoid, dual}
  extension [A](as: F[A])
    def foldRight[B](acc: B)(f: (A, B) => B): B =
      as.foldMap(f.curried)(using dual(endoMonoid[B]))(acc)
    def foldLeft[B](acc: B)(f: (B, A) => B): B =
      as.foldMap(a => b => f(b, a))(using endoMonoid[B])(acc)
    def foldMap[B](f: A => B)(using mb: Monoid[B]): B =
      as.foldRight(mb.empty)((a, b) => mb.combine(f(a), b))
    def combineAll(using ma: Monoid[A]): A =
      as.foldLeft(ma.empty)(ma.combine)
```

Тут мы реализовали `foldLeft` и `foldRight` через `foldMap`, а `foldMap` — через `foldRight`, используя импы из упр. 10.6. 
И вуаля — абстрактных операций ноль, чистый лист! 

Но инстанс обязан переопределить либо `foldRight`, либо `foldMap`, иначе рекурсия закрутит в бесконечный цикл, как собака за хвост — привет, `OutOfMemoryError`. 

Эта техника — чистый кайф: выбираешь, какие операции базовые (фундаментальные, как скала), а какие производные от них, под конкретный тип. 

Альтернатива — оставить `foldLeft` и `foldRight` абстрактными, потому что импы через `foldMap` — не фонтан производительности, скорее лужа после дождя.
