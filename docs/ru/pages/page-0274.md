## 245 9.8 Ответы на упражнения

Мы юзали `map2` и выкинули эти распарсенные пробелы в /dev/null,  
как ненужный мусор после код-ревью. 

Можем вынести универсальный комбинатор, который склеит два парсера  
и сольёт результат левого в унитаз. То же хуйня и для правого —  
отбрасываем его, оставляя только то, что справа:

```scala
extension [A](p: Parser[A])
def *>[B](p2: => Parser[B]): Parser[B] = p.map2(p2)((_, b) => b)
def <*[B](p2: => Parser[B]): Parser[A] = p.map2(p2)((a, _) => a)
def document: Parser[JSON] = whitespace *> (array | obj)
```

Мы их окрестили `<*` и `*>` — стрелочка, как в меме с "this is fine",  
показывает, куда попрёт ценный результат, а остальное летит нахуй. 

Наконец, убедимся, что после парсинга массива или объекта  
не болтается хвост инпута, как недоделанный фич-реквест:

```scala
def eof: Parser[String] =
regex("\\z".r).label("unexpected trailing characters")
def document: Parser[JSON] = whitespace *> (array | obj) <* eof
```

Тут мы ввели парсер `eof`, который сработает только на пустой строке —  
чистый `endOfInput` (end of input, конец ввода), как после деплоя в прод без багов. 

Юзали regex-парсер (regex-parser) с паттерном `\z`, хотя могли и руками накатать,  
без этой ленивой хуйни. 

Теперь давай реализуем те парсеры, что оставили висеть на TODO. 
JSON-массив — это список значений через запятую (CSV, comma-separated values,  
блядь, как в экселе от бухгалтерии) в квадратных скобках. 

Нам нужен парсер для JSON-значения и способ выразить список элементов,  
разделённых другим парсером. Накидаем схемку:

```scala
val value: Parser[JSON] = ???
extension [A](p: Parser[A])
def sep(separator: Parser[Any]): Parser[List[A]] = ???
def array: Parser[JSON] =
string("[") *> value.sep(string(",")).map(vs =>
JArray(vs.toIndexedSeq)) <* string("]")
```

Метод `sep` — это как `many` на стероидах, но перед повтором оригинального парсера  
он жрёт сепаратор и сплёвывает его в помойку (отсюда параметр `sep` имеет тип `Parser[Any]`,  
чисто для отбраковки). 

`sep` реализуем так же, как `many` — с вариантом на один или больше значений,  
чтоб не срать пустыми списками:

```scala
extension [A](p: Parser[A])
def sep(separator: Parser[Any]): Parser[List[A]] =
p.sep1(separator) | succeed(Nil)
def sep1(separator: Parser[Any]): Parser[List[A]] =
p.map2((separator *> p).many)(_ :: _)
```

Операция `sep1` — это вообще огонь, потому что она склеивает `p` с результатом  
кучи итераций `separator` `*>` `p`, как конструктор Lego из FP-кирпичиков.
