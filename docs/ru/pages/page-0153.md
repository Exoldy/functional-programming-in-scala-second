### 6.4 Более удобный API для *state actions* (действий с состоянием)

Оглянемся на наши реализации — сразу видно общий паттерн, как в коде после спринта: 
каждая функция имеет тип вида `RNG => (A, RNG)` для какого-нибудь `A`. 
Такие функции зовутся *state actions* (действия с состоянием) или *state transitions* (переходы состояний) — 
они жрут `RNG`-состояние и выплевывают следующее, как конвейер в фабрике по производству рандома. 
Эти action'ы можно миксовать по-всякому, чтоб слепить новые, и в этой секции мы набросаем кучу таких комбайнов. 
А то вручную таскать стейт от action к action — это как самому кофе варить в офисе, 
когда есть кофемашина: уныло и повторчиво до тошноты. 
Хотим, чтоб функции сами перекидывали стейт, как пас в футболе. 
Чтоб тип action'ов было удобно мусолить на код-ревью и не путаться в башке, 
замутим type alias для `RNG` state action:

```scala
type Rand[+A] = RNG => (A, RNG)
```

Значение типа `Rand[A]` можно вообразить как *рандомно сгенерированный* `A`, 
но это как сказать, что Scala — просто Java с плюшками: не совсем правда. 
На деле это полноценный state action — кусок программы, который зависит от `RNG`, 
жрёт его для генерации `A` и кидает `RNG` в новое состояние, 
чтоб следующий action подхватил, как эстафета в марафоне. 
Теперь методы вроде `RNG.nextInt` можно завернуть в этот тип, и вуаля:

> Можно даже шорткат `_.nextInt` юзать, чтоб не ебаться.

```scala
val int: Rand[Int] = rng => rng.nextInt
```

Хотим функции, которые комбайнят `Rand`-action'ы, не заставляя вручную пинать `RNG`-стейт, 
как мяч в одиночку. Получится DSL (domain-specific language, язык для конкретной области), 
который сам всё перекидает — чистый кайф, как когда Jenkins сам билдит. 
Например, базовый `RNG`-transition — action `unit` (юнит), 
который пропускает `RNG`-стейт насквозь, не трогая его, и всегда возвращает константу, 
а не рандом из жопы:

```scala
def unit[A](a: A): Rand[A] =
  rng => (a, rng)
```

Есть ещё `map` для трансформации аутпута state action, не ковыряя дальше результирующий стейт. 
Помнишь, `Rand[A]` — это просто алиас для функции `RNG => (A, RNG)`, 
так что это классическая композиция функций, только с FP-шиком, 
чтоб не чувствовать себя императивным нообом.
