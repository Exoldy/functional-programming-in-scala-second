## 321 12.4 Преимущества функторов applicative

`map2` комбинирует элементы попарно из каждого входа функцией, которую ему впарили в `map2`. В итоге длина выхода пляшет под дудку самого короткого входа — как в том меме про "самое слабое звено в цепи". Чтобы не просрать эту структуру-сохраняющую магию `map2`, нам подавай реализацию `unit` (единица, Applicative.unit), которая ни хуя не короче любой другой ленивой последовательности (LazyList). Достигаем этого хаком с бесконечным lazy list'ом (ленивым списком), блядь:

```scala
val zipLazyListApplicative: Applicative[LazyList] = new:
def unit[A](a: => A): LazyList[A] =
LazyList.continually(a)
```

![Страница 0350, изображение 1](./assets/page-0350/image-01.png)

> Бесконечный, константный

```scala
extension [A](fa: LazyList[A])
def map2[B, C](fb: LazyList[B])(
f: (A, B) => C): LazyList[C] =
fa.zip(fb).map(f.tupled)
```

> Комбинирует элементы попарно.

Пиздец как невозможно впихнуть `flatMap` или `join` для этого инстанса так, чтоб оно дружило с `map2` — то есть, чтоб `fa.map2(fb)(f)` выдавала тот же результат, что и `fa.flatMap(a => fb.map(b => f(a, b)))`. Мы слепили этот `Applicative[LazyList]` инстанс как обычное значение, а не given (given-инстанс), потому что на тип — только один given инстанс (иначе компилятор завоет ambiguity errors (ошибками неоднозначности) в твою жопу). Работает, но юзать — как геморрой: везде приходится впихивать явно в операции, где applicative на контексте. Вместо этого лепим новый тип и вешаем на него given с zipping applicative (applicative на основе парного соединения, zip) — чисто, как пластырь на рану:

```scala
opaque type ZipList[+A] = LazyList[A]
object ZipList:
def fromLazyList[A](la: LazyList[A]): ZipList[A] = la
extension [A](za: ZipList[A]) def toLazyList: LazyList[A] = za
given zipListApplicative: Applicative[ZipList] with
def unit[A](a: => A): ZipList[A] =
LazyList.continually(a)
extension [A](fa: ZipList[A])
override def map2[B, C](fb: ZipList[B])(f: (A, B) => C) =
fa.zip(fb).map(f.tupled)
```

![Страница 0350, изображение 2](./assets/page-0350/image-02.png)

За счёт лёгкого boilerplate (котельного кода) с конверсиями между `LazyList` и `ZipList` — типа, "ещё один слой абстракции, ещё один map" — мы гарантируем, что всегда юзаем именно zip-based applicative, без подвохов.

#### УПРАЖНЕНИЕ 12.4

