## 2.3 Функции высшего порядка: Передача функций в функции

![Страница 0052, изображение 1](./assets/page-0052/image-01.png)

Ручками расписывать такие трассировки — это часто пиздец как помогает въехать в рекурсивные функции, пацаны. Как будто стек вызовов разложил по полочкам, и видно, где оно вот-вот стековерфлоуанет.

Хвостовые вызовы в Scala. Вызов считается в *хвостовой позиции*, если вызывающий код нихуя больше не делает — просто возвращает значение рекурсивного вызова и на этом всё. Например, тот рекурсивный вызов `go(n - 1, n * acc)`, о котором мы базарили раньше, как раз в хвостовой позиции: метод берёт его результат и кидает наружу напрямую, без дополнительной хуйни. А вот если бы мы написали `1 + go(n - 1, n * acc)`, то `go` уже не в хвосте — методу после возврата от `go` ещё придётся попотеть (а именно, прибавить `1`).

Если все рекурсивные вызовы в функции — строго в хвостовой позиции, Scala сама компилит эту рекурсию в итеративный цикл, который не жрёт стек вызовов на каждой итерации, как старый добрый while в C. По умолчанию компилятор молчит, оптимизировал он хвост или нет, но если ты пишешь рекурсию и рассчитываешь на эту магию, можешь ткнуть компилятору носом с помощью аннотации `@tailrec` (https://mng.bz/499D) — и если он не сможет хвосты убрать, выдаст ошибку компиляции. Синтаксис вот такой:

```scala
def factorial(n: Int): Int =
@annotation.tailrec
def go(n: Int, acc: Int): Int =
if n <= 0 then acc
else go(n - 1, n * acc)
go(n, 1)
```

В книге мы больше не будем много ебаться с аннотациями (подробности на https://mng.bz/nYvV), но `@annotation.tailrec` будем юзать от души, чтоб не словить StackOverflowError в проде.

> **ПРИМЕЧАНИЕ** Информацию об упражнениях ищи в предисловии.

#### УПРАЖНЕНИЕ 2.1

Напиши рекурсивную функцию, чтоб вытащить `n`-е число Фибоначчи (http://mng.bz/C29s). Первые два — `0` и `1`. `n`-е всегда сумма двух предыдущих — последовательность стартует `0, 1, 1, 2, 3, 5`. Твоё определение обязано юзать локальную хвостово-рекурсивную функцию:

```scala
def fib(n: Int): Int
```
