> Часть 3: Общие структуры в функциональном дизайне / Глава 12: Applicative и traversable функторы / 12.7 Применения Traverse / 12.7.2 Traversals со State

Мы стартуем с пустого списка `Nil` как с начальным состоянием — классика, блядь, как в старом добром аккамуляторе. На каждом элементе traversal'а мы пихаем его в начало накопленного списка, и вуаля: список собирается в обратном порядке обхода, как если бы ты стопку блинов перекладывал снизу вверх. В конце просто реверсим этот хлам, который выдаст завершённый state action — и готово, чистый `toList`, без лишней хуйни. Обратите внимание, мы юзаем `yield` `()`, потому что нам похер на возвращаемое значение — только состояние важно, как в том меме про "state is the new black". А код для `toList` и `zipWithIndex`? Почти один в один, как близнецы из матрицы. И большинство traversal'ей с `State` именно так и пляшут: берём текущее состояние, мутим следующее, сетим его и сплёвываем значение. Это же скрининг паттерн, пацаны, надо зафакторить в функцию, чтоб не ебаться каждый раз заново — я через это прошёл, знаю, где собака порылась.

Листинг 12.11. Вытаскиваем нашу `mapAccum` функцию наружу

```scala
extension [A](fa: F[A])
def mapAccum[S, B](s: S)(f: (A, S) => (B, S)): (F[B], S) =
fa.traverse(a =>
for
s1 <- State.get[S]
(b, s2) = f(a, s1)
_
<- State.set(s2)
yield b
).run(s)
override def toList: List[A] =
fa.mapAccum(List[A]())((a, s) => ((), a :: s))(1).reverse
```

#### УПРАЖНЕНИЕ 12.16

Есть забавный сайд-эффект от того, что любой traversable functor можно слепить в reversed list: пишем раз и навсегда функцию реверса для *любого* traversable! Напишите её и покопайтесь в мыслях, что это значит для `List`, `Tree` и других traversable'ей — это как универсальный гаечный ключ в FP-гараже.

```scala
extension [A](fa: F[A])
def reverse: F[A]
```

Она должна подчиняться этому закону для всех `x` и `y` подходящих типов:

![Страница 0363, изображение 2](./assets/page-0363/image-02.png)

```scala
x.reverse.toList ++ y.reverse.toList == (y.toList ++ x.toList).reverse
```
