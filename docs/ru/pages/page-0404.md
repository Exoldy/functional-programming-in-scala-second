## 375 13.4 Более нюансированный тип I/O

![Страница 0404, изображение 1](./assets/page-0404/image-01.png)

```scala
extension [A](fa: Par[A])
def flatMap[B](f: A => Par[B]) =
Par.flatMap(fa)(f)
```

#### УПРАЖНЕНИЕ 13.4

*Сложное*: Выяснилось, что наша удобная хуйня `toThunk` не stack-safe, потому что `flatMap` сама по себе не stack-safe для `Function0` (та же херня, что и у нашего изначального наивного `IO`, где `unsafeRun` вызывала себя рекурсивно в реализации `flatMap`, и стек летит к чертям). Реализуй `translate` через `runFree`, а потом собери `unsafeRunConsole` на `translate` по-настоящему stack-safe, чтоб не срать кирпичами на проде:

```scala
enum Free[F[_], A]:
...
def translate[G[_]](fToG: [x] => F[x] => G[x]): Free[G, A] = ???
extension [A](fa: Free[Console, A])
def unsafeRunConsole: A = ???
```

Значение типа `Free[F,` `A]` — это как программа на ассемблере из инструкций, которые задаёт `F`. В случае с `Console` у нас две базовые команды: `PrintLine` и `ReadLine`. Рекурсивный скелет (`Suspend`) и монодическая подстановка переменных (`FlatMap` с `Return`) на блюдечке подают сам `Free`. Можем слепить другие варианты `F` под разные наборы команд — например, для фс-шных дел, `F` с правами на чтение/запись (или чисто на чтение, чтоб не разъебать). Или сетевой `F`, чтоб коннекты открывать и читать из них, и так далее, короче, freestyle.

### 13.4.3 Чистые интерпретаторы

Заметим, что в типе `Free[Console,` `A]` нихуя не сказано про то, что эффекты реально должны выстрелить! Это на совести интерпретатора, блядь. Можем перевести наши `Console`-действия в чистые значения, которые I/O не трогают вообще! Например, для тестов интерпретатор может игнорить `PrintLine` и всегда кидать константу на `ReadLine`. Делаем это, переводя `Console`-запросы в `String` `=>` `A` — это монода в `A`, как мы ковыряли в упражнении 11.20 (`readerMonad`), помните, ту самую мемную хрень.

Листинг 13.7 Создаём наш класс `ConsoleReader`

![Страница 0404, изображение 2](./assets/page-0404/image-02.png)

> Специализированный reader-монод как case class, а не opaque type — для краткости, чтоб не ебаться с абстракциями

```scala
case class ConsoleReader[A](run: String => A):
def map[B](f: A => B): ConsoleReader[B] =
ConsoleReader(r => f(run(r)))
def flatMap[B](f: A => ConsoleReader[B]): ConsoleReader[B] =
ConsoleReader(r => f(run(r)).run(r))
```
