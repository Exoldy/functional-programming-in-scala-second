## 383 13.6 Способности

```scala
extension [A](fa: Free[[x] =>> Files[x] | Console[x], A])
def toThunk: () => A =
fa.runFree([x] => (fx: Files[x] | Console[x]) => fx match
case c: Console[x] => c.toThunk
case f: Files[x] => f.toThunk
)
```

### 13.6 Способности

Давайте подведём итоги нашему подходу к I/O на данный момент, чтоб не запутаться в этих монадных дебрях, как в первом Cats-ревью. Мы определяем алгебры для операций I/O — чисто перечисляем примитивные действия, как меню в шаурме: только база, без излишеств. Каждая алгебра собирает в кучу связанные действия, те, что обычно летают вместе и рассуждаются как монолит (ну типа `Console`, `Files` или `Database` — знакомо, да?). Сцепляем эти примитивы в полноценные программы через `Free`, постепенно наращивая выразительность, просто беря union разных алгебр эффектов — как лего собираем, brick за brick'ом. В финале интерпретируем эти свободные программы в целевую монаду (типа `Par` или `Thunk`) — по интерпретатору на каждую алгебру, а потом запускаем монаду и едем, выполняем программу. Главный кайф этого подхода — зажимаем effectful-программы в узкий коридор операций, чтоб рассуждать проще и переиспользовать без гемора. Программа типа `Free` `[Console,` `Unit]` может только с консолью болтать — диски не тронет, не переживай. А вот `Free[[x]` `=>>` `Files[x]` `|` `Console[x]]` — это уже и файлы ковыряет, и консоль треплет, но базу и сеть не заденет, чисто по делу. Это классика принципа наименьшей власти<sup>18</sup> — даём программе ровно минимум, чтоб задачу закрыла, и local reasoning на уровне, без глобального пиздеца. Но вот encoding и композиция этих least-powerful программ — свой подвох: boilerplate через suspension-конструкторы на каждую опу, жирные типы (union алгебр, привет), и эта инверсия — сначала free-прогу лепим, потом интерпретируем в target monad. Можно ли лучше, не жертвуя принципом? Короче, вместо алгебры для примитива I/O лепим *capability trait*. Определяем трейт с параметром target monad<sup>19</sup>, а каждую примитивную операцию — как метод, возвращающий значение этой монады:

```scala
trait Console[F[_]]:
def readLn: F[Option[String]]
def printLn(line: String): F[Unit]
```

Операции трейта `Console` — это готовые конструкторы для примитивных консольных дел в target monad. Например, имея `Console[F]`, слепишь `F[Option[String]]`-прогу, которая строку из консоли жрёт — и похер, какая там монада `F` под капотом.

<sup>18</sup>Подробнее о принципе наименьшей власти — тут: https://mng.bz/aDpX.  
<sup>19</sup>Точнее, target type constructor. На практике это всегда монады, чтоб операции секвенились.
