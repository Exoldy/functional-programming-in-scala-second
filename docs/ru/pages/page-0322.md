## 293 11.4 Законы монад

```scala
Gen.nextString.flatMap(name =>
Gen.nextInt.map(price =>
Item(name, price))
).flatMap(item =>
Gen.nextInt.map(quantity =>
Order(item, quantity)))
```

Развернёшь их — и бац, эти две реализации разные как императорские пингвины и сибирские волки.  
А вот for-comprehension (for-выражение) смотрит на тебя невинными глазами и шепчет:  
«Да ладно, они ж одно и то же делают, пацаны».  
И ведь не поспоришь, иначе бы вообще крыша поехала — это ж как в том меме, где кот пытается собрать стул из ИКЕА без инструкции.  
Всё потому, что мы твёрдо стоим на том, что `flatMap` подчиняется ассоциативному закону, который выглядит вот так:

```scala
x.flatMap(f).flatMap(g) == x.flatMap(a => f(a).flatMap(g))
```

И этот закон должен держаться для всех значений `x`, `f` и `g` нужных типов — не только для `Gen`, но и для `Parser`, `Option` и любой другой монады, чтоб не было сюрпризов в проде.

### 11.4.2 Доказательство ассоциативного закона для конкретной монады

Давай на пальцах докажем, что этот закон работает для `Option` — без лишней хуйни, чисто по делу.  
Берём то уравнение выше, суём вместо `x` то `None`, то `Some(v)`, и разворачиваем обе стороны.  
Начинаем с случая, когда `x` — это `None`, и тогда слева и справа от равенства одинаково `None`:

```scala
None.flatMap(f).flatMap(g) == None.flatMap(a => f(a).flatMap(g))
```

Поскольку `None.flatMap(f)` всегда `None` для любого `f`, это упрощается до:

```scala
None == None
```

Короче, закон держится, если `x` — `None`.  
А если `x` — это `Some(v)` для произвольного `v`? Тогда имеем:

```scala
x.flatMap(f).flatMap(g) ==
x.flatMap(a => f(a).flatMap(g))
```

![Страница 0322, изображение 1](./assets/page-0322/image-01.png)

> Исходный закон  
> Подставляем `Some(v)` вместо `x` с обеих сторон.

```scala
Some(v).flatMap(f).flatMap(g) ==
Some(v).flatMap(a => f(a).flatMap(g))
```

> Применяем определение `Some(v).flatMap(…)`.

```scala
f(v).flatMap(g) ==
(a => f(a).flatMap(g))(v)
```

> Упрощаем применение функции `(a => ..)(v)`.

```scala
f(v).flatMap(g) . == f(v).flatMap(g)
```

Значит, закон и здесь сходится, когда `x` — `Some(v)` для любого `v`.  
Всё, допиздели — показали, что держится для `x` как `None`, так и `Some`, а других вариантов для `Option` и нет, чтоб не ебаться с кейсами.
