#### УПРАЖНЕНИЕ 5.15

Затопи `tails` через `unfold`. Для заданного `LazyList` этот `tails` возвращает `LazyList` всех суффиксов входной последовательности, начиная с оригинального `LazyList`. Например, для `LazyList(1, 2, 3)` он выдаст `LazyList(LazyList(1, 2, 3), LazyList(2, 3), LazyList(3), LazyList())`:

```scala
def tails: LazyList[LazyList[A]]
```

Теперь можем слепить `hasSubsequence` из функций, что уже накатали:

```scala
def hasSubsequence[A](l: LazyList[A]): Boolean =
  tails.exists(_.startsWith(l))
```

Эта реализация жрёт ровно столько же шагов, сколько монолит с вложенными лупами и early-брейками из каждого. Ленивость — это как супергерой: компонуем из простых кирпичей, а эффективность спецухи (которая verbose, как моя старая кодовая база до рефакторинга) сохраняется на уровне.

#### УПРАЖНЕНИЕ 5.16

*Сложняк*: Обобщи `tails` до функции `scanRight` — это как `foldRight`, только вместо финального результата возвращает ленивый список промежуточных. Например:

```scala
scala> LazyList(1, 2, 3).scanRight(0)(_ + _).toList
res0: List[Int] = List(6, 5, 3, 0)
```

Этот пример должен быть эквивалентен развёртке `List(1 + 2 + 3 + 0, 2 + 3 + 0, 3 + 0, 0)`. Твоя функция обязана переиспользовать промежуточку, чтоб проезд по `LazyList` с `n` элементами всегда был строго линейным по `n`. Получится ли на `unfold`? Как или хули нет? А на другой нашей функции замутишь?
