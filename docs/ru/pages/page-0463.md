# Страница 0463

> Часть 4: Эффекты и I/O / Глава 15: Обработка стримов и инкрементальный I/O / 15.3 Расширяемые пулы и стримы / 15.3.2 Обработка ошибок

ошибки. Эта хуйня особенно мощно выстреливает, когда стрим, который ты пихаешь в `onComplete`, реально эффекты вычисляет. А `onComplete` дает нам безопасно хватать и отстегивать ресурсы, типа файловых дескрипторов? Давай разберемся по косточкам:

```scala
def acquire(path: String): Task[Source] =
Task(Source.fromFile(path))
def use(source: Source): Stream[Task, Unit] =
Stream.eval(Task(source.getLines))
.flatMap(itr => Stream.fromIterator(itr))
.mapEval(line => Task(println(line)))
def release(source: Source): Task[Unit] =
Task(source.close())
val printLines: Stream[Task, Unit] =
Stream.eval(acquire("path/to/file")).flatMap(resource =>
use(resource).onComplete(Stream.eval(release(resource))))
```

Сначала отрабатываем таск `acquire`, чтоб файл открыть, потом `flatMap` этот результат — лепим стрим, который юзает открытый файл и релизит его через `onComplete`. Дальше каждую строчку в консоль херачим. Запустишь это дело, подкинь логи на захват ресурса и финалку — увидишь, что все пиздато, как по маслу. К сожалению, у этого подхода жирный подвох: он не композабельный, блядь. Представь, рефакторим чуток, выносим печать строк после `onComplete`:

```scala
val printLines: Stream[Task, Unit] =
Stream.eval(acquire("path/to/file"))
.flatMap: resource =>
Stream.eval(source => Task(source.getLines))
.flatMap(itr => Stream.fromIterator(itr))
.onComplete(Stream.eval(release(resource)))
.mapEval(line => Task(println(line)))
```

Эта прога должна вести себя как оригинал, и без ошибок так и есть. Но если финальный `mapEval` наебнется эксепшеном, то хэндлер от `onComplete` не сработает, вся хуйня упадет, и ресурс так и не релизнется — классика жанра, как в том меме про "leak in prod". Еще один пиздец — досрочный терминашн, когда берешь только часть элементов из источника. Допустим, вместо `use` хапаем первые 10 строк. Если `take(10)` внутри `use` — все ок. А если вынесем `take(10)` после `onComplete` — первые 10 вернутся, а ресурс повисит как висельник. В обоих случаях мы ждем, что ресурс релизнется нахер, независимо от прочего дерьма в стриме. Полагаться только на `flatMap` и `onComplete` — это как ездить без ремня, гарантий ноль. Ща вернемся к этой гарантии.
