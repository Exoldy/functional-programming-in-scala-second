## 237 9.6 Внедряем алгебру

Мы закинули хелпер-функцию в `ParseError`, тоже окрестили её `label`. Порешаем по-взрослому: `label` будет урезать стек ошибок, как мясник лишнее с туши, отсекая детальные мессаджи из вложенных скоупов и оставляя только свежий след с самого дна стека — чтоб не тонуть в логах, как в том классическом аду от legacy-монстра:

```scala
case class ParseError(stack: List[(Location, String)] = Nil):
def label(s: String): ParseError =
ParseError(latestLoc.map((_, s)).toList)
```

![Страница 0266, изображение 1](./assets/page-0266/image-01.png)

```scala
def latestLoc: Option[Location] =
latest.map(_(0))
```

> Вытаскивает последний элемент стека или None, если стек пустой, как обещания junior'а по дедлайну

```scala
def latest: Option[(Location, String)] =
stack.lastOption
```

#### УПРАЖНЕНИЕ 9.14

Представь парсер неотрицательного инта, который жрёт цифры, как Pac-Man точки:

```scala
val nonNegativeInt: Parser[Int] =
for
nString <- regex("[0-9]+".r)
n <- nString.toIntOption match
case Some(n) => succeed(n)
case None => fail("expected an integer")
yield n
```

Перелопать эту реализацию, чтоб юзала `scope` или `label` и выдавала осмысленный мессадж, когда парсер наебнётся, — а не эту generic хуйню "expected something".

### 9.6.4 Резервное переключение и бэктрэкинг Теперь разберём or и attempt. Вспомним, что мы наметили для or: запускаем первый парсер, если он обосрался в uncommitted состоянии (типа, ещё не жрал инпут всерьёз), то пускаем второй на тот же самый кусок. Съели хотя бы один символ — и привет, committed, назад не отмотаешь, как коммит в master без revert'а. А p.attempt как раз и превращает такие committed фейлы p в uncommitted, чтоб дать шанс альтернативе. Поддержим эту схему, добавив в Failure-кейс Result булевый флаг — committed ли был фейл, или просто покашлял и отступил:

```scala
case Failure(get: ParseError, isCommitted: Boolean)
```

Реализация `attempt` просто сливает коммит с любых фейлов, что вылезут, — чистый трюк. Юзает хелпер `uncommit`, который дефайним на `Result`, чтоб не плодить entity:

```scala
extension [A](p: Parser[A]) def attempt: Parser[A] =
l => p(l).uncommit
enum Result[+A]:
```
