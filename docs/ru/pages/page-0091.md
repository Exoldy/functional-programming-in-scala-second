> Часть 1: Введение в функциональное программирование / Глава 3: Функциональные структуры данных / 3.6 Ответы на упражнения

Тут мы впрягаем `foldRight`, чтоб собирать резалтинг-лист в правильном порядке, не переворачивая его потом как блин на сковородке, а `Cons` пихаем как нашу конкатенатор-функцию. 
Перед тем как слепить `Cons`-ячейку, подкручиваем интовый счётчик на тик — чтоб не потеряться в этой рекурсивной мясорубке.

![Страница 0091, изображение 1](./assets/page-0091/image-01.png)

#### ОТВЕТ 3.17

```scala
def doubleToString(l: List[Double]): List[String] =
  foldRight(
    l,
    Nil: List[String],
    (d, acc) => Cons(d.toString, acc)
  )
```

Та же стратегия, что и в `incrementEach`: `foldRight` с `Cons` внутри комбайнера. 
Единственная разница — зовём `d.toString` вместо `i + 1`. 
Никаких новых велосипедов, просто подкрутили гайки под новую задачу, чтоб стек не рвал.

![Страница 0091, изображение 2](./assets/page-0091/image-02.png)

#### ОТВЕТ 3.18

```scala
def map[A, B](as: List[A], f: A => B): List[B] =
  foldRight(as, Nil: List[B], (a, acc) => Cons(f(a), acc))
```

Опять та же тактика с `foldRight`, где `Cons` — наш верный комбайнер. 
Просто вынесли общую хуйню из `incrementEach` и `doubleToString` в отдельный метод, чтоб не дублировать код как лохи. 
Но учти, стек-сейфность только если `foldRight` сам по себе не словит переполнение стека (StackOverflow). 
Наш первый `foldRight` был с дыркой, но мы уже показали, как его залатать через `reverse` и `foldLeft` внутри — чистый трюк из продакшена.

![Страница 0091, изображение 3](./assets/page-0091/image-03.png)

#### ОТВЕТ 3.19

```scala
def filter[A](as: List[A], f: A => Boolean): List[A] =
  foldRight(as, Nil: List[A], (a, acc) =>
    if f(a) then Cons(a, acc)
    else acc
  )
```

Похоже на `map`, только лепим новую `Cons`-ячейку, если предикат пропустил — иначе просто пропускаем, как фильтр в пивной понке. 
Эффективно и без лишнего мусора в памяти.

![Страница 0091, изображение 4](./assets/page-0091/image-04.png)

#### ОТВЕТ 3.20

```scala
def flatMap[A, B](as: List[A], f: A => List[B]): List[B] =
  foldRight(as, Nil: List[B], (a, acc) => append(f(a), acc))
```

Тут опять `foldRight` в деле: сначала `A` в `List[B]` перегоняем, а потом с аккумулированным `List[B]` склеиваем.
