## 85 4.4 Тип данных Either

```scala
scala> val pair = map2All(p1, p2, (_, _))
val pair: Either[List[String], (Person, Person)] =
Right((Person(Name(Curry),Age(81)),Person(Name(Howard),Age(96))))
```

Ща замутим вариацию `traverse`, которая все ошибки в кучу сгребёт, как пылесос в хостел-комнате после тусы. Меняем только тип возврата — и сигнатура такая выкатывается, простая как правда:

```scala
def traverseAll[E, A, B](
as: List[A],
f: A => Either[E, B]
): Either[List[E], List[B]]
```

Лепим это по той же схеме, что и `traverse` раньше — берём `foldRight`, которая по каждому элементу `map2` гоняет, финальный список клепает, но вместо `map2` пихаем `map2All`, и готово:

```scala
def traverseAll[E, A, B](
as: List[A],
f: A => Either[List[E], B]
): Either[List[E], List[B]] =
as.foldRight(Right(Nil): Either[List[E], List[B]])((a, acc) =>
map2All(f(a), acc, _ :: _)
)
```

Заметим заодно, что тип `f` перелопатили с `A => Either[E, B]` на `A => Either[List[E], B]`. Могли б и оригинал оставить вместо `A => Either[E, B]`, а результат `f` перед `map2All` подкрутить, но это как ноги связать — компоновать новые ф-и из этой хуйни потом будешь с матом. С новой подписью, к примеру, `sequenceAll` через `traverseAll` определить — раз плюнуть, айдентичку для `f` суй и не парься:

```scala
def sequenceAll[E, A](
as: List[Either[List[E], A]]
): Either[List[E], List[A]] =
traverseAll(as, identity)
```
