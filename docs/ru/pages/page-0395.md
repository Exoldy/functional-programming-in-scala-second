```scala
FlatMap(a1, a1 =>
FlatMap(a2, a2 =>
FlatMap(a3, a3 =>
...
FlatMap(aN, aN => Return(aN)))))
```

Если теперь закинуть `unsafeRun` на нашу тестовую прогу `p`, она будет вертеться вечно без стек-оверфлоу — именно то, что доктор прописал. Наша `unsafeRun` не даст стеке лопнуть даже на бесконечно рекурсивных `IO`-монстрах. Что мы тут наворотили, пацаны? Когда JVM-машина зовёт функцию, она как пьяный на код-ревью кидает фрейм в колл-стек, чтоб потом вернуться и допить кофе. Мы этот цирк с контролем потоком вытащили на свет в нашем `IO`-типе. `unsafeRun`, интерпретируя `IO`-прогу, чует: то ли эффект через `Suspend(s)` запускать, то ли подпрограмму через `FlatMap(x, f)` дёргать. Вместо того чтоб стек раздувать как воздушный шарик на tail-call-оптимизации, которая в Scala иногда подводит, `unsafeRun` сам вызывает `x()`, потом `f` на результате, а `f` мгновенно кидает `Suspend`, `FlatMap` или `Return` — и контроль снова у `unsafeRun`, как в кооперативной многозадачности из 90-х. Наш `IO` — чистой воды *coroutine*, который дружит с `unsafeRun` как старые кореша: шлёт запросы `Suspend` или `FlatMap`, приостанавливается, отдаёт руль, а `unsafeRun` толкает вперёд по одному саспенду, как трамплин для рекурсивного прыжка. Функция вроде `unsafeRun` — это *trampoline*, а вся херня называется *trampolining* — возвращаешь контроль в один большой цикл, и стек остаётся чистым, как совесть после рефакторинга.

### 13.3.2 Траполин: Общее решение проблемы переполнения стека

В нашем `IO`-монaде `resume`-функции вообще не обязаны ебаться с сайд-эффектами — это не требование. Этот `IO` — универсальный траполин для любых вычислений, даже чистых, где I/O и не пахнет! `StackOverflowError` в Scala выскакивает везде, где композная функция накидала вызовов больше, чем стек выдержит — классика, через которую каждый FP-ветеран прошёл, как через чистилище рекурсивных парсеров в проде. Демонстрировать проще простого:

> Мы строим композную функцию `g` из 100 000 функций, где каждая вызывает следующую.

```scala
scala> val f = (x: Int) => x
val f: Int => Int = <function1>
scala> val g = List.fill(100000)(f).foldLeft(f)(_ compose _)
val g: Int => Int = <function1>
scala> g(42)
java.lang.StackOverflowError
```

Она, бля, упадёт и на куда меньших композах — попробуй сам, если не веришь, стек JVM не резиновый. К счастью, фиксим нашим `IO`-монaдом на раз-два:

```scala
scala> val f = (x: Int) => IO.Return(x)
val f: Int => IO[Int] = <function1>
```
