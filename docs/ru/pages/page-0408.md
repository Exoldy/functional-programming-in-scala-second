#### УПРАЖНЕНИЕ 13.5

*Сложное*: Полноценную неблокирующую (non-blocking) I/O библиотеку мы тут пилить не станем — это был бы марафон на выживание, — но если у тебя чешется, ковыряйся сам на базе `java.nio` (API зацени по [ссылке](http://mng.bz/uojM)). На разогрев попробуй замутить асинхронное чтение (read) из `AsynchronousFileChannel` (API [тут](http://mng.bz/X30L)):

```scala
def read(file: AsynchronousFileChannel,
fromPosition: Long,
numBytes: Int): Free[Par, Either[Throwable, Array[Byte]]]
```

### 13.5.1 Композиция свободных алгебр

Вместо того, чтоб ебаться напрямую с файловым I/O через `Free[Par, A]`, как в каком-то imperative аду из нулевых, мы вводим свежий алгебраический тип данных — он описывает все поддерживаемые операции с файлами, ровно как мы мутили с `Console`:

```scala
enum Files[A]:
case ReadLines(file: String) extends Files[List[String]]
case WriteLines(file: String, lines: List[String]) extends Files[Unit]
def toPar: Par[A] = ???
def toThunk: () => A = ???
object Files:
def readLines(file: String): Free[Files, List[String]] =
Free.Suspend(Files.ReadLines(file))
def writeLines(file: String, lines: List[String]): Free[Files, Unit] =
Free.Suspend(Files.WriteLines(file, lines))
```

Метод `toPar` переводит наши примитивы в значения `Par`, используя те самые неблокирующие имплы, что мы напилили раньше. А `toThunk` скорее всего юзает блокирующие версии доступа к файловой системе — классика, чтоб не расслаблялись. `Files` даёт те же плюшки, что и с `Console`: чистые интерпретаторы на любой вкус, типа `toState`, который вообще ни хуя не трогает файловую систему, просто имитирует.
