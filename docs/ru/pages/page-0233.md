> Часть 2: Функциональный дизайн и библиотеки комбинаторов / Глава 8: Property-based testing / Ответы на упражнения 8.6

Вызов `flatMap` на `State` не прокатит — улетит в бесконечную рекурсию, как в те старые добрые времена, когда стек оверфлоуил от неосторожного рекурса без хвоста. Чтобы чётко указать, что нам нужен именно метод с `State`, вызываем extension-метод напрямую, без этих имплицитных фокусов:

```scala
extension [A](self: Gen[A]) def flatMap[B](f: A => Gen[B]): Gen[B] =
State.flatMap(self)(f)
```

С помощью `flatMap` теперь можно слепить вариант `listOfN`, который жрёт размер как `Gen[Int]`, на базе той `listOfN`, что мы раньше намутили с целочисленным размером — чистый реюз, пацаны, без копипасты:

```scala
extension [A](self: Gen[A]) def listOfN(size: Gen[Int]): Gen[List[A]] =
size.flatMap(listOfN)
```

#### ОТВЕТ 8.7

Чтобы вероятности были равными, как в честной лотерее, берём нашу старую `boolean`-генераторку для подбрасывания монетки: если выпадает `true` — хватаем `g1`, а на `false` — `g2`. Используем `flatMap` на `boolean`, чтоб монетка была доступна для выбора следующего генератора — никаких магических peek'ов, всё по-честному:

```scala
def union[A](g1: Gen[A], g2: Gen[A]): Gen[A] =
boolean.flatMap(b => if b then g1 else g2)
```

#### ОТВЕТ 8.8

`weighted` лепим по аналогии с `union`, только вместо монетки генерим вероятность из [0, 1) — как нормализованный рандомный вес. Если сгенерированная хрень меньше нормализованного веса — берём `g1`, иначе — в топку. Классика, без лишнего дерьма:

```scala
def weighted[A](g1: (Gen[A], Double), g2: (Gen[A], Double)): Gen[A]
val g1Threshold = g1(1).abs / (g1(1).abs + g2(1).abs)
State(RNG.double).flatMap(d => if d < g1Threshold then g1(0) else g2(0))
```

#### ОТВЕТ 8.9

Для `&&` возвращаем функцию, которая сначала прогоняет свойство слева от `&&`, а если оно не обосралось — тогда и справа. Последовательность как в пайплайне CI/CD: сначала тесты слева, потом справа, и только если оба зелёные — деплоим. Никаких параллелизмов, чистая линейка:

```scala
extension (self: Prop) def &&(that: Prop): Prop =
(n, rng) => self(n, rng) match
case Passed => that(n, rng)
case x => x
```
