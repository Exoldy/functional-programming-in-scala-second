## 367 13.3 Избегаем StackOverflowError

```scala
scala> val g = List.fill(10000)(f).foldLeft(f)((a, b) =>
|
x => IO.suspend(a(x).flatMap(b))
val g: Int => IO[Int] = <function1>
```

![Страница 0396, изображение 1](./assets/page-0396/image-01.png)

> Создаём огромную левосторонне вложенную цепочку flatMap-вызовов — стек в ахуе.

```scala
scala> val x1 = g(0).unsafeRun()
val x1: Int = 0
scala> val x2 = g(42).unsafeRun()
val x2: Int = 42
```

Но здесь, бля, никакого I/O и в помине нет. `IO` — это чисто маркетинговое название, короче, от того, что `Suspend` может проглотить функцию с побочками. А по сути, это не монада для I/O, а батут для хвостовых вызовов — чтоб стек не рвал жопу! Давай переименуем в `TailRec`, чтоб не ебать мозги:

```scala
enum TailRec[A]:
case Return(a: A)
case Suspend(resume: () => A)
case FlatMap[A,B](sub: TailRec[A],
k: A => TailRec[B]) extends TailRec[B]
def flatMap[B](f: A => TailRec[B]): TailRec[B] =
FlatMap(this, f)
```

![Страница 0396, изображение 2](./assets/page-0396/image-02.png)

> Переименовали в `run`, потому что ловить побочки теперь не надо — чистый трамполин.

```scala
def map[B](f: A => B): TailRec[B] =
flatMap(a => Return(f(a)))
@annotation.tailrec final def run: A = this match
case Return(a) => a
case Suspend(r) => r()
case FlatMap(x, f) => x match
case Return(a) => f(a).run
case Suspend(r) => f(r()).run
case FlatMap(y, g) => y.flatMap(a => g(a).flatMap(f)).run
```

`TailRec` позволяет накинуть трамполин на любую функцию `A => B` — просто вертай `TailRec[B]` вместо голого `B`. Только что видели, как `Int => Int` превратили в `Int => TailRec[Int]`. Прогу переписали: `flatMap` для склейки функций<sup>8</sup> и `Suspend` перед каждым сраным вызовом. Медленнее прямых звонков? Может, чуток, но стек предсказуемый, как швейцарские часы, — и это, пацаны, победа над JVM-имбой.<sup>9</sup>

<sup>8</sup> Это клейсли-композиция из 11-й главы. Короче, трамполин юзает клейсли в монаде `TailRec`, а не ту примитивную функцию-склейку.  
<sup>9</sup> Когда хвостовые вызовы без `TailRec` JVM не оптимизирует, то трамполин даже быстрее прямых (стек-сейвти в придачу). Оверхед от разборки/сборки стек-фреймов жирнее, чем обёртка в `Suspend`. Вариации `TailRec` есть — не копали глубоко, не обязательно после каждого вызова в главный цикл прыгать, хватит периодически, чтоб стек не тонул. Можно и на эксепшенах слепить то же самое. Загляни в `Throw.scala` из кода главы.
