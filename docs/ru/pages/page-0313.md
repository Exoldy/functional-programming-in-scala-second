### 11.1 Функторы: Обобщение функции map

В первых двух частях мы наворотили кучу разных либ комбинаторов — от парсеров до генераторов. Каждый раз стартовали с горстки базовых примитивов, потом на их костях лепили комбинаторы чисто алгебраически, и неизбежно замечали: блядь, похожие паттерны торчат из всех этих либов, как одинаковые баги в разных проектах. Например, для каждого типа данных мы пилили `map` — чтоб поднять чистую функцию с одним аргументом в контекст этого типа данных. Для `Gen`, `Parser` и `Option` сигнатуры были такие:

```scala
extension [A](ga: Gen[A])
  def map[B](f: A => B): Gen[B]
extension [A](pa: Parser[A])
  def map[B](f: A => B): Parser[B]
extension [A](oa: Option[A])
  def map[B](f: A => B): Option[B]
```

Эти сигнатуры отличаются только конкретным типом данных (`Gen`, `Parser` или `Option`). Идею типа данных с `map` можно зацепить в трейт Scala — как универсальный интерфейс, чтоб не ебаться с каждым контейнером по-отдельности:

```scala
trait Functor[F[_]]:
  extension [A](fa: F[A])
    def map[B](f: A => B): F[B]
```

Тут трейт параметризован конструктором типов (type constructor) `F[_]`, как мы делали с `Foldable` в прошлой главе.<sup>2</sup> Вместо выбора конкретного `F[_]` вроде `Gen` или `Parser`, трейт `Functor` параметричен по выбору `F`. Вот инстанс для `List`:

```scala
given listFunctor: Functor[List] with
  extension [A](as: List[A])
    def map[B](f: A => B): List[B] =
      as.map(f)
```

Говорим, что конструктор типов вроде `List` (или `Option`, `F`) — это функтор, а инстанс `Functor[F]` — доказательство, что `F` реально функтор.<sup>3</sup>

Что с этой абстракцией можно вытворять? Как мы не раз в книге делали — берём интерфейс и алгебраически играем с ним, как с Lego в FP-продакшене, и нарываем полезные функции. Притормози тут, подумай, что крутого можно слепить чисто на `map` (если вообще). Взглянем на пример. Если у нас `F[(A, B)]`, где `F` — функтор, то можно размазать `F` по паре и получить `(F[A], F[B])` — чистая дистрибутивность, как апдейт в React, только без мутаций.

<sup>2</sup> Вспомним: конструктор типов (type constructor) применяется к типу, чтоб родить новый тип. Например, `List` — это конструктор, а не тип сам по себе. Значений типа `List` не существует, но применим его к `Int` — и вуаля, тип `List[Int]`. То же с `Parser` на `String` даёт `Parser[String]`. Как higher-kinded types (HKT) в Cats — сначала охуеваешь, потом не отпускаешь.

<sup>3</sup> Заметим: компилится инстанс — не значит, что это функтор. Законы алгебры проверяй, чтоб не было сюрпризов в runtime, как с ленивыми мемами.
