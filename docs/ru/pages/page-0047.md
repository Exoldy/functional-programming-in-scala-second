# Страница 0047
[<- Страница 0046](./page-0046) | [Индекс страниц](./) | [Страница 0048 ->](./page-0048)

> Часть 1: Введение в функциональное программирование / Глава 2: Первые шаги с функциональным программированием в Scala / 2.1 Знакомство с языком Scala

Этот метод объявлен как `private`, то есть его хуй вызовешь из кода за пределами объекта `MyProgram` — чисто внутренний цирк. Функция жрёт `Int` и возвращает `String`, но заметьте, тип возврата не прописан. Scala обычно сам просекает типы возврата методов, так что можно забить, но по хорошему тону — всегда явно пиши типы для методов, которые другие пацаны будут юзать. А этот наш приватный, так что нахуй аннотации, опускаем.

Тело метода — два стейта, отступы одинаковые, как в кодерской тусовке. Такая последовательность стейтов на одном уровне отступа зовётся *блоком*. Стейты разлучают либо переносами строк, либо semicolons. Тут мы на переносах, так что точка с запятой не нужна — Scala не Java, не ссытся без неё. Scala тянет фигурные скобки вокруг блоков, как в Java или C, но чаще всего их можно выкинуть нахуй. В этой книге мы скобки жмём, где можно.

Первый стейт в блоке объявляет `String` с ником `msg` через ключ `val`. Просто чтобы дать имя строке, чтоб потом ссылаться, а не копипастить как лох. `val` — это неизменяемая переменная, так что внутри тела метода `formatAbs` имя `msg` всегда будет тыкать в одну и ту же `String`-ценность. Компилятор Scala взвоет, если попробуешь переприсвоить `msg` чему-то другому в том же блоке — как строгий сержант на плацу.

Scala тянет *мутабельные* (mutable) переменные через `var`, но `val` юзают в разы чаще, и слава богу.

Помните, метод просто отдаёт значение правой стороны, так что `return` не нужен — это не C с его return-ебливыми ритуалами. Тут правая сторона — блок. В Scala ценность многостейтного блока равна тому, что возвращает последний экспрешн в нём, как финальный вердикт после код-ревью. Короче, результат метода `formatAbs` — это то, что отвалил вызов `msg.format(x, abs(x))`.

Наконец, наш `printAbs` — это внешняя оболочка, которая ныряет в чисто-функциональное нутро и херачит ответ в консоль. Такие методы мы иногда зовём *процедурами* (procedures, или *нечистыми функциями*), а не функциями, чтоб подчеркнуть, что они с сайд-эффектами, как вечеринка с алкашом в углу:

```scala
@main def printAbs(): Unit =
  println(formatAbs(-42))
```

Мы решили определить `printAbs` с пустым списком параметров. Но могли и скобки вообще выкинуть:

```scala
@main def printAbs: Unit =
  println(formatAbs(-42))
```

По конвенции[^2] скобки лепят на процедурах (то есть нечистых функциях) без параметров. Если список параметров задан пустыми скобками, то и вызывать метод надо с пустыми скобками.

[^2]: Идея в том, чтоб скобками сигнализировать о сайд-эффектах, как мигалкой на тачке. Но это чисто для методов без параметров. Для тех, что параметры жрут, скобки всегда обязательны, и не все либы (libraries) эту конвенцию держат. В итоге некоторые скалозники путаются в этом цирке. Позже в книге разберём другие способы отличать чистые функции от нечистых.
