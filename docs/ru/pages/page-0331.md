> Часть 3: Общие структуры в функциональном дизайне / Глава 11: Мон ады / 11.7 Ответы на упражнения

- Мон ада — это реализация одного из минимальных сэтов монадических комбинаторов, которая пляшет под законы ассоциативности и идентичности, как по нотам, без единого косяка.

- Минимальные сэты монадических комбинаторов — это `unit` и `flatMap`; `unit` и `compose`; `unit`, `map` и `join`. Выбирай любой, главное — чтобы Scala не сломалась на flatMap'ах в проде.

- Законы монад — это **ассоциативность** — `x.flatMap(f).flatMap(g) == x.flatMap(a => f(a).flatMap(g))`; **правый идентичник** — `x.flatMap(unit) == x`; **левый идентичник** — `unit(y).flatMap(f) == f(y)`. Нарушил — и привет, баг в цепочке flatMap'ов, который вылезет только под лоадом.

- Все монады — функторы, но не все функторы монады. Функтор — это как базовый велик без скоростей, монада — с полным фаршем для цепочек трансформаций.

- Мон ады есть для кучи типов из книги: `Option`, `List`, `LazyList`, `Par` и `State[S, _]`. Не спи на них — в проде они спасут от null'ов и асинхронных дерьмовин.

- Контракт `Monad` не лезет в душу: ему пох, что там внутри твоего типа, главное — чтобы ассоциативность и идентичность держались, как обещания в контракте с заказчиком.

- Написать инстанс `Monad` для твоего type constructor'а — это не фигня для резюме, а реальный профит: за минимальный сет комбинаторов ты получаешь весь арсенал derived-операций, типа for-компрехеншенов и sequence. Экономия на кофе после код-ревью.

### 11.7 Ответы на упражнения

#### ОТВЕТ 11.1

Инстансы `Option`, `List` и `LazyList` просто делегируют всё в метод `flatMap` на соответствующих типах — никаких изысков, чистый прокси, чтоб не дублировать код:

```scala
given optionMonad: Monad[Option] with
  def unit[A](a: => A) = Some(a)
  extension [A](fa: Option[A])
    override def flatMap[B](f: A => Option[B]) =
      fa.flatMap(f)
given listMonad: Monad[List] with
  def unit[A](a: => A) = List(a)
  extension [A](fa: List[A])
    override def flatMap[B](f: A => List[B]) =
      fa.flatMap(f)
given lazyListMonad: Monad[LazyList] with
  def unit[A](a: => A) = LazyList(a)
  extension [A](fa: LazyList[A])
    override def flatMap[B](f: A => LazyList[B]) =
      fa.flatMap(f)
```

Помнишь, `Par` — это opaque type alias с методом `flatMap`, который висит на extension method? Тут надо аккуратно сказать Scala, что нам нужен именно этот `flatMap`-extension, а не какой-то импортный хлам из ада.
