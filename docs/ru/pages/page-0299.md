# Страница 0299
[<- Страница 0298](./page-0298) | [Индекс страниц](./) | [Страница 0300 ->](./page-0300)

> Часть 3: Общие структуры в функциональном дизайне / Глава 10: Монойды / 10.8 Заключение

Ассоциативность — это как cheat code, который позволяет фолдить любой тип ``Foldable``, и ещё параллельно, без лишнего геморроя. Монойды композабельны на раз, собираешь из них фолды декларативно и reusable, как лего в продакшене. ``Monoid`` — наша первая чистая абстрактная алгебра, только операции и законы, без конкретики, которая всех бесит. Мы уже видели, как писать полезные функции, которые нихуя не знают про аргументы, кроме что тип — монойд, и это работает. Этот абстрактный вайб, когда мозг переключается на высший уровень, будем качать дальше в части 3. Посмотрим другие алгебраические интерфейсы и разберём, как они упаковывают те паттерны, которые мы тут по книге мусолили до тошноты.

### Итоги

Монойд — чисто алгебраическая хрень: ассоциативная бинарная операция плюс нейтральный элемент для неё.

Ассоциативность позволяет переставлять скобки в выражении, не меняя результата — чистый кайф для параллелизма.

Примеры монойдов: конкатенация строк с пустой строкой как единицей, сложение интов с ``0``, умножение интов с ``1``, булев ``and`` с ``true``, булев ``or`` с ``false``, конкатенация листов с ``Nil``.

Монойды моделируем трейтами с операциями ``combine`` и ``empty``.

Монойды дают полезные generic-функции для кучи типов данных — универсальный инструмент в FP-арсенале.

Функция ``combineAll`` фолдит элементы листа в одно значение по монойду.

``foldMap`` мапит каждый элемент листа в новый тип, а потом комбайнит по монойду.

``foldMap`` реализуется через ``foldLeft`` или ``foldRight``, а и ``foldLeft``, и ``foldRight`` — через ``foldMap``.

Все монойды из главы объединены только монойдальной структурой — ничего лишнего, чистая абстракция.

Типклассы описывают алгебраическую структуру и дают канонические инстансы для типов.

Контекстные параметры начинаются с ключевого слова ``using``. На месте вызова Scala ищет given-инстансы. Given определяются ключевым словом ``given``.

Контекстные параметры можно передать явно ключевым словом ``using`` на месте вызова.

Given-инстансы и контекстные параметры в Scala сбрасывают type-driven композицию на компилятор — лень в лучшем виде.

Метод ``summon`` возвращает given-инстанс в скоупе для типа. Нет инстанса — компиляция в могилу.
