## 71 4.3 Тип данных Option

- *Не катит для полиморфного кода.* Для некоторых типов выхода мы даже sentinel-значение не слепим, хоть обосрёмся от желания! Возьмём функцию вроде `max`, которая шарит максимум в последовательности по кастомному сравнению: `def` `max[A](xs:` `Seq[A])(greater:` `(A,A)` `=>` `Boolean):` `A`. Если инпут пустой — хуй придумаешь значение типа `A`, а `null` тут не прокатит, потому что `null` только для непримитивов, и `A` может быть примитивам типа `Double` или `Int`.

- *Требует от коллеров специальной политики или конвеншена.* Чтобы правильно юзать `mean`, коллерам придётся не просто вызвать `mean` и слопать результат, а ещё и подстраиваться. Такие функции с приколами хуй передашь в higher-order, они все аргументы одинаково жрут.

Второй вариант — заставить коллера впарить аргумент, что делать, если инпут не переварить:

```scala
def mean(xs: Seq[Double], onEmpty: Double): Double =
if xs.isEmpty then onEmpty
else xs.sum / xs.length
```

Это делает `mean` тотальной функцией, но с кучей гемора; немедленные коллерам надо заранее знать, как хэндлить undefined-кейс, и возвращать только `Double`. А если `mean` — часть большой вычислухи, и надо её нахуй оборвать, если `mean` undefined? Или свернуть в совсем другую ветку? Просто параметр `onEmpty` такой свободы не даёт. Нам нужно отложить решение по undefined на уровень повыше, где оно к месту.

### 4.3 Тип данных Option

Решение — прямо в типе возврата сказать: функция может и не ответить. Это как сказать коллеру: "Твоя стратегия обработки ошибок, бро". Вводим новый тип: `Option`. Как я говорил раньше, он есть в стандартке Scala, но для профита перепишем сами:

```scala
enum Option[+A]:
case Some(get: A)
case None
```

`Option` имеет два конструктора: либо определено — тогда `Some`, либо пиздец — `None`. Юзаем `Option` для нашей `mean` вот так:

```scala
import Option.{Some, None}
def mean(xs: Seq[Double]): Option[Double] =
if xs.isEmpty then None
else Some(xs.sum / xs.length)
```
