### Ответы на упражнения

#### Ответ 2.1

```scala
def fib(n: Int): Int =
  @annotation.tailrec
  def go(n: Int, current: Int, next: Int): Int =
    if n <= 0 then current
    else go(n - 1, next, current + next)
  go(n, 0, 1)
```

Как и с `factorial`, лепим локальную хвостовую рекурсивную функцию `go`, которая жрёт `n` и на каждом шаге откусывает по единичке. Кроме `n`, `go` ещё параметры таскает — текущий (n-й) и следующий (n+1-й) числа Фибоначчи. Допёрли до нуля — вуаля, n-е значение готово, просто кидаем текущее фибоначчи. Иначе рекурсим дальше: следующее число Фибоначчи сдвигаем в текущую позицию, а новое следующее считаем как сумму тех двух фибоначчишек, что прилетели на эту итерацию. Это как конвейер на заводе — не даём стеку переполниться, чтоб не словить StackOverflowError (ошибку переполнения стека), как те новички в мемах.

#### Ответ 2.2

```scala
def isSorted[A](as: Array[A], gt: (A,A) => Boolean): Boolean =
  @annotation.tailrec
  def loop(n: Int): Boolean =
    if n + 1 >= as.length then true
    else if gt(as(n), as(n + 1)) then false
    else loop(n + 1)
  loop(0)
```

Реализация почти один в один с `findFirst` — локальная хвостовая рекурсия прёт по массиву. Вызываем `gt` на n-м и (n+1)-м элементах: если n-й больше (n+1)-го, сразу `false`. Иначе рекурсим дальше, пока не упрёмся в конец массива. Классический паттерн, чтоб не мутировать и не плодить промежуточные коллекции, — чисто, хвостово, без лишнего говна в памяти.
