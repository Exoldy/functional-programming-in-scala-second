> Часть 2: Функциональный дизайн и библиотеки комбинаторов / Глава 9: Комбинаторы парсеров / 9.1 Сначала проектируем алгебру

Но выбор между двумя парсерами — это та ещё универсальная хуйня, которая пригодится в любой жопе, независимо от типа их результата, так что давай сделаем полиморфным, как и положено в FP:

```scala
extension [A](p: Parser[A]) def or(p2: Parser[A]): Parser[A]
```

Мы ожидаем, что `string("abra").or(string("cadabra"))` отработает, как часы, когда хотя бы один из `string`-парсеров преуспеет (succeeds) — типа, или abra, или cadabra, или вообще похуй:

```scala
string("abra").or(string("cadabra")).run("abra") == Right("abra")
string("abra").or(string("cadabra")).run("cadabra") == Right("cadabra")
```

Кстати, этому `or`-комбинатору можно заюзать сочный инфиксный синтаксис, типа `s1 | s2` или, на худой конец, `s1 or s2` — как в жизни, выбирай удобное.

**Listing 9.1** Добавляем инфиксный синтаксис к парсерам

```scala
trait Parsers[ParseError, Parser[+_]]:
  def char(c: Char): Parser[Char]
  def string(s: String): Parser[String]
```

> Модификатор `infix` позволяет юзать `or` как оператор — без него никуда, если хочешь сахарного синтаксиса (syntactic sugar).

```scala
extension [A](p: Parser[A])
  def run(input: String): Either[ParseError, A]
```

> Оператор `|` — чистый алиас для `or`. Инфикс-модификатор не нужен, потому что все символы по умолчанию операторы, Scala такая Scala.

```scala
infix def or(p2: Parser[A]): Parser[A]
def |(p2: Parser[A]): Parser[A] =
  p.or(p2)
```

Мы пока не выбрали, как `Parser` репрезентировать в железе, но имея `P: Parsers`, импорт `P.*` позволяет лепить `string("abra") | string("cadabra")` — и это прокатит на любой имплементации `Parsers`, без подвохов. Весь остаток главы будем швырять `a | b` направо-налево, как мемные стрелки в Haskell. Можем жрать разные строки, но повторений нет — как парсить три повтора `string("abra") | string("cadabra")`? Опять комбинатор в помощь, добавляем `listOfN`<sup>6</sup>:

```scala
extension [A](p: Parser[A]) def listOfN(n: Int): Parser[List[A]]
```

Сделали `listOfN` параметрическим по `A`, потому что ему похуй, `Parser[String]`, `Parser[Char]` или какая-нибудь экзотика — универсал, как `map` в коллекциях. Вот что ждём от него в деле:

```scala
val p = (string("ab") | string("cad")).listOfN(3)
p.run("ababcad") == Right(List("ab", "ab", "cad"))
p.run("cadabab") == Right(List("cad", "ab", "ab"))
p.run("ababab") == Right(List("ab", "ab", "ab"))
```

<sup>6</sup> Это должно напомнить тебе похожую функцию из прошлой главы — ну, ту, где мы уже еблись с повторениями, помнишь?
