> Часть 3: Общие структуры в функциональном дизайне / Глава 10: Моноиды / 10.1 Что такое моноид?

![Страница 0286, изображение 1](./assets/page-0286/image-01.png)

Чисто абстрактная херня — алгебраическая структура. Заметьте, пацаны, что кроме как по законам моноида, разные инстансы `Monoid` вообще ни хуя не пересекаются — как случайные типы в пивнухе после дедлайна. Ответ на вопрос *«Что такое моноид?»* проще простого: моноид — это тип, операции моноида и строгий набор законов. Моноид — это чистая алгебра, и точка. Конечно, можно накидать свою интуицию, пялясь на кучу конкретных примеров, типа «ааа, это как сложение строк или произведение матриц», но эта интуиция всегда будет размытой, как кофе из автомата, и ничто не гарантирует, что следующий моноид из подвала не обосрёт вашу картину мира!

#### УПРАЖНЕНИЕ 10.1

Дайте инстансы `Monoid` для сложения и умножения целых чисел, а также для булевых операторов:

```scala
val intAddition: Monoid[Int]
val intMultiplication: Monoid[Int]
val booleanOr: Monoid[Boolean]
val booleanAnd: Monoid[Boolean]
```

![Страница 0286, изображение 2](./assets/page-0286/image-02.png)

#### УПРАЖНЕНИЕ 10.2

Дайте инстанс `Monoid` для комбайна значений `Option`:

```scala
def optionMonoid[A]: Monoid[Option[A]]
```

![Страница 0286, изображение 3](./assets/page-0286/image-03.png)

#### УПРАЖНЕНИЕ 10.3

Функция, у которой аргумент и возвращаемый тип совпадают, иногда зовётся `endofunction`.<sup>2</sup> Напишите моноид для эндоморфизмов:

![Страница 0286, изображение 4](./assets/page-0286/image-04.png)

```scala
def endoMonoid[A]: Monoid[A => A]
```

#### УПРАЖНЕНИЕ 10.4

Используйте фреймворк property-based тестирования из части 2, чтоб реализовать свойство для законов моноида. Проверьте им моноиды, что мы написали:

```scala
def monoidLaws[A](m: Monoid[A], gen: Gen[A]): Prop
```

<sup>2</sup> Греческий префикс *endo-* значит *«внутри»*, в смысле, что кодомен эндоморфизма лежит внутри его домена.
