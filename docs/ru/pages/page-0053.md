### 2.3.2 Пишем нашу первую функцию высшего порядка

Раз уж у нас теперь есть `factorial`, давай подкрутим предыдущую программу и засунуть её внутрь — классический рефакторинг, пацаны.

**Листинг 2.2** Простая программа с функцией факториала

![Страница 0053, изображение 1](./assets/page-0053/image-01.png)

```scala
object MyProgram:
...
private def formatAbs(x: Int) =
val msg = "The absolute value of %d is %d."
msg.format(x, abs(x))
```

> Здесь определения `abs` и `factorial`.

```scala
private def formatFactorial(n: Int) =
val msg = "The factorial of %d is %d."
msg.format(n, factorial(n))
@main def printAbsAndFactorial: Unit =
println(formatAbs(-42))
println(formatFactorial(7))
```

Две эти функции, `formatAbs` и `formatFactorial`, — почти клоны друг друга, бля, копипаста чистой воды. Но если не ленимся, обобщим их в одну `formatResult`, которая жрёт функцию как аргумент и применяет её к своему аргументу — вот она, магия FP, где код перестаёт быть одноразовым.

> `f` должна быть функцией от `Int` к `Int`.

```scala
def formatResult(name: String, n: Int, f: Int => Int) =
val msg = "The %s of %d is %d."
msg.format(name, n, f(n))
```

Наша `formatResult` — это полноценная функция высшего порядка (higher-order function), которая принимает другую функцию по имени `f` (смотри сайдбар (sidebar) про конвенции имён, ха). Тип для `f` даём как для любого параметра — `Int => Int` (читается *int ту инт* или *функшн фром инт ту инт*), то есть ждёт инт на входе, инт на выходе. `abs` идеально ложится: инт влетает, инт вылетает. То же с `factorial`. Короче, пихаем `abs` или `factorial` как `f` в `formatResult` — и вуаля, универсальность на марше, без этой imperative хуйни.

```scala
scala> formatResult("absolute value", -42, abs)
val res0: String = "The absolute value of -42 is 42."
```

![Страница 0053, изображение 2](./assets/page-0053/image-02.png)

```scala
scala> formatResult("factorial", 7, factorial)
val res1: String = "The factorial of 7 is 5040."
```

**Конвенции именования переменных.**  
Стандарт — юзать `f`, `g`, `h` для параметров высшего порядка. В FP мы фанаты минимализма: короткие имена, даже однобуквенные — норма, потому что контекст всё равно яснее любой эпичной переменной. Я сам так кодил 16 лет, и мир не рухнул.
