## 399 14.2 Тип данных, который заставит сайд-эффекты сидеть в своей песочнице

Некоторые примитивы. Язык для разговора о мутабельных ячейках памяти должен уметь эти базовые команды, как старый добрый ассемблер (assembler), только без боли в жопе:

- Выделить новую мутабельную ячейку
- Записать в мутабельную ячейку
- Прочитать из мутабельной ячейки

Структура данных для наших мутабельных ссылок — это просто обёртка вокруг приватного `var`, типа как сейф с токеном доступа, чтоб не всякий дебил мог полезть:

```scala
final class STRef[S, A] private (private var cell: A):
def read: ST[S,A] = ST(cell)
def write(a: => A): ST[S, Unit] = ST.lift[S, Unit]:
s =>
cell = a
((), s)
object STRef:
def apply[S, A](a: A): ST[S, STRef[S, A]] =
ST(new STRef[S, A](a))
```

Методы на `STRef` для чтения и записи в ячейку — чистые, возвращают `ST`-действия, без всякой мутации на лету. Обратите внимание: тип `S` — это не тип мутируемой ячейки, и мы никогда не юзаем значение типа `S` по-настоящему. Но чтоб запустить `apply` и реально исполнить одно из этих `ST`-действий, нам нужен экземпляр типа `S`. Этот токен — как пропуск в зону мутации или доступа к ячейке, но больше ни хуя не делает. 

Единственный способ слепить `STRef` — вызвать метод `apply` на компаньоне `STRef`. `STRef` конструируется с начальным значением для ячейки типа `A`. Но голый `STRef` не возвращается — вместо этого выдаётся `ST[S, STRef[S, A]]`-действие, которое слепит `STRef` при запуске с токеном типа `S`. Важно: это `ST`-действие и `STRef`, который оно создаёт, помечены одним и тем же типом `S` — scoping на замке, как в тюрьме с камерами. 

Теперь давай набросаем тривиальный `ST`-программку. Пока кринжово, потому что приходится выбирать тип `S` наобум. Здесь берём `Nothing` чисто для прикола:

```scala
for
r1 <- STRef[Nothing, Int](1)
r2 <- STRef[Nothing, Int](1)
x
<- r1.read
y
<- r2.read
_
<- r1.write(y + 1)
_
<- r2.write(x + 1)
a
<- r1.read
b
<- r2.read
yield (a, b)
```

Эта мелкая программка выделяет две мутабельные `Int`-ячейки, меняет их содержимое местами, прибавляет по единице к обеим и читает новые значения. Но запустить её пока не выйдет — базовая функция всё ещё запакована в непрозрачность `ST` (и мы бы никогда не смогли передать ей значение типа `Nothing`). Давай пофиксим это дерьмо.
