### 6.5 Общий тип данных для действий со состоянием

Функции, которые мы только что накатали — `unit`, `map`, `map2`, `flatMap` и `sequence`, — нихуя не завязаны на генерацию рандома. Это универсальный арсенал для ковыряния в state actions, тип состояния им до пизды, как алкашу цена на нефть. Взять ту же `map` — ей похер, что она жрёт `RNG`-действия со стейтом, дадим ей сигнатуру пошире, чтоб не стесняться:

```scala
def map[S, A, B](action: S => (A, S))(f: A => B): S => (B, S)
```

Сигнатуру поменяли — имплементацию `map` не трогаем, она и так универсальная. Более общий вариант торчал на поверхности, как баг в PR, который все пропустили, пока не въехали. Пора тогда слепить тип покруче `Rand`, чтоб любой стейт жрал без вопросов:

```scala
type State[S, +A] = S => (A, S)
```

Тут `State` — это сокращёнка от *вычисления, которое таскает стейт за собой, как рюкзак с кирпичами*, *state action*, *state transition* или даже *statement* (загляни в след. секцию, там разжую). Можем вместо алиаса свой тип забабахать, чтоб методы свои навесить без пиздеца от конфликтов с функциями — просто функцию в wrapper завернём, классика, как матрёшку:

```scala
case class State[S, +A](run: S => (A, S))
```

Альтернатива — фича Scala 3 под названием *opaque types*. Внутри определяющего скоупа это как алиас, а снаружи — отдельный тип, не имеющий ни хуя общего с репрезентацией, как клон без ДНК. *Defining scope* — это объект с определением или пакет, если топ-левел. Вот как `State` с opaque type смотрится:

```scala
opaque type State[S, +A] = S => (A, S)
object State:
extension [S, A](underlying: State[S, A])
def run(s: S): (A, S) = underlying(s)
def apply[S, A](f: S => (A, S)): State[S, A] = f
```

Кроме opaque-определения, кидаем extension-метод `run`, чтоб подложную функцию вызвать, и `apply` на компаньоне, чтоб из функции `State` слепить. В обоих случаях внутри скоупа все знают, что `State[S,` `A]` == `S` `=>` `(A,` `S)`, так что конверсии — как дважды два, тривиально и на первый взгляд redundantно. Но снаружи, в другом пакете например, эквивалентность `State[S,` `A]` и `S` `=>` `(A,` `S)` — тёмный лес, вот и нужны эти конвертеры. Подробности в офдоках по opaque types (https://mng.bz/gRR8) и extension methods (https://mng.bz/5mm7).
