## 81 4.4 Тип данных Either

### 4.4 Тип данных Either

Короче, главная фишка этой главы — мы можем пихать фейлы и эксепшены в обычные значения и лепить функции,  
которые выдирают общие паттерны обработки ошибок и восстановления, как универсальный швейцарский нож  
для всей этой хуйни. 

`Option` — не единственный тип для такого цирка, хоть и юзается везде, как дешёвый кофе на код-ревью,  
но он простой, блядь, как валенок. Вы наверняка заметили, что `Option` ни хуя не рассказывает,  
что именно наебнулось в случае эксепшена. Всё, что он может — выдать `None`, мол, значения нет, иди нахуй. 

Но иногда хочется копнуть глубже, как в дебаге продакшена в три часа ночи. Например, `String` с толковой инфой,  
или если эксепшен вылез, то какой именно кусок дерьма это был. Можем слепить свой тип, который закодирует  
любую инфу о фейлах — от 'пиздец' до 'пиздец из-за null в строке 42'. 

Иногда хватает просто знать, что фейл случился — юзаем `Option` и не парься;  
а иногда надо мясо с кровью. В этой секции разберём простое расширение `Option` — тип `Either`,  
который позволяет трекать *почему* именно обосрались. Глянем на определение, пацаны:

```scala
enum Either[+E, +A]:
case Left(value: E)
case Right(value: A)
```

`Either` имеет всего два конструктора, как и `Option` — классика жанра.  
Главное отличие: оба случая тащат за собой значение, не как `Option`, где один пустой, как обещания менеджера. 

Тип `Either` в общем виде — это значения, которые могут быть *одной из двух вещей*,  
как выбор между пивом и энергетиком перед дедлайном. Можно сказать, это *дизъюнктное объединение*  
двух типов, чтоб не путаться в теориях категорий. 

Когда юзаем для успеха/фейла, по священной конвенции `Right` — для успеха  
(каламбур на *right* = правильно, ха-ха, Scala-юмор), а `Left` — для фейла.  
Левому параметру дали говорящее имя `E` (от `error`).[^6]

Давай глянем на `mean` ещё раз — теперь возвращаем `String` при фейле, чтоб не гадать:

```scala
import Either.{Left, Right}
def mean(xs: Seq[Double]): Either[String, Double] =
if xs.isEmpty then
Left("mean of empty list!")
else
Right(xs.sum / xs.length)
```

Иногда хочется закинуть больше мяса об ошибке — например, стек-трейс с точным местом,  
где код решил самоуничтожиться. Тогда просто пихаем эксепшен в `Left`-сторону `Either`,  
и вуаля, дебаг на стероидах:

![Страница 0110, изображение 1](./assets/page-0110/image-01.png)

```scala
import scala.util.control.NonFatal
```

> Паттерн-матч `NonFatal` гарантирует, что не ловим фатальные ошибки, типа `OutOfMemoryError` —  
> чтоб JVM не сдохла окончательно.

```scala
def safeDiv(x: Int, y: Int): Either[Throwable, Int] =
try Right(x / y)
catch case NonFatal(t) => Left(t)
```

[^6]: `Either` часто юзают шире, чтоб закодировать одну из двух возможностей,  
когда плодить новый тип — перебор, как микросервис для 'hello world'.  
Примеры по книге увидим, не ссы.
