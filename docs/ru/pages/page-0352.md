#### УПРАЖНЕНИЕ 12.6

Напишите инстанс `Applicative` для `Validated`, который копит ошибки в `Invalid`, как белка орехи на зиму. В главе 4 мы уже ковырялись в сигнатуре `map2`, подкинув туда лишний `combineErrors:` `(E,` `E)` `=>` `E`-функтор, чтоб было побогаче. Здесь юзайте монадроид для типа ошибок — он как клей для багов, склеит их в кучу. Не забудьте, что `summon` — это ваш крючок, чтоб схватить контекст явно, без имплицитных фокусов-покусов:

```scala
given validatedApplicative[E: Monoid]: Applicative[Validated[E, _]] with
def unit[A](a: => A) = ???
extension [A](fa: Validated[E, A])
override def map2[B, C](fb: Validated[E, B])(f: (A, B) => C) =
???
```

Чтобы добить пример, замутим веб-форму: имя, дата рождения, телефон — классика жанра, как в 90-х на дискотеке.

```scala
case class WebForm(name: String, birthdate: LocalDate, phoneNumber: String)
```

Данные эти, бля, прилетят строками от юзера — грязными, кривыми, как после трёхдневного спринта без сна. Нам надо их проверилить по специфу, чтоб не было сюрпризов. Не прошёл — бац, список ошибок в ебало: "исправь, дебил". Скажем, `name` не пустая (типа, фантомное имя — это не катит), `birthdate` строго в формате `"yyyy-MM-dd"`, а `phoneNumber` — ровно 10 цифер, без букв и пробелов, чтоб не звонить на "мамкин-телефон".

Листинг 12.5 Валидация ввода юзера в веб-форме

```scala
def validName(name: String): Validated[List[String], String] =
if name != "" then Validated.Valid(name)
else Validated.Invalid(List("Name cannot be empty"))
def validBirthdate(birthdate: String): Validated[List[String], LocalDate] =
try Validated.Valid(LocalDate.parse(birthdate))
catch case _: java.time.format.DateTimeParseException =>
Validated.Invalid(List("Birthdate must be in the form yyyy-MM-dd"))
def validPhone(phoneNumber: String): Validated[List[String], String] =
if phoneNumber.matches("[0-9]{10}") then Validated.Valid(phoneNumber)
else Validated.Invalid(List("Phone number must be 10 digits"))
```

Чтобы провалидировать всю эту херню целиком, просто лифтим конструктор `WebForm` аппликативом `map3` — и вуаля, ошибки сами слепятся, как в конструкторе Lego из багов:

```scala

birthdate: String,
```
