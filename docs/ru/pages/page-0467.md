# Страница 0467
[<- Страница 0466](./page-0466) | [Индекс страниц](./) | [Страница 0468 ->](./page-0468)

> Часть 4: Эффекты и I/O / Глава 15: Обработка потоков и инкрементальный ввод-вывод / 15.3 Расширяемые pull'ы и стримы / 15.3.3 Обеспечение безопасности ресурсов

```scala
case Some(p) => p.open(finalizer)
State.Closed() -> next
.flatten
```

У нас два кейса на обработку: когда `open` вызывается на `Open` скоупе и когда на `Closed`. Если скоуп открыт, то сперва лепим новый скоуп под свежий финализатор, а потом вычисляем новое состояние `Open`, которое уже ссылается на этот новенький в своих подсоузах — как будто подвешиваешь новую ветку на семейном древе. Вспомни, сигнатура `modify` позволяет притащить с собой любое значение вместе с апдейченным стейтом; тут мы впариваем `F[Scope[F]]` как то самое "любое". Отложим кейс `Closed` на "после обеда" — у нас `state.modify(…)`, которая возвращает `F[F[Scope[F]]`, и мы её `flatten` в `F[Scope[F]]`. Этот паттерн — чистый кайф для любой state machine, какую ни нафантазируешь, расширяй на здоровье, как Lego в руках FP-маньяка. Текущее состояние пихаем в `Ref`, а переходы крутим через `ref.modify(f).flatten`, где `f` из текущего стейта лепит следующий и возвращает экшн на выполнение после успешного апдейта стейта `Ref`. Возвращаемся к `Closed`: можно было бы рубануть ошибкой на попытку открыть подсоуп на закрытом, но мы хитрее — пропагаем запрос вверх к папочке. Тот тоже может быть в отключке, и так рекурсивно по стеку вверх, как exception в callstack'е, пока не упрёмся в открытый скоуп или в рут. Ошибку кидаем только если рут мёртвый, без вариантов. `close` реализуется в том же духе:

```scala
def close: F[Unit] =
state
.modify:
case State.Open(finalizer, subscopes) =>
val finalizers = (subscopes.reverseIterator.map(_.close) ++
Iterator(finalizer)).toList
def go(rem: List[F[Unit]], error: Option[Throwable]): F[Unit] =
rem match
case Nil => error match
case None => F.unit(())
case Some(t) => F.raiseError(t)
case hd :: tl =>
hd.attempt.flatMap(res =>
go(tl, error orElse res.toEither.swap.toOption))
State.Closed() -> go(finalizers, None)
case State.Closed() => State.Closed() -> F.unit(())
.flatten
```

Вызов close на открытом скоупе переводит его стейт в `Closed`. Лепим экшн, который затыкает все подсоуы и потом запускает финализатор оригинала. Аккуратно реверси подсоуы, чтоб финализаторы шли стеком — LIFO, last in first out, иначе будет как в том меме с перевернутым стаканом пива на код-ревью. И не забывай: все финализаторы должны отработать, даже если ошибки посыплются, как баг-репорты в понедельник. Нельзя просто foreach по списку экшнов — short-circuit на первой херне, и привет, утечка ресурсов.
