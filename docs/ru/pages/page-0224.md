## 195 8.2 Минимизация тестовых случаев

А как бы мы это заебенить? Один вариант — через `forAll`, как типичный костыль:

![Страница 0224, изображение 1](./assets/page-0224/image-01.png)

> Тут мы ленивые, как всегда в Скале, non-strict (ненестрогая оценка) по полной.

```scala
def verify(p: => Boolean): Prop =
  lazy val result = p
  forAll(unit(()))(_ => result)
```

> Результат закеширован в мемо, чтоб не ебаться с повторными вычислениями.

Но это же хуйня полная, пацаны. Мы пихаем unit-генератор, который выдаёт одну единственную херню, а потом её нахуй игнорим — просто чтоб прогнать оценку `Boolean`. 

Хоть и мемоизируем, чтоб не пересчитывало по сто раз, тест-раннер всё равно как пьяный матрос нагуляет куеву тучу кейсов и протестирует `Boolean` в многократном дублировании. 

Допустим, пишем `verify(true).run()` — оно свойство 100 раз прогоняет и выдаёт `OK, passed 100 tests`. 

А проверять свойство, которое всегда `true`, сто раз — это как бензопилой ногти стричь, полный пиздец по ресурсам. 

Нам нужен свежий примитив, чтоб не тратить CPU на фигню. Вспомните, наш `Prop` пока — чисто функция типа `(MaxSize, TestCases, RNG) => Result`, где `Result` либо `Passed` (прошло), либо `Falsified` (опровергнуто). 

Простая реализация примитив `verify` — слепить `Prop`, который на число тестов плюёт с высокой колокольни:

```scala
def verify(p: => Boolean): Prop =
  (_, _, _) => if p then Passed else Falsified("()", 0)
```

Это уже получше, чем `forAll` юзать, но `verify(true).run()` всё равно выплюнет `passed 100 tests`, хоть и протестирует свойство ровно один раз. 

А на деле-то такое свойство не просто "прошло", а доказано после одного прогона — как теорема в Coq, без лишнего шума. 

Короче, нужен новый вид `Result`, чтоб раннер понимал разницу:

```scala
enum Result:
  case Passed
  case Falsified(failure: FailedCase, successes: SuccessCount)
  case Proved
```

Тогда в пропе от `verify` вместо `Passed` кидаем `Proved` (доказано). И тест-раннер подкрутим, чтоб этот кейс разруливал.

**Листинг 8.7.** Как юзать `run`, чтоб разрулить объект `Proved`

```scala
extension (self: Prop)
  def run(
    maxSize: MaxSize = 100,
    testCases: TestCases = 100,
    rng: RNG = RNG.Simple(System.currentTimeMillis)
  ): Unit =
    self(maxSize, testCases, rng) match
      case Falsified(msg, n) =>
        println(s"! Falsified after $n passed tests:\n $msg")
      case Passed =>
        println(s"+ OK, passed $testCases tests.")
      case Proved =>
        println(s"+ OK, proved property.")
```
