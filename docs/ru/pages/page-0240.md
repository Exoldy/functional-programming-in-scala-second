#### ОТВЕТ 8.18

Требование `as.takeWhile(f).forall(f)` `==` `true` ни хуя не гарантирует, что `takeWhile` вернёт самый длинный префикс, для которого свойство держится, как слон в посудной лавке. Чтобы это прочеканить, замутим ещё одно свойство: голова оставшегося хвоста, если он не пустой, должна обосраться на предикате — типа `!f(as(as.takeWhile(f).size))`, но с обработкой кейса, когда `as.size` `==` `as.takeWhile(f).size`. Ещё одно свойство расписывает, как `takeWhile` и `dropWhile` дружат: склеиваем результат `takeWhile` с результатом `dropWhile` на том же списке — и бац, должен вылезти исходный список, как по маслу. Попробуем хоть одно из этих свойств на нашей библиотеке завести — и привет, стена. Каждое требует `Gen[Int` `=>` `Boolean]`, а генерить функции мы ещё не научились, как первокурсники FP. Можно, конечно, тестить высшие функции только на ручных аргументах, без выебонов. Вот, к примеру, конкретное свойство для `takeWhile`:

```scala
def isEven(i: Int) = i % 2 == 0
val takeWhileProp =
Prop.forAll(Gen.int.list)(ys => ys.takeWhile(isEven).forall(isEven))
```

Работает, как часы, но нельзя ли фреймворку поручить генерить функции для `takeWhile` самому, без наших соплей?15<sup>15</sup> Давай разберём варианты по полочкам. Конкретно: есть `Gen[Int]`, хотим слепить `Gen[String` `=>` `Int]`. Как это провернуть? Один вариант — лепим `String` `=>` `Int` функции, которые инпут игнорят нахуй и делегируют всё underlying `Gen[Int]`, как ленивые прокси:

```scala
def genStringIntFn(g: Gen[Int]): Gen[String => Int] =
g.map(i => (s => i))
```

Но это полумеры, бля; мы просто константы генерим, которые инпут в жопу засовывают. В кейсе `takeWhile`, где нужна функция, возвращающая `Boolean`, выйдет вечный `true` или вечный `false` — тестировать такое, как смотреть краску на стене сохнет, нихуя интересного для поведения нашей функции.

<sup>15</sup> Вспомни главу 7: замутили free theorems и как параметричность отрывает нас от необходимости ковырять поведение функции на каждом пиздеце аргумента. Но генерить функции для тестов — это часто спасает задницу в реальной заварушке.
