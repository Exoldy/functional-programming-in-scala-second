#### ОТВЕТ 12.13

Нам надо заимплементить и `traverse`, и `map` для каждого дата-типа — классика жанра. Для `map` просто переиспользуем стандартный `map` на этих типах, без лишней хуйни. А для инстанса `List` берём right fold, который `map2`-ит каждый элемент в хвост аккумулированного выходного списка — как конвейер на заводе, где каждый болт приваривается к общей раме. Начальный аккумулятор — пустой список, поднятый в `G` через `unit` аппликатива `Applicative[G]`, чтоб не срать ошибками на старте:

```scala
given listTraverse: Traverse[List] with
extension [A](as: List[A])
override def traverse[G[_]: Applicative, B](f: A => G[B]): G[List[B]] =
val g = summon[Applicative[G]]
as.foldRight(g.unit(List[B]()))((a, acc) => f(a).map2(acc)(_ :: _))
def map[B](f: A => B): List[B] =
as.map(f)
```

Инстанс `Option` — вообще песня, матчим оригинальный опционал. Если `Some(a)`, то валим `f(a)`, а потом `Some`-конструктор через `map` на результате — и `G[B]` превращается в `G[Option[B]]`, как по маслу. А если `None`, то поднимаем этот `None` в `G` через `unit` аппликатива и забываем, жизнь — боль:

```scala
given optionTraverse: Traverse[Option] with
extension [A](oa: Option[A])
override def traverse[G[_]: Applicative, B](
f: A => G[B]
): G[Option[B]] =
oa match
case Some(a) => f(a).map(Some(_))
case None
=> summon[Applicative[G]].unit(None)
def map[B](f: A => B): Option[B] =
oa.map(f)
```

С `Tree` посложнее, тут ветвистая рекурсия как в семейном древе с побочками. Сразу бьём `f` по голове — получаем `G[B]`. Потом `traverse` на хвосте поддеревьев, рекурсивно по каждому — выходит `G[List[B]]`. `map2` их склеивает в один `G[Tree[B]]`, как пазл из ада собирает:

```scala
given treeTraverse: Traverse[Tree] = new:
extension [A](ta: Tree[A])
override def traverse[G[_]: Applicative, B](f: A => G[B]): G[Tree[B]] =
f(ta.head).map2(ta.tail.traverse(a => a.traverse(f)))(Tree(_, _))
def map[B](f: A => B): Tree[B] =
Tree(f(ta.head), ta.tail.map(_.map(f)))
```
