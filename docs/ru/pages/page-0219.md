```scala
if f(a) then Passed
else Falsified(a.toString, i)
catch
  case e: Exception =>
    Falsified(buildMsg(a, e), i)
.find(_.isFalsified)
  .getOrElse(Passed)

```

> Когда тест накрывается пиздецом, запиши этот гребаный провальный кейс и его индекс — чтоб знать, сколько тестов проскочило до него без трупов. Если тестовый кейс рождает exception, тоже зафиksiруй в результате. Это генерит бесконечный ленивый список значений A, повторно сэмпля из генератора. Вспомни, что `run` на `State` эвалирует state action — как запуск дизеля на холостых.

```scala
def randomLazyList[A](g: Gen[A])(rng: RNG): LazyList[A] =
  LazyList.unfold(rng)(rng => Some(g.run(rng)))
```

> Это синтаксис интерполяции строк, пацаны. Строка с `s"` позволяет пихать Scala-значение `v` как `$v` или `${v}` прямо в текст. Компилятор Scala разворачивает эту хуйню в `v.toString` — чисто магия под капотом.

```scala
def buildMsg[A](s: A, e: Exception): String =
  s"test case: $s\n" +
  s"generated an exception: ${e.getMessage}\n" +
  s"stack trace:\n ${e.getStackTrace.mkString("\n")}"
```

Заметили, как мы ловим exceptions и репортем их как тестовые фейлы, а не даём `run` разъебаться с ошибкой (а это сольёт всю инфу о том, какой аргумент подставил свинью). Как в проде: лучше поймать и локализовать, чем тонуть в стек-трейсе без адреса.

### 8.2 Минимизация тестовых случаев

Раньше мы заикались про минимизацию тестовых случаев — это когда фреймворк, как хирург, вырезает самый мелкий и простой фейлящий кейс, чтоб проблема светилась как на ладони и дебаг не превращался в квест по аду. Давай подкрутим наши репрезентации, чтоб это работало. Два главных подхода на выбор, как в баре: пиво или вискарь.

- **Shrinking (сжатие)** — Нашёл фейлящий кейс? Запускай отдельную мясорубку: последовательно урезай размер, пока не перестанет ломаться. Это *shrinking*, короче, и обычно жрёт отдельный код под каждый тип данных — пиши сам, как дедовские шампуры точить.

- **Sized generation (генерация с размером)** — Забей на постфактум-обрезку, генери тесты сразу по возрастанию размера и сложности: стартуй с песчинки, расти как на дрожжах, пока не нарвёшься на фейл. Идею можно тюнинговать — пусть раннер прыгает по размерам большими шагами, как кенгуру, но минимальный труп всё равно выловит.
