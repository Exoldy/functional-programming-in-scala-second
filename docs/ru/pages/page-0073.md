> Часть 1: Вступление в функциональное программирование / Глава 3: Функциональные структуры данных / 3.3 Шаринг данных в функциональных структурах данных / 3.3.2 Рекурсия по спискам и апгрейд до высших функций

`foldRight` примет в аргументах значение-аккумулятор (`acc`), которое слить для пустого списка (`Nil`), и функцию (`f`), которая приклеит элемент к результату хвоста для непустого.[^7]

## Листинг 3.2. Правые фолды и простые юзкейсы

```scala
def foldRight[A, B](as: List[A], acc: B, f: (A, B) => B): B =
as match
case Nil => acc
case Cons(x, xs) => f(x, foldRight(xs, acc, f))
def sumViaFoldRight(ns: List[Int]) =
foldRight(ns, 0, (x,y) => x + y)
```

![Страница 0073, изображение 1](./assets/page-0073/image-01.png)

> _ * _ — это покороче запись для `(x, y) => x * y`.

```scala
def productViaFoldRight(ns: List[Double]) =
foldRight(ns, 1.0, _ * _)
```

`foldRight` не завязан на конкретный тип элементов, и пока обобщаем, доходит озарение: возвращаемый тип может быть вообще не из той оперы, что элементы списка! Один способ описать, что вытворяет `foldRight`, — он просто подменяет конструкторы списка, `Nil` и `Cons`, на твои `acc` и `f`, как нарисовано тут:

```scala
Cons(1, Cons(2, Nil))
f
(1, f
(2, acc))
```

Давай разберём полный пример на костях. Проследим, как вычисляется  
`foldRight(Cons(1, Cons(2, Cons(3, Nil))), 0)((x, y) => x + y)` —  
путём подстановки определения `foldRight` вместо каждого его вызова. Такие трэйсы будем гонять по всей книге, чтоб не путаться в рекурсии, как в первом пиве после спринта:

```scala
foldRight(Cons(1, Cons(2, Cons(3, Nil))), 0, (x,y) => x + y)
1 + foldRight(Cons(2, Cons(3, Nil)), 0, (x,y) => x + y)
1 + (2 + foldRight(Cons(3, Nil), 0, (x,y) => x + y))
1 + (2 + (3 + (foldRight(Nil, 0, (x,y) => x + y))))
1 + (2 + (3 + (0)))
6
```

> Заменяем `foldRight` на его определение.

Заметим, что `foldRight` прёт до самой жопы списка (наваливая фреймы на стек рекурсии по пути), прежде чем начнёт сворачивать эту телegu обратно. Имя `foldRight` как раз отсылка к тому, как фолд стартует с правого края и жрёт список справа налево, как правый ассоциативный оператор в чистом виде — никаких тебе левой рекурсии, чтоб стек не рвало.

[^7]: В стандартной либе Scala `foldRight` — это метод на `List`, аргументы каррированы, юзается как `as.foldRight(acc)(f)`. В Scala 2 инференс типов работал по каждому параметр-листу по очереди — сначала тип `acc` подтягивался, а потом он уже сужал тип `f`. В Scala 3 инференс не такой скованный, и мотивация для кучи параметр-листов меньше, но синтаксический сахар никуда не делся: higher-order функции (функции высшего порядка) с финальным аргументом-функцией в отдельном списке позволяют писать `as.foldRight(acc): a => …` или `as.foldRight(acc) { a => … }` — чистый кайф, без лишнего скобочного ада.
