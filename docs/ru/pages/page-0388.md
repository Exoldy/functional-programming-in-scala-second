В Scala `readLine` — это деф с сайд-эффектом, как пьяный бармен, который хапает строку с консоли и кидает тебе `String` в ответ. Можем завернуть вызов в `IO`, но куда, на хуй, сунуть этот результат? У нас пока нет монстра, чтоб выразить такой эффект. Проблема в том, что наш текущий I/O — как слепой котёнок: интерпретатор просто чихает `Unit` и на том всё. Сдаться и скатиться обратно в сайд-эффекты, как лохи? Ни хуя подобного! Расширяем `IO` параметром типа, чтоб впихнуть ввод:

```scala
trait IO[A]:
self =>
def unsafeRun: A
def map[B](f: A => B): IO[B] = new:
def unsafeRun = f(self.unsafeRun)
def flatMap[B](f: A => IO[B]): IO[B] = new:
def unsafeRun = f(self.unsafeRun).unsafeRun
```

Теперь вычисление `IO` может родить настоящее осмысленное `A` через `unsafeRun`, а не просто пернуть `Unit`. Закинули `map` и `flatMap` — и вуаля, `IO` влетает в for-comprehensions, как ракета, и вообще формирует полноценный `Monad`:3
