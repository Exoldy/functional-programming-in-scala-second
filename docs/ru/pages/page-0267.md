Теперь реализация `or` просто заглядывает в флаг `isCommitted` перед тем, как спустить второго парсера с цепи. В комбинаторе `p` `or` `p2`, если `p` отстрелялся успешно, то вся эта херня succeeds целиком. Если `p` обосрался в committed-состоянии, мы фейлимся на раз-два и пропускаем `p2` — типа, не трать время на фигню. А если `p` фейл в uncommitted, то пускаем `p2` в бой и сливаем результат `p` в /dev/null:

```scala
extension [A](p: Parser[A]) def or(p2: => Parser[A]): Parser[A] =
l => p(l) match
case Failure(e, false) => p2(s)
case r => r
```

### 9.6.5 Контекстно-зависимое парсинг

Теперь финальный примитив из нашего списка: `flatMap`. Вспомним, что `flatMap` позволяет контекстно-зависимым парсерам выбирать второго парсера на основе результата первого — как в жизни, где от первого впечатления зависит, кого звать на второе свидание. Реализация простая, как дважды два: сдвигаем локацию перед вызовом второго парсера. Опять юзаем хелпер `advanceBy` на `Location`. Есть один подвох — если первый парсер сожрал хоть один символ, заставляем второго committed'нуться через хелпер `addCommit` на `ParseError`. Иначе он может отвалиться по-тихому, а мы даже не поймём.

![Изображение страницы 0267 №2](./assets/page-0267/image-02.png)

Листинг 9.3. Юзаем `addCommit`, чтоб наш парсер точно был committed

> Сдвигаем локацию источника перед вызовом второго парсера.

```scala
extension [A](p: Parser[A]) def flatMap[B](f: A => Parser[B]): Parser[B] =
l => p(l) match
case Success(a, n) => f(a)(l.advanceBy(n))
.addCommit(n != 0)
.advanceSuccess(n)
case f @ Failure(_, _) => f
```

> Коммитимся, если первый парсер сожрал хоть символ.

> Если успешно, инкрементим счётчик сожранных символов на n — за то, что уже слопал p.

> Поскольку Failure extends Result[Nothing], а Result ковариантен по единственному параметру, можем вернуть f напрямую — Scala-магия в деле.

`advanceBy` имеет очевидную реализацию. Просто инкрементим offset, без лишней хуйни:

```scala
def advanceBy(n: Int): Location =
copy(offset = offset + n)
```

Тут же `addCommit`, определённый на `ParseError`, — прямолинейно, как код на первом собеседовании:

```scala
def addCommit(isCommitted: Boolean): Result[A] = this match
case Failure(e, c) => Failure(e, c || isCommitted)
case _ => this
```
