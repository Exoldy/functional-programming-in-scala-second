#### УПРАЖНЕНИЕ 6.7

*Сложное*: Если ты уже умеешь склеивать два RNG-действия, то хули не всю толпу из списка? Реализуй `sequence`, чтоб комбайнить `List` действий в одно жирное. Используй это, чтоб переписать `ints`, которую ты раньше намутил. Для последней юзай стандартную `List.fill(n)(x)`, чтоб слепить список с `x`, повторённой `n` раз:

```scala
def sequence[A](rs: List[Rand[A]]): Rand[List[A]]
```

### 6.4.2 Вложение state-действий

Начинает вырисовываться паттерн, бля, как в том меме про emergent gameplay: мы прём к имплементациям, где `RNG` не торчит мордой наружу и не таскается по рукам, как бомжовский рюкзак. Функции `map` и `map2` дали нам элегантно и кратко набросать то, что раньше было сплошным пиздецом с ошибками и копипастой. Но есть функции, которые на `map` и `map2` не накатать — чисто по типу. Взять ту же `nonNegativeLessThan`, которая генерит инт от 0 (включительно) до `n` (не включая). Вот она, сука:

```scala
def nonNegativeLessThan(n: Int): Rand[Int]
```

Первая наводка на имплементацию — генерить неотрицательный инт и взять по модулю `n`:

```scala
def nonNegativeLessThan(n: Int): Rand[Int] =
  map(nonNegativeInt)(_ % n)
```

Это даст число в диапазоне, но с перекосом, как пьяный матрос на палубе — потому что `Int.MaxValue` может не делиться на `n` нацело. Числа меньше остатка выпадут чаще, чем надо. А когда `nonNegativeInt` выдаст число круче наибольшего кратного `n`, что влезет в 32-битный инт, — надо перегенерить и молиться на меньшее. Попробуем так:

> Рекурсивно перегенерируй (retry), если инт вышел больше наибольшего кратного `n`, что влезет в 32-битный `Int`.

```scala
def nonNegativeLessThan(n: Int): Rand[Int] =
  map(nonNegativeInt): i =>
    val mod = i % n
    if i + (n-1) - mod >= 0 then
      mod
    else
      nonNegativeLessThan(n)(???)
```

Это уже в правильную сторону, но `nonNegativeLessThan(n)` имеет не тот тип, помнишь? Должна возвращать `Rand[Int]` — функцию, которая жрёт `RNG`. А у нас нет `RNG`, чтоб скормить результату `nonNegativeLessThan(n)`. Нам надо их зачейнить (chain), чтоб `RNG` из `nonNegativeInt` полетел в рекурсивный вызов `nonNegativeLessThan`.
