# Страница 0465

> Часть 4: Эффекты и I/O / Глава 15: Обработка стримов и инкрементальный I/O / 15.3 Расширяемые пулы и стримы / 15.3.3 Гарантия безопасности ресурсов

Короче, стрим `s` может накрыться по трём причинам, как старая телега на бездорожье:

- **Истощение продюсера** (producer exhaustion), когда источник выдохся и больше не выдаст ни хуя ценного.
- **Досрочное завершение** (early termination), когда остаток источника просто выкидывают в помойку.
- **Аномальное завершение** (abnormal termination), когда вылетает exception, и всё летит к чертям.

Не важно, откуда прилетело, — в любом случае ресурсы под капотом надо закрыть наглухо. Теперь, когда правила на столе, как это заюзать на практике? Чтобы справляться с досрочным сваливанием, нужен механизм, который запустит cleanup-действия стрима даже если его выкинули нахуй. Это намекает на дизайн, где cleanup-акции для захваченных ресурсов трекает интерпретатор, а не лепит их примитивно в стрим через `flatMap` и `handleErrorWith`, как в каком-то императивном аду. Давай введём новый конструктор стрима, который ловит этот паттерн за яйца:

```scala
def resource[F[_], R](acquire: F[R])(release: R => F[Unit]): Stream[F, R]
```

Операция `resource` клепает стрим из одного элемента от `acquire` и гарантирует, что `release` сработает строго по нашим правилам — без подвохов, как в продакшене после 16 лет ебли с FP. А как реализовать `resource`? В остатке секции разберём один подход шаг за шагом, но не парьтесь деталями имплемента — фокус на поведении `resource`, а не на том, как оно сварено внутри. Начнём с понятия `scope` (scope) — области, где стрим интерпретируется. `Scope` бывает *открытым* или *закрытым*. Открытый `scope` тащит за собой cleanup-акцию, которую зовём `finalizer` (finalizer) — типа, "последний гвоздь в крышку гроба". У открытого `scope` может быть куча подсекопов. Регистрируем новый финализатор — открываем подсекоп с ссылкой на него. Закрываем открытый `scope` — сначала рекурсивно закроем все подсекопы, потом запустим финализатор. Поскольку финализаторы — это значения типа `F[Unit]`, тип `Scope` параметризуем нашим эффектом. Чтобы жонглировать состояниями — открывать подсекопы, закрывать дерево `scope`-ов, — нужен манипулятор состояния в эффекте `F`. Делаем это через новый тип — `Ref` (Ref):

```scala
final class Ref[F[_], A] private (
  underlying: AtomicReference[A],
  delay: [A] => (() => A) => F[A]
):
```

> Поднимает произвольный thunk (thunk) в эффектную акцию — как лифт из чистого мира в грязный I/O-ад

```scala
def get: F[A] = delay(() => underlying.get)
def set(a: A): F[Unit] = delay(() => underlying.set(a))
def modify[B](f: A => (A, B)): F[B] = delay: () =>
  @annotation.tailrec
  def loop(): B =
    val oldA = underlying.get
    val (newA, result) = f(oldA)
    if underlying.compareAndSet(oldA, newA) then
      result
    else
      loop()
  loop()
```
