# Страница 0293

> Часть 3: Общие структуры в функциональном дизайне / Глава 10: Монойды / 10.5 Типклассы

Мы выдернули параметр `Monoid[B]` в отдельный параметр-лист и пришили к нему ключевое слово `using`. Такие параметр-листы, что валят с `using`, определяют *контекстные параметры*.7

Эти контекстные параметры не пихаешь в функцию явно — Scala сама шарит по *given-инстансам* для каждого такого параметра прямо на месте вызова. А given-инстансы, или просто givens, клепаются с помощью ключевого слова `given`. Давай определим `intAddition` как given для `Monoid[Int]`:

```scala
given intAddition: Monoid[Int] with
def combine(a1: Int, a2: Int) = a1 + a2
val empty = 0
```

Имя для given-инстанса — опционалка,8 так что можно и так заюзать:

```scala
given Monoid[Int] with
def combine(a1: Int, a2: Int) = a1 + a2
val empty = 0
```

С любой из этих дефиниций можно юзать `foldMap`, не называя монойд явно — как будто Scala угадывает твои мысли:

```scala
val strings = List("abra", "ca", "dabra")
val charCount = foldMap(strings)(_.length)
```

Scala просекает, что функция, которую ты пихнул в `foldMap`, возвращает `Int`, и сразу лезет искать given-инстанс для `Monoid[Int]`. Находит `intAddition` — и все, кайф. А где шарит Scala? Алгоритм хитрый, как квантовая запутанность, но по-простому:

 В текущем лексическом скоупе — на предмет дефинишенов given или импортов given;

 В компаньонах типов, что торчат в контекстном параметре.

Вторая фича — чистый gold. Ищешь `Monoid[Int]`, не нарыл ничего в скоупе — Scala нырнет в компаньоны и `Monoid`, и `Int` за нужным инстансом. Используем это по полной: все канонические инстансы запихаем в компаньон `Monoid`, но помни — канонический только один, не множь сущностей. Переделаем пару старых инстансов:

```scala
object Monoid:
given intAddition: Monoid[Int] with
def combine(a1: Int, a2: Int) = a1 + a2
val empty = 0
```

7 Контекстные параметры — фича чисто Scala 3. В Scala 2 вместо них юзали implicit-параметры, помните этот цирк?  
8 Scala сама насрёт имя для безымянных givens. В остальной книге будем лепить именованные, чтоб не путаться.
