## 7 1.1 Понимание плюсов функционального программирования

### 1.1.2 Функциональное решение: Выметаем побочки нахуй

Функциональный подход — это как раз выкинуть все эти побочные эффекты в топку и заставить `buyCoffee` возвращать не только `Coffee`, но и саму зарядку как чистое значение. А вся херня с обработкой — отправкой в кредитку, сохранением в логах, персистенсом и прочей бюрократией — пусть где-то в другом углу системы вертится, не лезет в святая святых. Функциональное решение может выглядеть вот так, пацаны:

![Страница 0036, изображение 1](./assets/page-0036/image-01.png)

> Теперь `buyCoffee` возвращает пару — `Coffee` и `Charge`, тип `(Coffee, Charge)`. Система платежей тут вообще не светится, как призрак в машине. Чтобы слепить пару, кидаем чашку и `Charge` в скобки через запятую, классика жанра.

```scala
class Cafe:
def buyCoffee(cc: CreditCard): (Coffee, Charge) =
val cup = Coffee()
(cup, Charge(cc, cup.price))
```

Вот так мы развели по бабам заботы: *создание* зарядки отделяем от её *обработки* или *интерпретации* — чистый развод, без драм. Функция `buyCoffee` теперь выдаёт `Charge` как значение вместе с `Coffee`. Скоро увидите, как это позволяет переиспользовать её для покупки пачки кофе в одной транзакции, без этой импровизированной хуйни. А что за `Charge` такая? Это дата-тип, который мы только что на коленке слепили: внутри `CreditCard` и `amount`, плюс удобная функция `combine` для склейки зарядок с одинаковой `CreditCard` — как конструктор Лего для платежей:

![Страница 0036, изображение 2](./assets/page-0036/image-02.png)

> `if`-выражение синтаксически как в Java, но возвращает значение из взятой ветки — не просто управляшка, а полноценный экспрессор результата. Если `cc == other.cc`, то `combine` выдаст `Charge(..)`, иначе exception из `else` вылетит с визгом.

> `case class` имеет один primary constructor сразу после имени класса (`Charge` тут). Параметры из списка — публичные, неизменяемые (immutable) поля, достаёшь через OO-точку, типа `other.cc`. Без геттеров-сеттеров, как в нормальном мире.

```scala
case class Charge(cc: CreditCard, amount: Double):
def combine(other: Charge): Charge =
if cc == other.cc then
Charge(cc, amount + other.amount)
else
```

> Синтаксис `throw` — один в один с Java и кучей других языков. Но позже разберём по-настоящему функциональные способы обработки ошибок, без этих exception-танцев шамана.
