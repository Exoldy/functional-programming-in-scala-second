## 125 6.4 Более охуенный API для state-действий

```scala
def map[A, B](s: Rand[A])(f: A => B): Rand[B] =
  rng =>
    val (a, rng2) = s(rng)
    (f(a), rng2)
```

Смотрите, пацаны, как `map` в деле крутится — вот `nonNegativeEven`,  
она `nonNegativeInt` перехватывает и `map`-ит её,  
чтоб нагенерить `Int` не отрицательную и чётную на два,  
без этой modulo-ебли с отрицательными,  
как в дешёвом рандомайзере из 90-х:

![Страница 0154, изображение 1](./assets/page-0154/image-01.png)

```scala
def nonNegativeEven: Rand[Int] =
  map(nonNegativeInt)(i => i - (i % 2))
```

### 6.4.1 Комбинирование state-действий

Беда в том, что `map` слабовата, чтоб вытянуть `intDouble` и `doubleInt` из 6.3 —  
не вывезет, как старый сервер под лоадом.  
Приходится пилить новую `map2`, которая два `RNG`-экшена сольёт в один бинаркой,  
а не унарной хуйнёй, чтоб комбайнить по-настоящему.

![Страница 0154, изображение 2](./assets/page-0154/image-02.png)

#### УПРАЖНЕНИЕ 6.6

Допили `map2` по этой сигнатуре.  
Функция берёт два экшена — `ra` и `rb`, — плюс `f` для склейки их результатов,  
и возвращает свежий экшен, который их комбит в один кулак:

```scala
def map2[A, B, C](ra: Rand[A], rb: Rand[B])(f: (A, B) => C): Rand[C]
```

`map2` допиливаем разок — и вперёд, комбайним любые `RNG` state-акшены,  
как конструктор Лего в продакшене.  
Допустим, один экшен генерит `A`, другой — `B`, бац — и пары `A` с `B` готовы,  
без лишнего стейта в башке:

```scala
def both[A, B](ra: Rand[A], rb: Rand[B]): Rand[(A, B)] =
  map2(ra, rb)((_, _))
```

С этим добром `intDouble` и `doubleInt` из 6.3 переписываем короче,  
чем твой любимый one-liner (однострочник) на Reddit:

```scala
val randIntDouble: Rand[(Int, Double)] =
  both(int, double)

val randDoubleInt: Rand[(Double, Int)] =
  both(double, int)
```
