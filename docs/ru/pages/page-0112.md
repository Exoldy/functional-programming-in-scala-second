## 83 4.4 Тип данных Either

```scala
def sequence[E, A](as: List[Either[E, A]]): Either[E, List[A]]
def traverse[E, A, B](as: List[A])(
f: A => Either[E, B]): Either[E, List[B]]
```

Напоследок, в качестве финального примера: вот как `map2` пашет в деле, где функция `Person.make` сначала перетряхнёт имя и возраст на валидность, а уж потом слепит годный `Person`. Без этого — сплошной пиздец с полувариантами.

Листинг 4.4. Использование `Either` для валидации данных

![Страница 0112, изображение 1](./assets/page-0112/image-01.png)

```scala
case class Name private (value: String)
object Name:
def apply(name: String): Either[String, Name] =
if name == "" || name == null then Left("Name is empty.")
else Right(new Name(name))
```

> `Name` и `Age` — кейс-классы с приватными конструкторами, так что плодить их можно только в компаньонах. Метод `apply` в каждом компаньоне сначала проверит входные данные, а потом уж выдаст значение.

```scala
case class Age private (value: Int)
object Age:
def apply(age: Int): Either[String, Age] =
if age < 0 then Left("Age is out of range.")
else Right(new Age(age))
case class Person(name: Name, age: Age)
object Person:
def make(name: String, age: Int): Either[String, Person] =
Name(name).map2(Age(age))(Person(_, _))
```

### 4.4.1 Накопление ошибок

Реализация `map2` может отчитаться только об одной ошибке, даже если оба аргумента — полная хуйня, то есть оба `Left`s`)`. А было бы заебись накопить и вывалить все косяки разом. Скажем, лепишь `Person` из имени и возраста, оба накосячили в валидации — юзеру под нос весь букет ошибок, чтоб не гадал, где собака зарыта. Давай замутим функцию а-ля `map2`, но с полным репортом. Придётся тип возврата подкрутить под `List[E]`:

```scala
def map2Both[E, A, B, C](
a: Either[E, A],
b: Either[E, B],
f: (A, B) => C
): Either[List[E], C] =
(a, b) match
case (Right(aa), Right(bb)) => Right(f(aa, bb))
case (Left(e), Right(_)) => Left(List(e))
case (Right(_), Left(e)) => Left(List(e))
case (Left(e1), Left(e2)) => Left(List(e1, e2))
```
