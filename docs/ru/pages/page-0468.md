## 439 15.3 Расширяемые пулы и стримы

Теперь замутим интеграцию `Scope` в `Stream` и `Pull`.  
Стартуем с добавления двух свежих кейсов в enum `Pull`:

```scala
case OpenScope[+F[_], +O, +R](
  source: Pull[F, O, R],
  finalizer: Option[F[Unit]]
) extends Pull[F, O, R]

case WithScope[+F[_], +O, +R](
  source: Pull[F, O, R],
  scopeId: Id,
  returnScope: Id
) extends Pull[F, O, R]
```

`OpenScope` — это ссылка на source pull и его финализатор, типа как аквариум с золотой рыбкой и крючком на выходе.  

Как только интерпретатор упирается в `OpenScope`, он орёт `open` на активном скоупе, впаривая финализатор.  

Потом ныряет интерпретировать source pull, где свеженародный скоуп становится новым боссом.  

А `WithScope` — это source pull, ID скоупа для его обработки (как временный гараж) и ID, куда свалить обратно, когда шоу закончится.  

Из этих новинок логично вытекает, что интерпретатор (он же `fold` и `step`, наши старые кореша) должен тащить за собой активный скоуп и позволять степам его менять на лету, как флаг в захвате флага.  

Чтобы это прокатило, результат `step` надо доработать — впихнуть туда новый активный скоуп.  

Можно было бы перелопатить тип результата с `Either[R, (O, Pull[F, O, R])]` на `(Scope[F], Either[R, (O, Pull[F, O, R])])`, но это как переписать весь Cats Effect под себя — пиздец, короче.  

Вместо этого слепим новый тип для результата степа, чтоб не раздувать всё до небес:

```scala
enum StepResult[F[_], +O, +R]:
  case Done(scope: Scope[F], result: R)
  case Out(scope: Scope[F], head: O, tail: Pull[F, O, R])
```

Давай подкрутим `fold`, чтоб он рулил активным скоупом как заправский дирижёр.  

Надо слепить рут-скоуп, закрыть его после фолда (чтоб не утекло, как память в GC), и менять активный скоуп на каждом `StepResult.Out`, чтоб жизнь не была скучной:

```scala
def fold[F2[x] >: F[x], R2 >: R, A](init: A)(f: (A, O) => A)(
  using F: MonadThrow[F2]
): F2[(R2, A)] =
  val scope = Scope.root[F2]
  def go(scope: Scope[F2], p: Pull[F2, O, R2], acc: A): F2[(R2, A)] =
    p.step(scope).flatMap:
      case StepResult.Done(_, r) => F.unit((r, init))
      case StepResult.Out(newScope, hd, tl) =>
        go(newScope, tl, f(init, hd))
  go(scope, this, init).attempt.flatMap(res =>
    scope.close.flatMap(_ => res.fold(F.raiseError, F.unit))
  )
```

Операция `step` теперь жрёт активный скоуп как аргумент, а результат `step` тычет пальцем в следующий скоуп — классика "кто следующий?".  

Подкрутить `step` под `OpenScope` — дело нехитрое, без подвохов: делегируем `open` на активном скоупе, а потом юзаем `WithScope` для оценки
