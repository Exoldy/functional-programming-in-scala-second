## 12.3 Разница между монадами и аппликативными функторами

```scala
val temp: Parser[Double] = ...
val row: Parser[Row] = d.map2(temp)(Row(_, _))
val rows: Parser[List[Row]] = row.sep("\n")
```

Если порядок колонок — хуй поймёшь заранее, и приходится ковыряться в хедере, чтоб эту херню вытащить, то без `flatMap` никуда. Вот пример файла, где колонки, блядь, в обратном порядке перепутаны:

```scala
# Temperature, Date
25, 1/1/2010
28, 2/1/2010
42, 3/1/2010
53, 4/1/2010
...
```

Чтобы распарсить такую хуйню, где парсер строки (Row)-приходится подбирать динамически — сначала хедер проглотить (ту первую строку, что с `#` начинается), — нам позарез нужен `flatMap`:

```scala
case class Row(date: Date, temperature: Double)
val d: Parser[Date] = ...
val temp: Parser[Double] = ...
val header: Parser[Parser[Row]] = ...
val rows: Parser[List[Row]] =
header.flatMap(row => row.sep("\n"))
```

Тут мы хедер жрём, он нам `Parser[Row]` как результат отрыгивает. Потом этим парсером остальной табличный фарш пережёвываем. Порядок колонок заранее — тёмный лес, так что парсер строки (Row) выбираем на лету, по свежему результату хедера. Разница между `Applicative` и `Monad` — она везде торчит, как баг в продакшене. Конечно, типы всё расскажут, алгебра интерфейсов — как рентген, но пацанам на код-ревью проще по-простому:

- Вычисления в `Applicative` — структура жёсткая, как рельсы, эффекты просто в цепочку нанизываются, а монадические — дикий запад, структура подбирается динамически по ходу предыдущих эффектов, блядь.

- `Applicative` лепит *контекстно-независимые* (context-free) вычисления, как контекстно-свободные грамматики (CFG) в универе, а `Monad` впускает *чувствительность к контексту*.²

- `Monad` делает эффекты первоклассными (first-class) гражданами; их генеришь на этапе интерпретации, а не вперёд кодом планируешь. Помним наш `Parser`-пример? Там `Parser[Row]` родили прямо в процессе парсинга и этим же `Parser[Row]` дальше жевали.

² Например, монадический парсер тянет контекстно-зависимые (context-sensitive) грамматики, а аппликативный — только контекстно-свободные (context-free), без этих изысков.
