![Страница 0462, изображение 1](./assets/page-0462/image-01.png)

#### УПРАЖНЕНИЕ 15.12

Определи законы для `MonadThrow`. Подумай, как `attempt` должен взаимодействовать с `raiseError` и `unit` — чтоб не было сюрпризов, как в том меме с `flatMap`, который вдруг мутировал в `for`-comp.

Теперь апдейтнем дефолт `step`, чтоб он жрал инстанс `MonadThrow` — без этого никуда, пацаны, в реальном проде без хендлинга ошибок стрим — как машина без тормозов:

```scala
def step[F2[x] >: F[x], O2 >: O, R2 >: R](
  using F: MonadThrow[F2]
): F2[Either[R2, (O2, Pull[F2, O2, R2])]] =
  this match
    ...
    case Handle(source, f) =>
      source match
        case Handle(s2, g) =>
          s2.handleErrorWith(x =>
            g(x).handleErrorWith(y => f(y))).step
        case other =>
          other.step
            .map:
              case Right((hd, tl)) => Right((hd, Handle(tl, f)))
              case Left(r) => Left(r)
            .handleErrorWith(t => f(t).step)
```

> Перепиши левостыкованные хендлеры в правостыкованные — чтоб не было той левой рекурсии, от которой мозг плавится.
>
> Обрабатывай ошибки, которые вылезли при степе исходного пула.

Эта имплементация сначала чекнет левостыкованные хендлеры ошибок и перевернёт такую херню в правостыкованную, как мы делаем с левостыкованными вызовами `flatMap` — помните, как в Cats Effect это спасало от stack overflow в проде? Иначе степнем оригинальный пул и обработаем любую подставу[^6], делегируя в метод `handleErrorWith` для таргет-эффекта `F`. Если ошибок не случилось, пропагаем хендлер в хвост пула — чисто, как в нормальной монаде. 

Из-за того, что `step` теперь жрёт `MonadThrow`, все элиминаторы тоже апгрейдим `MonadThrow`: `fold`, `toList` и `run`. По аналогии закинем саппорт для явного рейзинга ошибок в пул и стрим через новый конструктор `raiseError` для каждого — код главы покажет все мясные детали, не ленитесь чекнуть. 

Теперь с `handleErrorWith` и `raiseError` для стрима можем норм дефайнить `onComplete`:

```scala
extension [F[_], O](self: Stream[F, O])
  def onComplete(that: => Stream[F, O]): Stream[F, O] =
    self.handleErrorWith(t => that ++ raiseError(t)) ++ that
```

Метод `onComplete` даёт заценить стрим после финиша исходного, хуй там — успешно ли он доскакал или фейльнулся с ошибкой.

[^6]: Эта имплементация не ловит исключения (exceptions), кинутые самим хендлером (ну, когда `f(t)` решил поебенить и throw'нул).
