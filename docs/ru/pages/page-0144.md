def mapViaUnfold[B](f: A => B): LazyList[B] =
unfold(this):

Для `take` мы юзаем оригинальный список и счётчик оставшихся элементов, которые ещё надо взять — классика, чтоб не плодить лишние копии, как в том меме про "не копируй, если можно ссылаться".

```scala
def takeViaUnfold(n: Int): LazyList[A] =
unfold((this, n)):
case (Cons(h, t), 1) => Some((h(), (empty, 0)))
case (Cons(h, t), n) if n > 1 => Some((h(), (t(), n-1)))
case _ => None
```

А `takeWhile`, как и `map`, просто держит оригинальный список как состояние — минимум гемора, максимум лени, пацаны, я сам через это в проде проходил, когда drop жрал стек без остановки.

```scala
def takeWhileViaUnfold(f: A => Boolean): LazyList[A] =
unfold(this):
case Cons(h, t) if f(h()) => Some((h(), t()))
case _ => None
```

Для `zipAll` берём оба ленивых списка как состояние. На каждой итерации unfold патчим структуру этого состояния и расписываем кейсы для всех комбинаций `Empty` и `Cons` — как разбираешь заваленный конструктор в Cats, чтоб ничего не упустить. Например, если оба списка на `Cons`, кидаем `Some`, чтоб сказать "всё, поезд приехал". А если оба с элементами — возвращаем `Some` с головами, упакованными в `Some`, как значение для эмиссии, и хвосты пихаем в новое состояние. В ответе юзнули `=>`, чтоб скобок поменьше — `a` `=>` `b` это то же самое, что `(a,` `b)`, чисто сахар для глаз на код-ревью.

```scala
def zipAll[B](that: LazyList[B]): LazyList[(Option[A], Option[B])] =
unfold((this, that)):
case (Empty, Empty) => None
case (Cons(h1, t1), Empty) =>
Some((Some(h1()) -> None) -> (t1() -> Empty))
case (Empty, Cons(h2, t2)) =>
Some((None -> Some(h2())) -> (Empty -> t2()))
case (Cons(h1, t1), Cons(h2, t2)) =>
Some((Some(h1()) -> Some(h2())) -> (t1() -> t2()))
```

Тут куча годных поводов для обобщений, бля: можно завести вариант `zip`, который стопорится, как только любой из списков кончился, или зиппить элементы с функцией вместо возврата `(Option[A],` `Option[B])` — чтоб не просто склеивать, а сразу маппить, как в реальном продакшене с Stream'ами из Akka.
