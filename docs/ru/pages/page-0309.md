> Часть 3: Общие структуры в функциональном дизайне / Глава 10: Моноиды / 10.9 Ответы на упражнения

```scala
as.foldRight(acc)(f)

override def foldLeft[B](acc: B)(f: (B, A) => B) =
  as.foldLeft(acc)(f)
```

Реализация `IndexedSeq` делает то же самое, но ещё переопределяет `foldMap`, используя `foldMapV`, чтоб результат вычислить по-умному и эффективно, без этой ленивой хуйни (`lazy evaluation`), которая жрёт стек как бешеная:

```scala
given Foldable[IndexedSeq] with
  extension [A](as: IndexedSeq[A])
    override def foldRight[B](acc: B)(f: (A, B) => B) =
      as.foldRight(acc)(f)
    override def foldLeft[B](acc: B)(f: (B, A) => B) =
      as.foldLeft(acc)(f)
    override def foldMap[B](f: A => B)(using mb: Monoid[B]): B =
      Monoid.foldMapV(as, mb)(f)
```

Реализация `LazyList` похожа на `List`, просто делегирует `foldLeft` и `foldRight` подложенному типу данных — классический прокси-паттерн (proxy pattern), чтоб не дублировать код, как лохи:

```scala
given Foldable[LazyList] with
  extension [A](as: LazyList[A])
    override def foldRight[B](acc: B)(f: (A, B) => B) =
      as.foldRight(acc)(f)
    override def foldLeft[B](acc: B)(f: (B, A) => B) =
      as.foldLeft(acc)(f)
```

#### ОТВЕТ 10.13

Реализуем все три операции рекурсией по веткам — через это говно все мы прошли в FP, когда стек чуть не рвал нахуй, но моноиды спасают:

```scala
given Foldable[Tree] with
  import Tree.{Leaf, Branch}
  extension [A](as: Tree[A])
    override def foldRight[B](acc: B)(f: (A, B) => B) = as match
      case Leaf(a) => f(a, acc)
      case Branch(l, r) => l.foldRight(r.foldRight(acc)(f))(f)
    override def foldLeft[B](acc: B)(f: (B, A) => B) = as match
      case Leaf(a) => f(acc, a)
      case Branch(l, r) => r.foldLeft(l.foldLeft(acc)(f))(f)
    override def foldMap[B](f: A => B)(using mb: Monoid[B]): B = as match
      case Leaf(a) => f(a)
      case Branch(l, r) => mb.combine(l.foldMap(f), r.foldMap(f))
```
