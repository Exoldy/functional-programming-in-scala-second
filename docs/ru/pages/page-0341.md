#### ОТВЕТ 11.20

```scala
opaque type Reader[-R, +A] = R => A
object Reader:
def ask[R]: Reader[R, R] = r => r
def apply[R, A](f: R => A): Reader[R, A] = f
extension [R, A](ra: Reader[R, A])
def run(r: R): A = ra(r)
extension [R, A](ra: Reader[R, A])
def run(r: R): A = ra(r)
given readerMonad[R]: Monad[Reader[R, _]] with
def unit[A](a: => A): Reader[R, A] = _ => a
extension [A](fa: Reader[R, A])
override def flatMap[B](f: A => Reader[R, B]) =
r => f(fa(r))(r)
```

Тип `Reader` — это opaque-алиас для функции `R` `=>` `A`. Не стоит думать о `Reader` как о банальной функции — лучше представь его как вычисление, которое сплёвывает значение типа `A`, имея под рукой окружение типа `R` (environment, он же env, как config в продакшене). То есть, это computation, которое может почитать env `R` при расчёте `A`. По сути, чистый dependency injection — без этой хуйни с мутабельным стейтом, чтоб не ебаться потом с race conditions. Похоже на `State`, с которым мы ковырялись, только env (он же state) только на чтение, никаких изменений, чтоб не разъебать цепочку.

Его базовые операции — это как кирпичи для постройки:

- Подъём функции `R` `=>` `A` в `Reader[R,` `A]` через `Reader.apply` — классический map, чтоб не плодить сущностей.

- Запуск всей хуйни с env типа `R` через `reader.run(r)` — run, чтоб наконец-то получить результат.

С помощью `Reader.apply` лепим `ask`, просто закинув identity-функцию — и вуаля, `Reader[R,` `R]`, ридер, который просто возвращает env себе в зубы, как эхо в пещере. `unit(a)` возвращает функцию, которая плюёт на env и выдаёт `a` — чистый unit, игнор всё нахуй. `flatMap(f)` строит ридера, который раздаёт env и оригиналу, и тому ридеру, что родится из `f` — как broadcast в Kafka, чтоб все в курсе. `sequence` берёт список ридеров и склеивает в один, раздавая env каждому по кругу и собирая результаты в общий список — sequence, чтоб не дёргать по одному, как лузер. А `join` — это вообще огонь, если вспомнить, что `Reader[R,` `A]` — алиас для `R` `=>` `A`. Значит, `Reader[R,` `Reader[R,` `A]]` — это синоним `R` `=>` `R` `=>` `A`, или короче `(R,` `R)` `=>` `A`. Короче, `join` пропускает env бинарной функции — как аппликативный zip, но с env в кармане. `replicateM(n)` генерит ридера, который вычисляет результат из env и дублирует его `n` раз — replicate, чтоб не пересчитывать заново, лол. Монадно-законы дают гарантию: рефакторишь ридер-цепочки хоть в хлам, результат не изменится — это как immutable стейт в ZIO, прошёл через это в проде, знаю, где собака зарыта.
