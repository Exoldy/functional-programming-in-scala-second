> Часть 4: Эффекты и I/O / Глава 15: Обработка потоков и инкрементальный I/O / 15.3 Расширяемые пуллы (pulls) и потоки / 15.3.1 Вычисления с эффектами в стриминге

```scala
object Pull:
  def unfoldEval[F[_], O, R](
    init: R
  )(
    f: R => F[Either[R, (O, R)]]
  ): Pull[F, O, R]
```

Короче, наконец-то апдейтим `Pipe`, чтоб он эффекты учитывал, как нормальный пацан:

```scala
type Pipe[F[_], -I, +O] = Stream[F, I] => Stream[F, O]
```

Теперь у нас выбор по типу `Pipe` — чисто дизайн-дело. 
Мы впихнули один параметр типа `F[_]` прямо в определение `Pipe`, и он описывает эффект и для входного стрима, и для выходного. 
Из-за этого приходится ставить его инвариантно (попробуй ковариантно — компилятор тебе такую порцию ошибок выдаст, что кофе из носа полезет). 
Но за это теряем чуток общей мощи. 
Альтернатива — определить `Pipe` вот так:

```scala
type Pipe[-F[_], -I, +G[_], +O] = Stream[F, I] => Stream[G, O]
```

Это, конечно, выразительнее, но и запутаннее, как спагетти после трёхдневного хардкорного кодинга. 
А поскольку `Pipe` — это просто алиас, всегда можно юзать явный функциональный тип, когда припрёт. 
Так что остаёмся на простом варианте, без изысков.

### 15.3.1 Вычисления с эффектами в стриминге

Теперь, когда `Pull`, `Stream` и `Pipe` научились эффекты эвалировать, можно строить вычисления, которые по чуть-чуть тянут данные из источника, трансформируют их на лету и сливают в сток. 
Источники — это стримы, трансформации — пайпы, а стоки как представлять? 
Ха, и не нужно ничего специального! 
Метод `mapEval` из упражнения 15.10 — это база для стоков с эффектами. 
По конвенции сток — это пайп, который меняет выход на `Unit` или `Nothing`, по сути, жрёт каждый элемент из исходного стрима. 
Вот примеры стоков, чтоб было понятно:

```scala
def log[O]: Pipe[IO, O, Unit] =
  _.mapEval(o => IO(println(o)))
def drain[F[_], O]: Pipe[F, O, Nothing] =
  _.flatMap(_ => Stream.empty)
```

Сток `log` юзает `mapEval`, чтоб каждый выходной элемент из источника на консоль выкидывать. 
А `drain` — просто всё в /dev/null, как старый добрый игнор. 
Собираем пазл воедино — и вуаля, стриминговое вычисление с эффектами:

```scala
case class Message(
  id: UUID,
  timestamp: Instant,
  ...
)

def format(m: Message): String =
  s"id: $id, timestamp: $timestamp, ..."
```
