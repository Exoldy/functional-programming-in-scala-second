## 325 12.5 Законы аппликативов

С этой новой абстракцией мы можем подкорректировать наш инстанс `Applicative[Validated[E, _]]`, чтоб он требовал только `Semigroup[E]`:

```scala
given nelSemigroup[A]: Semigroup[NonEmptyList[A]] with
  def combine(x: NonEmptyList[A], y: NonEmptyList[A]) =
    NonEmptyList(x.head, x.tail ++ (y.head :: y.tail))

given validatedApplicative[E: Semigroup]: Applicative[Validated[E, _]] with
  def unit[A](a: => A) = Valid(a)

extension [A](fa: Validated[E, A])
  override def map2[B, C](fb: Validated[E, B])(f: (A, B) => C) =
    (fa, fb) match
      case (Valid(a), Valid(b)) => Valid(f(a, b))
      case (Invalid(e1), Invalid(e2)) =>
        Invalid(summon[Semigroup[E]].combine(e1, e2))
      case (e @ Invalid(_), _) => e
      case (_, e @ Invalid(_)) => e
```

Можем ли мы так заебать? Ебать, да! Когда выдираешь новую абстракцию из старой, как мясо из кости, тащи с собой и алгебраические законы. Иначе выйдет хуйня полная — операция есть, а толку ноль, никаких derived-операций (derived operations) с гарантией, что они сработают на любом инстансе. Давай-ка выдернем это для `Semigroup`. Для моноидов мы задали три закона: ассоциативность, левый и правый нейтрал. Ассоциативность можно вынести в определение `Semigroup`, и остаётся вот что:

- Семигруппа (Semigroup) для типа `A` — это ассоциативная операция `combine`, которая из двух входных `A` выдаёт `A`.

- Моноид (Monoid) — это семигруппа с нейтральным элементом `empty`, таким что `combine(empty, a) == combine(a, empty) == a` для всех `a: A`.

Зачем строить слабые абстракции из сильных, как слоёный пирог из бетона? Как мы видели с `NonEmptyList`, слабые часто имеют больше инстансов — те, что не влезут в сильные из-за лишних кандалов. Как связь между `Semigroup` и `Monoid`, абстракция `Applicative` слабее `Monad`, даёт больше инстансов и следовательно больше универсальности. `Validated` с аппликативом, но без монады — это как `NonEmptyList` с семигруппой, но без моноида.

### 12.5 Законы аппликативов

В этой секции пробежимся по законам аппликативных функторов.<sup>5</sup> Хотите — проверьте, что они держатся на тех типах данных, с которыми мы ковырялись раньше (лёгкий кейс — `Option`).
