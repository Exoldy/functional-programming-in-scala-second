## 331 12.7 Применения Traverse

- `Tree[Option[A]] => Option[Tree[A]]`  
  (вызов `Traverse[Tree].sequence` с `Option` в роли `Applicative[F]`)  
  возвращает `None`, если хоть один входной `Tree` — `None`;  
  иначе выдаёт `Some` с `Tree` всех значений из входных `Tree`.

- `Map[K, Par[A]] => Par[Map[K, A]]`  
  (вызов `Traverse[Map[K, _]].sequence` с `Par` в роли `Applicative[F]`)  
  запускает параллельный компьютинг, который вычисляет все значения мапы параллельно,  
  как настоящие пацаны в пуле потоков.

Оказывается, пиздец какая куча операций можно определить максимально общо,  
чисто через `sequence` и `traverse` — без лишней хуйни.  
Разберём их в следующей секции.  
Траверс — это как фолд-братан: берёт структуру данных, прогоняет по ней функцию, выплёвывает результат.  
Только вот `traverse` бережёт оригинальную форму, как мамка твой скелет,  
а `foldMap` её нахуй выкидывает и лепит из неё моноидные операции, как из Lego фарш.  
Взгляни на сигнатуру `Tree[Option[A]] => Option[Tree[A]]` — мы структуру `Tree` сохраняем,  
а не просто значения в моноид сливаем, чтоб не различить, где был List, а где Set.

#### УПРАЖНЕНИЕ 12.14

*Хардкор*: Реализуй `map` через `traverse` как метод на `Traverse[F]`.  
Это докажет, что `Traverse` — расширение `Functor`, а функция `traverse` — обобщение `map`  
(потому их иногда *traversable functors* (траверсабельные функторы) зовут, чтоб не путать с простыми функторами).  
Обратите внимание: при реализации `map` зови `traverse` с `Applicative[G]` на твой вкус:

```scala
trait Traverse[F[_]] extends Functor[F]:
extension [A](fa: F[A])
def traverse[G[_]: Applicative, B](f: A => G[B]): G[F[B]] =
fa.map(f).sequence
def map[B](f: A => B): F[B] = ???
extension [G[_]: Applicative, A](fga: F[G[A]])
def sequence: G[F[A]] =
fga.traverse(ga => ga)
```

А какая связь между `Traverse` и `Foldable`?  
Ответ кроется в связке между `Applicative` и `Monoid` — классический FP-трюк,  
через который я сам спотыкался, пока не въехал.
