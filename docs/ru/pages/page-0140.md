## 111 5.6 Ответы на упражнения

Эта реализация стек-сейф (stack-safe)? На первый взгляд — хуй там, рекурсивный вызов не в хвостовой позиции, стек вот-вот обоссытся. Но рекурсия висит в ленивом ахуе, пока не форсишь хвост возвращаемого ``Cons``, так что стек в безопасности, пацаны. 

Функцию ``drop`` можно забабахать чистой хвостовой рекурсией, чтоб вообще не париться:

```scala
@annotation.tailrec
final def drop(n: Int): LazyList[A] = this match
case Cons(_, t) if n > 0 => t().drop(n - 1)
case _ => this
```

Это один в один как определяем ``drop`` на ``List``, только опять же — не форсим вычисления дропнутых элементов, чтоб не тратить циклы на фигню.

#### ОТВЕТ 5.3

``takeWhile`` лепим так же, как ``take``, без лишнего пиздеца:

```scala
def takeWhile(p: A => Boolean): LazyList[A] = this match
case Cons(h, t) if p(h()) => cons(h(), t().takeWhile(p))
case _ => empty
```

![Страница 0140, изображение 2](./assets/page-0140/image-02.png)

#### ОТВЕТ 5.4

Берём ``foldRight`` так же, как для ``exists``, и форсим ``b`` только если предикат вернул ``true`` на текущем элементе — чтоб не ебаться с ненужным:

![Страница 0140, изображение 3](./assets/page-0140/image-03.png)

```scala
def forAll(p: A => Boolean): Boolean =
foldRight(true)((a, b) => p(a) && b)
```

#### ОТВЕТ 5.5

```scala
def takeWhile(p: A => Boolean): LazyList[A] =
foldRight(empty)((a, b) => if p(a) then cons(a, b) else empty)
```

Стартуем фолд с пустого ленивого списка. По каждому элементу: если предикат отвалил ``true``, то строим ``Cons`` из этого элемента и остатка компьютации, который прилетел в ``b`` нашей фолдинговой хуйне.
