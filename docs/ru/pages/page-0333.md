> Часть 3: Общие структуры в функциональном дизайне / Глава 11: Монды / 11.7 Ответы на упражнения

Чтобы заимплементить `traverse`, давайте сперва глянем на нашу реализацию `Option` из четвёртой главы — ту, что мы там намутили:

```scala
def traverse[A, B](as: List[A])(f: A => Option[B]): Option[List[B]] =
as.foldRight(Some(Nil): Option[List[B]])((a, acc) =>
map2(f(a), acc)(_ :: _))
```

Ещё заценим имплементацию `State` из шестой главы:

```scala
def traverse[S, A, B](as: List[A])(f: A => State[S, B]): State[S, List[B]] =
as.foldRight(unit[S, List[B]](Nil))((a, acc) => f(a).map2(acc)(_ :: _))
```

Обе эти хрени — это правый фолд по элементам списка: берёшь каждый элемент, пихаешь в функцию, а результат конс в накопленный список через `map2`. Короче, классика жанра. А `traverse` для любой монады лепим через `unit` и `map2` — вот так:

```scala
def traverse[A, B](as: List[A])(f: A => F[B]): F[List[B]] =
as.foldRight(unit(List.empty[B]))((a, acc) => f(a).map2(acc)(_ :: _))
```

#### ОТВЕТ 11.4

Можно замутить по-разному. Давайте сперва рекурсивный вариант зафигачим, как в старые добрые времена:

```scala
def replicateM[A](n: Int, fa: F[A]): F[List[A]] =
if n <= 0 then unit(Nil)
else fa.map2(replicateM(n - 1, fa))(_ :: _)
```

Работает, но с подковыркой, блядь. Рекурсивный вызов не в хвостовой позиции — на больших `n` стек просто нахрен порвёт, привет OOM. А если стек нахуй забить, то видно, что логика дублируется, как в дешёвом коде-ревью: базис на `unit`, шаг на `map2` и `::`. Короче, части `traverse` переписали заново, сам через это говно проходил. Избежим этой хуйни с дублированием:

```scala
def replicateM[A](n: Int, fa: F[A]): F[List[A]] =
sequence(List.fill(n)(fa))
```

Тут мы слепили список из `n` копий `fa`, типом `List[F[A]]`. Потом секвенсим его в `F[List[A]]`, типа местами `List` и `F` перекинули — чисто алхимия монад.
