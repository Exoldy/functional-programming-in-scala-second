Это почти один в один с той предыдущей реализацией `unit`, которую мы специально слепили под `Rand`. Единственная реальная разница — в сигнатуре типа, где вместо хардкода `Rand` теперь универсальный параметр `S`[^6]. Как апгрейд от костыля к нормальному инструменту, без лишнего гемора.

В упражнении 6.9 мы уже просекли, что `map` и `map2` можно набить на базе `flatMap`, так что их имплементации просто портируем в методы расширения (extension methods) — копипаст с минимальными правками. То же самое и с `flatMap`, подкручиваем только типы да имена. А теперь, когда у нас есть полноценные методы для `map` и `flatMap` вместо этих левых standalone-функций, лепим `map2` через for-выражение (for-comprehension) — чистый кайф, как сахарный сироп на FP-панкейке:

```scala
extension [S, A](underlying: State[S, A])
  def run(s: S): (A, S) = underlying(s)
  def map[B](f: A => B): State[S, B] =
    flatMap(a => unit(f(a)))
  def map2[B, C](sb: State[S, B])(f: (A, B) => C): State[S, C] =
    for
      a <- underlying
      b <- sb
    yield f(a, b)
  def flatMap[B](f: A => State[S, B]): State[S, B] =
    s =>
      val (a, s1) = underlying(s)
      f(a)(s1)
```

Наконец, `sequence` вешаем на объект-сопровождаемый (companion object) `State`, ковыряя минимально, чтоб параметр `S` везде прокатился без сбоев:

```scala
def sequence[S, A](states: List[State[S, A]]): State[S, List[A]] =
  states.foldRight(unit[S, List[A]](Nil))((s, acc) => s.map2(acc)(_ :: _))
```

#### УПРАЖНЕНИЕ 6.11

Начинаем с правил перехода состояний — пишем их как функцию от `Input` и текущего `Machine`, которая сплевывает новое `Machine`. Каждое правило реализуем через паттерн-матч (pattern match) на инпуте и текущем состоянии машины — классика, как в старом добром автомате из аркады:

```scala
def update(i: Input, s: Machine): Machine =
  (i, s) match
    case (_, Machine(_, 0, _)) => s
    case (Input.Coin, Machine(false, _, _)) => s
    case (Input.Turn, Machine(true, _, _)) => s
    case (Input.Coin, Machine(true, candy, coin)) =>
      Machine(false, candy, coin + 1)
    case (Input.Turn, Machine(false, candy, coin)) =>
      Machine(true, candy - 1, coin)
```
