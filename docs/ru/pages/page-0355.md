### 12.5.1 Левый и правый идентификатор

Какие законы аппликативные функторы должны соблюдать, пацаны? Бля, однозначно, они обязаны уважать законы функторов, это как базовый контракт, без него вся херня развалится:

```scala
x.map(a => a) == x
x.map(f).map(g) == x.map(f andThen g)
```

Это подразумевает ещё кучу других законов для аппликативов, потому что мы реализуем `map` через `map2` и `unit` — классика, как в том меме про "map и flatMap, которые правят миром". Вспомним определение `map`:

```scala
extension [A](fa: F[A])
def map[B](f: A => B): F[B] =
fa.map2(unit(()))((a, _) => f(a))
```

Конечно, в этом определении есть доля произвола — мы могли бы так же легко запихнуть `unit` слева от вызова `map2`, и никто бы не пикнул:

```scala
extension [A](fa: F[A])
def map[B](f: A => B): F[B] =
unit(()).map2(fa)((_, a) => f(a))
```

Первые два закона для `Applicative` можно свести к тому, что обе эти имплементации `map` уважают законы функторов. Другими словами, `map2` некоторого `fa:` `F[A]` с `unit` сохраняет структуру `fa` — как скелет, который не гнётся под нагрузкой. Назовём их законами левого и правого идентификатора (показаны здесь в первой и второй строках кода соответственно, чтоб не путаться):

```scala
unit(()).map2(fa)((_, a) => a) == fa
fa.map2(unit(()))((a, _) => a) == fa
```

### 12.5.2 Ассоциативность

Чтобы узреть следующий закон — ассоциативность, — глянем на сигнатуру `map3`, она как старый добрый кореш, который всегда в теме:

```scala
extension [A](fa: F[A])
def map3[B,C,D](
fb: F[B],
fc: F[C]
)(f: (A, B, C) => D): F[D]
```

Можем слепить `map3` из `apply` и `unit`, но давайте подумаем, как её определить через `map2` — чисто по-фпшному, без костылей. Эффекты комбинируем по двое, и выборов два: слить `fa` с `fb`, а потом эту хрень с `fc`, или ассоциировать наоборот — сгруппировать `fb` с `fc` и добить `fa`. Закон ассоциативности для аппликативов говорит: результат один хрен одинаковый, независимо от скобок. Это ж как ассоциативность монад и моноидов (или полугрупп), помните, как в Cats мы это ковыряли до посинения?

```scala
combine(a, combine(b, c)) == combine(combine(a, b), c)
compose(f, compose(g, h)) == compose(compose(f, g), h)
```
