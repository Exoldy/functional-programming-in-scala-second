Now that we have `map2`, is `many` really primitive? Let’s think about what `p.many` will do. It tries running `p`, followed by `p.many` again, then again, and so on, until the attempt to parse `p` fails. It’ll accumulate the results of all successful runs of `p` into a list. As soon as `p` fails, the parser returns the empty `List`.

Now let’s try to implement `many`. Here’s an implementation in terms of `|`, `map2`, and `succeed`:

```scala
extension [A](p: Parser[A]) def many: Parser[List[A]] =
p.map2(p.many)(_ :: _) | succeed(Nil)
```

This code looks nice and tidy. We’re using `map2` to say we want `p` followed by `p.many` again and that we want to combine their results with `::` to construct a list of results. Or if that fails, we want to `succeed` with the empty list—but there’s a problem with this implementation. Can you spot what it is?
