#### ANSWER 12.14

Let’s compare the signature of `map` to `traverse`:

```scala
def map[B](f: A => B): F[B]
def traverse[G[_]: Applicative, B](f: A => G[B]): G[F[B]]
```

Note that `traverse` is similar to `map` but introduces an extra type constructor `G`. Can we select a type for `G` that causes the type constructor to evaporate? Consider a slightly different version of the `Id` type we saw in the previous chapter, this time defined as a type alias instead of a case class:

```scala
type Id[A] = A
given idMonad: Monad[Id] with
def unit[A](a: => A) = a
extension [A](a: A)
override def flatMap[B](f: A => B): B = f(a)
```

Now let’s substitute `Id` for `G` in the signature of `traverse`:
