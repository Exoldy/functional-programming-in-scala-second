### 10.1 What is a monoid?

Let窶冱 consider the algebra of string concatenation. We can add `"foo"` `+` `"bar"` to get `"foobar"`, and the empty string is an *identity element* for that operation. That is, if we say `(s` `+` `"")` or `(""` `+` `s)`, then the result is always `s`. Furthermore, if we combine three strings by saying `(r` `+` `s` `+` `t)`, then the operation is *associative*; it doesn窶冲 matter whether we parenthesize it: `((r` `+` `s)` `+` `t)` or `(r` `+` `(s` `+` `t))`. The same applies for multiplication, whose identity element is `1`. The Boolean operators `&&` and `||` are likewise associative, and they have identity elements `true` and `false`, respectively. These are just a few simple examples, but algebras like this are virtually everywhere. The term for this kind of algebra is *monoid*, and the laws of associativity and identity are collectively called the *monoid laws*. A monoid consists of the following:

We can express this with a Scala trait:

```scala

```

> Satisfies combine(x, empty) == x and combine(empty, x) == x
