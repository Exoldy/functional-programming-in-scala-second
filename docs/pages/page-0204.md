#### ANSWER 7.6

We could first filter the list with the supplied predicate and lift the resulting list into `Par`:

```scala
def parFilter[A](as: List[A])(f: A => Boolean): Par[List[A]] =
unit(as.filter(f))
```

This solution doesn’t have any parallelism, though. We could try to fix that by using `lazyUnit` instead of `unit`:

```scala
def parFilter[A](as: List[A])(f: A => Boolean): Par[List[A]] =
lazyUnit(as.filter(f))
```

This is a little better, as the filtering is done on a different logical thread, but all filtering is done on the same logical thread. We really want each invocation of the predicate to get its own logical thread. Like in `parMap`, we can use `asyncF`, but this time, instead of passing `f` directly, we pass an anonymous function that converts an `A` into a `List[A]`—a single element list if the predicate passes or an empty list otherwise:

![Page 0204 image 2](./assets/page-0204/image-02.png)

> Like in parMap, we fork immediately, so the mapping over the original list is done on a separate logical thread rather than the caller’s thread.

```scala
def parFilter[A](as: List[A])(f: A => Boolean): Par[List[A]] =
fork:
val pars: List[Par[List[A]]] =
as.map(asyncF(a =>
if f(a) then List(a) else Nil))
sequence(pars).map(_.flatten)
```

> We use asyncF to convert our A => List[A] function to an A => Par[List[A]] function.

> sequence(pars) returns a Par[List[List[A]]], so we map over that and flatten the inner nested lists.
