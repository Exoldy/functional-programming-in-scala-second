> Part 4: Effects and I/O / Chapter 13: External effects and I/O / 13.9 Conclusion

## 389 Summary

Finally, note that real-world `IO` monads, like the `IO` type provided by the Cats Effect library (https://mng.bz/p2vw), typically have much more functionality than we built here, including better support for concurrency, cancellation, and resource management.

### Summary

´éíIt is always possible to factor an impure procedure into a pure core function and two procedures with side effects: one that supplies the pure functionÔÇÖs input and one that does something with the pure functionÔÇÖs output. In other words, inside every function with side effects is a pure function waiting to get out.

´éíThe first `IO[A]` type developed in this chapter has a number of benefits, despite its simple implementation. It allows computations to be expressed as ordinary values, which allows them to be stored in lists, passed to functions, created dynamically, and so on. The interpreter of our computations is defined separately from the computations themselves, allowing the interpreter to be changed without affecting client code.

´éíThe first `IO[A]` type developed in this chapter also has a number of problems: itÔÇÖs not stack safe; itÔÇÖs completely opaque, disallowing all inspection; and it doesnÔÇÖt support concurrency.

´éíTrampolining can be used to make monads stack safe.

´éíThe `TailRec[A]` type uses trampolining to provide stack-safe function composition.

´éíThe `Async[A]` type is similar to `TailRec[A]` but allows composition of `Par[A]` values instead of functions.

´éíThe `Free[F[_],` `A]` type generalizes `TailRec[A]` and `Async[A]`, providing a way of sequencing primitive operations of the supplied `F[_]` algebra.

´éíFree monads allow us to sequence primitive operations defined by an algebra into a value representing a program and later interpret that value into a target monad.

´éíFree monads limit the expressive power of a computation to sequencings of the primitive operations of the algebra.

´éíPolymorphic function types are function types parameterized by one or more type parameters.

´éíFree monads of different algebras can be composed into free monads of unioned algebras. For example, `Free[[x]` `=>>` `Files[x]` `|` `Console[x]]` describes a computation that may use operations from either the `Files` algebra or the `Console` algebra.

´éíCapability traits, also known as tagless final interpreters, allow us to compose programs consisting of various primitive I/O algebras, while retaining the local reasoning and constrained power benefits of free monads.

´éíThe general-purpose `IO` data type allows us to maintain referential transparency while writing programs that perform input and outputÔÇöprograms that, at first glance, appear to be inextricably linked to side effects and incompatible with pure functional programming.
