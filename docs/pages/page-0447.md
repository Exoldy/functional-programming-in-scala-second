> Part 4: Effects and I/O / Chapter 15: Stream processing and incremental I/O / 15.2 Simple stream transformations / 15.2.1 Creating pulls

We can build lots of interesting pull transformations by combining recursive functions with `uncons`. Here’s how we can transform the output elements of a pull:

```scala
def mapOutput[O2](f: O => O2): Pull[O2, R] =
uncons.flatMap:
case Left(r) => Result(r)
case Right((hd, tl)) => Output(f(hd)) >> tl.mapOutput(f)
```

We can use the same approach for adding or removing the elements of a pull. Here’s a combinator that filters the output elements based on a predicate:

```scala
def filter(p: O => Boolean): Pull[O, R] =
uncons.flatMap:
case Left(r) => Result(r)
case Right((hd, tl)) =>
(if p(hd) then Output(hd) else Pull.done) >> tl.filter(p)
```

We can even maintain state in our pull transformations by passing the state as an argument to our recursive function:

```scala
def count: Pull[Int, R] =
def go(total: Int, p: Pull[O, R]): Pull[Int, R] =
p.uncons.flatMap:
case Left(r) => Result(r)
case Right((_, tl)) =>
val newTotal = total + 1
Output(newTotal) >> go(newTotal, tl)
Output(0) >> go(0, this)
```
