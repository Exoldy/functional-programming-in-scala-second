## Local effects and mutable state

![Page 0423 image 2](./assets/page-0423/image-02.png)

### This chapter covers

Introducing mutable state

Discussing the `ST` monad

Providing a better definition of referential transparency

In the first chapter of this book, we introduced the concept of referential transparency, setting the premise for purely functional programming. We declared that pure functions can’t mutate data in place or interact with the external world. In chapter 13, we learned this isn’t exactly true; we can write purely functional and compositional programs that describe interactions with the outside world. These programs are unaware that they can be interpreted with an evaluator that has an effect on the world. In this chapter, we’ll develop a more mature concept of referential transparency. We’ll consider the idea that effects can occur locally inside an expression and that we can guarantee no other part of the larger program can observe these effects occurring. We’ll also introduce the idea that expressions can be referentially transparent with regard to some programs and not others.
