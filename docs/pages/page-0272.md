## 243 9.8 Exercise answers

Note that we don’t need an implementation of `defer`. We assume it’s a primitive to be supplied by implementations of the `Parsers` trait.19

```scala
extension [A](p: Parser[A])
def many: Parser[List[A]] =
p.map2(defer(p.many))(_ :: _) | succeed(Nil)
```

In the parallelism chapter, we were particularly interested in avoiding having `Par` objects that took as much time and space to build as the corresponding serial computation, and the `delay` combinator let us control this more carefully. This isn’t much of a concern here, and having to think carefully each time we `map2` to decide whether we need to call `defer` seems like unnecessary friction for users of the API.

#### ANSWER 9.6

Let’s break this problem up into two pieces—parsing a nonnegative integer and then parsing a specified number of `'a'` characters:

> Parse one or more digit characters using a regular expression.

```scala
val nonNegativeInt: Parser[Int] =
for
nString <- regex("[0-9]+".r)
n <- nString.toIntOption match
case Some(n) => succeed(n)
case None => fail("expected an integer")
yield n
```

> Convert the parsed digit string to an integer.

> If integer conversion succeeded, return the parsed value as a parser via succeed. Otherwise, return a parser, which fails with an error message.

We used `flatMap` on the regex parser to further validation and transform the parsed string—in this case, converting the string to an integer. We had to create a new primitive, `fail`, which constructs a parser that always fails with the supplied error message. With `nonNegativeInt` we can use `flatMap` to implement `nConsecutiveAs` using various combinators we’ve already defined:

```scala
val nConsecutiveAs: Parser[Int] =
for
n <- nonNegativeInt
_ <- char('a').listOfN(n)
yield n
```
